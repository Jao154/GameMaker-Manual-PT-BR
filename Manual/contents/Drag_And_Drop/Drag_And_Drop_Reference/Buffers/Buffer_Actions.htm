Biblioteca de Ação de Buffers

Um buffer é uma região de memória física usada para armazenar dados temporariamente enquanto está sendo movida de um lugar para outro ou manipulada de alguma forma, por exemplo, você pode usar um buffer para armazenar dados do estado do jogo. O GameMaker tem uma série de ações específicas relacionadas à criação, modificação e carregamento/salvamento buffers, mas antes de utilizá-los, observe os seguintes pontos:

  • Quando um buffer é criado, ele é automaticamente liberado e preenchido a 0.
  • O argumento do "tamanho" das funções buffer refere-se sempre ao tamanho em bytes do buffer.
  • GML Visual buffers estão sempre 1 byte alinhados. O "alinhamento" refere-se a como os dados são armazenados dentro de um buffer e é usado quando se usa as funções do GML buffer , mas para o GML Visual, não é relevante.
  • Os dados em buffers são lidos e escritos de forma seqüencial (ou seja: você chama a ação Write Buffer três vezes, depois escreverá os primeiros dados, avançará, à direita os segundos dados, avançará e depois escreverá os últimos dados), com algo chamado de valor "procurar" para dizer onde no buffer ocorrerá a próxima leitura/escrita.
  • Todos os dados são escritos no buffer e retornados do buffer como uma string, portanto, se você tiver armazenado algum número real, você terá que usar uma das Ações de Tipo de Dados para convertê-lo do string para o valor.

A memória utilizada para criar buffers é a memória do sistema, portanto, mesmo quando o jogo não tem foco, qualquer dado armazenado em um buffer deve ser seguro, no entanto, se o aplicativo for fechado ou reiniciado, ele será perdido. Também vale a pena notar que buffers também pode ser salvo como arquivos e depois carregado de volta ao jogo usando as ações do arquivo.
