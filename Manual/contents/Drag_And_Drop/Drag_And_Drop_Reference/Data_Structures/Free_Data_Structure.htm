Estrutura de dados livre

Esta ação libertará a estrutura de dados da memória e deve ser chamada sempre que uma estrutura de dados não for mais necessária, por exemplo, no Destroy mesmo de uma instância ou no Room End event, etc... Liberar uma estrutura de dados significa que nenhuma outra ação pode ser usado para referenciá-lo, pois ele não existirá mais, embora você também possa verificar isso usando a ação Se a Estrutura de Dados Existir.

Observe que a menos que a estrutura de dados tenha um índice atribuído a uma variável de escopo global, ela não será transportada de um room para outro. Por exemplo, se você criar uma instância de jogador e uma estrutura de dados de lista for atribuída a uma variável nessa instância, a lista precisará ser liberada se o jogador for destruído ou se a room mudar com a instância do jogador nela (a menos que a instância do jogador seja marcada como persistente), então a instância do jogador teria duas ações de Estrutura de Dados Livres nela para ambos os eventos. É importante que todas as estruturas de dados criadas sejam destruídas desta forma quando não forem mais necessárias para evitar vazamentos de memória que podem causar problemas de desempenho ou até mesmo travar seu jogo.

Argumentos:
