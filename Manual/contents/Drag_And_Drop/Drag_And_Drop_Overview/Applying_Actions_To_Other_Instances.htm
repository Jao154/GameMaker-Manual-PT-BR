Aplicação de ações em outras instâncias

A maioria das ações nas bibliotecas visuais GML tem uma opção para aplicar a ação de diferentes maneiras. Isto se chama definir o escopo da ação e pode ser uma de várias coisas:

Essencialmente, o que você está dizendo ao GameMaker é qual instância deve executar a ação. O escopo padrão da ação é self, o que significa que assim que o object for criado como instância no room, essa instância executará o código de ação. Entretanto, isto nem sempre é o que você quer, e você pode querer que algumas ações afetem outras, ou até tudo, instâncias no room. É aqui que entra a mudança do escopo de ação.

Os diferentes escopos de execução das ações estão listados abaixo, mas deve-se observar que mudar o escopo de uma ação desta forma somente aplicará o novo escopo a essa ação e não às ações subseqüentes na cadeia. Se você quiser aplicar uma mudança de escopo de ação para múltiplas ações encadeadas, então use primeiro a opção Aplicar à... ação.

Auto

Este é o escopo padrão para uma ação e simplesmente declara que a ação deve ser chamada apenas pela instância que está executando o código.

Outros

O escopo other tem duas funções principais, e o valor que ele retorna dependerá de onde e como você o utiliza. No evento de colisão, other retornará o valor exclusivo de identificação da instância (um valor único que é usado para distinguir instâncias individuais de cada object), para que você possa, por exemplo, criar uma instância "bullet" e ter um evento de colisão com uma instância "player" e, nesse caso, usar o escopo other para remover pontos de golpe do "player" object e depois retornar ao escopo self para destruir a instância "bullet".

Fora do evento de colisão, a configuração other se comportará como se estivesse configurada para noone a menos que esteja sendo chamada de dentro de um bloco de ações delimitado. O que isto significa é que se você mudar o escopo de um grupo de ações a um object específico, então enquanto essas ações estão sendo chamadas, o escopo other devolverá a identificação da instância que inicialmente chamou o grupo de ação. Por exemplo, você poderia executar um Aplicar a... ação e depois no próximo bloco de código definir o escopo para other para executar uma ação na instância executando o bloco de eventos inteiro, e não o exemplo que está sendo aplicado aos blocos de código. Todos

Quando você amplia uma ação para todos, você está dizendo ao GameMaker para executar esse bloco para cada instância ativa dentro da sala atual. Por exemplo, criando uma ação Destruir Object Instância e definir seu escopo para all fará desaparecer todas as instâncias do room, não importa de onde tenham sido criadas, object.

Objeto

Uma ação também pode ser dada a um objeto por seu escopo. O que isto significa é que todas as instâncias do dado object executarão essa ação ao mesmo tempo em que ela é chamada. Portanto, se você tiver 100 instâncias inimigas no room, por exemplo, e você quer defini-los todos para apontar para um ponto específico. Você chamaria o Set Point Direction e definiria o escopo de ação para o object "obj_Enemy" e quando ele é chamado, todas as instâncias desse object mudarão de direção.

Expressão

O campo de entrada Expressão é para que você insira a identificação de uma instância específica na qual você deseja que a ação trabalhe. Pode ser o valor de ID exclusivo atribuído a uma instância do Editor Room ou pode ser o ID de uma instância que você tenha armazenado em uma variável (onde a variável seria o valor de entrada) ou pode até mesmo ser uma expressão usando código, como mostrado na imagem abaixo:
