Opções de jogo

As Opções de Jogo são divididas em duas seções, Opções Principais e as Configurações da Plataforma. As opções principais começam com as Opções Gerais, que é onde você pode definir certas coisas que afetarão o jogo independentemente da plataforma na qual ele está sendo projetado para funcionar.

A janela tem as seguintes seções:

  • Linguagem de script padrão: Escolha se Objects eventos e Scripts no projeto atual devem usar o GML Code ou GML Visual, ou se deve sempre perguntar sobre a criação de um novo script.
  • Quadros de jogo por segundo: Este valor estabelece o limite do número de quadros por segundo que o GameMaker deve tentar manter seus jogos durante a execução. Este valor representa o número de vezes que o jogo loop será executado em um segundo (um único jogo loop é considerado um único "frame") e é definido para um valor padrão de 60, embora 30 também seja usado com freqüência. Há muitos argumentos sobre qual room velocidade é o ideal, mas a resposta real depende do tipo de jogo que você está fazendo, do tamanho do room, do número de instâncias e de quão intensivo é o código. Por exemplo, se você tiver um room grande e uma room velocidade de 30, o jogo pode parecer "agitado", pois quanto maior a área que você estiver olhando, mais fácil é para o olho perceber as baixas velocidades do room, mas se o jogo for intensivo, então ajustar a velocidade do room para 120 pode fazer com que o jogo "atrase", pois o fps cai devido à quantidade de processamento que está sendo feita. Portanto, tome cuidado com este valor e tente sempre ajustá-lo ao que seu jogo precisa.
  • Cor padrão do desenho: A cor padrão para o texto do desenho ou fóruns (e outras coisas) quando nenhuma cor tiver sido definida usando código.
  • Cor padrão do desenho fora da região room: A cor padrão que será usada para limpar o fundo de qualquer área fora do jogo room.
  • Gerar imagens do projeto: Clicando aqui, será aberta a ferramenta Gerador de Imagens do Projeto.
  • Data de início do projeto: A data em que o projeto foi iniciado.
  • Tempo de uso do projeto: O tempo decorrido desde que você iniciou o projeto.
  • GUIA DO PROJETO: Este é o identificador único que é criado para cada projeto dentro do GameMaker. Se você desejar mudar isto, você pode clicar no botão Criar novo GUID para gerar um novo GUID.
  • Gerar mipmaps para páginas texture separadas: Esta opção fará com que o GameMaker crie mipmaps para aqueles sprites que tenham sinalizado a opção de página separada Texture nas configurações Texture do editor Sprite. Habilitar o mipmaps significa que a página original texture para o sprite deve ser escalada e filtrada em múltiplas resoluções dentro do arquivo texture. Isto normalmente está desativado, mas se você estiver usando vistas para aumentar ou diminuir o zoom em grandes áreas do jogo room, ou se estiver trabalhando com um jogo 3D, então você pode querer ativar isto e então usar as Funções de Mipmapping para ajustar como o jogo lida com os diferentes níveis do mipmap. Note que se você não estiver sinalizando sprites como utilizando páginas individuais texture, você ainda pode habilitar o mipmapping em uma base de grupo por textura a partir do Gerente de Grupo Texture. Esta opção está desligada por padrão.
  • Habilitar o controle da fonte: Se você deseja que o projeto esteja sob SCM (Source Control Management), então você precisa verificar isto. Está desligado por padrão (veja aqui para mais informações em source control).
  • Autor: O nome do autor do jogo (ou editora ou estúdio).
  • Projeto Última Alteração: Esta seção simplesmente mostra o carimbo de data/hora pela última vez que o projeto foi alterado (isto não é editável).
  • Aceitar Spine Licença: Aqui você pode marcar a caixa para ser exibida a Spine Runtime Licença e aceitá-la. Isto permitirá que você utilize Spine sprites em seu jogo (esta licença também será mostrada na primeira vez que você tentar adicionar um Spine sprite ao seu projeto e não tiver verificado a licença aqui de antemão).
  • Modo de Compatibilidade de Colisão: Isto diz a GameMaker para usar o sistema de colisão antigo usado antes das versões de 2022 (que usavam valores inteiros em vez de flutuadores). Leia Modo de Compatibilidade de Colisão para mais informações. Isto é desativado por padrão e é recomendado mantê-lo desativado para novos projetos.
  • Habilitar Cópia no Comportamento de Escrita para Arrays: Isto permite o comportamento depreciado da Copy on Write array para o projeto atualmente aberto. Leia Copy on Write para mais informações. Isto é desativado por padrão e é recomendado mantê-lo desativado para novos projetos.

A seção de Opções Principais também contém um menu para Informações do Modelo.

O Navegador de Ativos também terá outras opções de jogo disponíveis para você, dependendo das plataformas alvo que estão disponíveis para a licença que você possui e nem todas elas podem estar disponíveis ou visíveis.

Configurações da plataforma

Existem as seguintes opções de jogos específicos de plataforma:

  • Ópera GX
  • Windows
  • macOS
  • Ubuntu
  • Windows UWP
  • HTML5
  • Android
  • iOS
  • tvOS

NOTA As várias opções de jogos de console não estão listadas aqui devido às várias restrições legais mantidas pelas diferentes empresas. Se você tem uma Licença de Console, então você pode obter estas informações no Helpdesk YoYo Games, mas primeiro você deve contatar YoYo Games com seu e-mail de identificação do console e solicitar permissão para visualizar as diferentes seções.

Alguns projetos do GameMaker também podem mostrar opções de jogos para o alvo do Amazon Fire. Estas são opções legadas, somente leitura, pois o GameMaker agora suporta a plataforma Fire diretamente usando as opções de jogo do Android. Se seu projeto ainda mostrar Amazon Fire, você pode encontrar informações sobre as opções na página seguinte:

  • Fogo da Amazônia (Legado)
