Arquivos Incluídos

OGameMaker não se limita apenas ao assets que você adiciona no Browser Asset, pois você também pode adicionar arquivos ao projeto como Arquivos Incluídos para que eles possam ser acessados enquanto o jogo está sendo executado. Estes são simplesmente arquivos que são incluídos com seu pacote de jogo, para que você possa fazer com eles o que desejar.

Os tipos de arquivos que você adiciona dependerão do que você quer fazer, mas coisas como arquivos padrão *.ini, sprites e imagens de fundo, ou planilhas são comumente usadas como Arquivos Incluídos.

Acréscimo de arquivos incluídos

A maneira mais simples de adicionar um Arquivo Incluído é arrastá-lo de seu Explorador/Finder para IDE. Você será solicitado a confirmar se o arquivo arrastado deve ser adicionado aos Arquivos Incluído. Após pressionar Sim, a janela Arquivos Incluídos será aberta, contendo o arquivo que você acabou de adicionar.

Janela de Arquivos Incluída

Abra o menu Extras do navegador Asset (Fig. 1), para abrir manualmente a janela Arquivos Incluídos (Fig. 2):

Esta janela mostra apenas os Arquivos Incluídos que estão presentes em seu projeto. Para reorganizar ou renomear seus arquivos, você precisará editar o diretório real onde estes arquivos estão armazenados.

Clicando em Open In Explorer/Finder, será aberta a pasta de arquivos de dados em seu diretório de projetos, que contém todos os arquivos incluídos. Você pode adicionar e remover arquivos neste diretório como desejar, e a interface em GameMaker será automaticamente atualizada para refletir a estrutura em sua pasta de arquivos de dados.

Uma vez que você tenha adicionado arquivos ao editor, você pode clicar duas vezes em sobre eles para abrir as Propriedades do Arquivo:

Esta janela mostra o nome do arquivo, seu tamanho em disco, e também as plataformas para as quais ele deve ser exportado quando você compila/testa seu jogo. Você tem botões para selecionar "Todos" ou "Nenhum", ou você pode descer na lista e verificar/desmarcar os alvos para os quais você deseja que o arquivo seja exportado.

Para maiores informações sobre como os Arquivos Incluídos são armazenados com seu jogo e como eles são acessados, consulte Sistema de Arquivos.

IMPORTANTE Ao incluir arquivos de som, você deve garantir que os arquivos evitem nomes que entrem em conflito com o som do streaming assets. Por exemplo, um som asset chamado "main_music" fará com que um Arquivo incluído chamado "main_music.ogg" se torne undefined quando o projeto for executado. Isto porque os arquivos streamed também são armazenados fora do pacote principal do projeto, portanto, é preciso ter cuidado nestas situações.

Você também pode incluir DLL (em Windows) ou DyLib (em macOS) arquivos como Arquivos Incluídos e depois criar suas próprias funções personalizadas para lidar com eles usando external_define(). Estes podem então ser usados chamando o external_call() função.

O uso de Arquivos Incluídos para criar funcionalidades extras desta forma é incompatível com o sistema de Extensão, pois você não pode usar o externo_*() funções com arquivos adicionados em um extension. Extensions assets já tem um método de adição de constantes e funções.
