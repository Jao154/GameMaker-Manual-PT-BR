Opções básicas

A base Feather Settings contém as seguintes opções:

  • Habilitar o Feather: Habilite ou desabilite Feather aqui. Quando desativado, nenhum recurso de Feather estará disponível em todo o site IDE.
  • Atraso no documento Parse Delay: O tempo que o Feather espera para verificar seu script após fazer uma alteração nele.
  • Pontas de ferramentas compactas: A ativação disto removerá certas informações de ponta de ferramenta que aparecem ao pairar sobre palavras-chave, tais como descrições de parâmetros para funções, tornando, em última análise, suas pontas de ferramenta menores.
  • O número máximo de campos struct que são exibidos pelas pontas das ferramentas: Quando você passa o mouse sobre uma referência struct, uma dica de ferramenta exibe as variáveis dentro daquele struct, até o número introduzido aqui.
  • Referências do fator a asset nome quando asset é renomeado em Asset Browser: Ao habilitar isso, você garantirá que quando você renomear um asset em seu Asset Browser, qualquer menção a ele em seu código também será editada para que essas referências não se quebrem.
  • Prefira um estilo de comentário JSDoc multilinha ao criar a partir do Quick Fix: Quando você gera comentários JSDoc para uma função através do menu Quick Fixes, o JSDoc gerado utilizará comentários de uma ou várias linhas, dependendo desta opção.
  • Habilite o modo de Tipo Estrito: Quando isto é habilitado, Feather é mais rigoroso para garantir que você utilize os tipos de dados corretos em seu código.

Severidade da mensagem

Esta seção contém todas as regras que o Feather usa para verificar seu código, e permite ajustar a severidade de cada regra.

Perfil

Nesta caixa suspensa, você pode escolher entre algumas predefinições para as Regras GM*. Os seguintes perfis estão disponíveis:

  • Nenhuma: Este perfil não mostrará mensagens de erro.
  • SintaxeErrores: Se você usar este perfil, serão mostrados erros de sintaxe, juntamente com o que é mostrado no perfil Nenhum.
  • TipoErrores: Este perfil também mostra erros de tipo, além do perfil SyntaxErrors .
  • Todos: Este perfil mostra tudo o que Feather pode analisar.

Regras GM*

Você pode definir uma regra como "Erro", "Aviso" ou "Sugestão", que afeta a forma como o código de quebra de regra é relatado a você no Editor de Código e na janela Mensagens em Pena.

Você pode estabelecer uma regra para "Ignorar" que ignorará todas as instâncias de violação dessa regra.

Todas as regras que começam com "GM1000" verificam sua sintaxe para possíveis erros fatais, e as que começam com "GM2000" são regras de melhores práticas que ajudam a evitar bugs em seu jogo.

Regras de nomenclatura

Esta seção permite definir regras de nomenclatura para assets, variáveis e várias outras partes do código GML.

A regra GM2017 sob "Message Severity" deve ser habilitada para que as regras de nomenclatura entrem em vigor. Por padrão, ela está desativada.

Os ajustes nesta seção são os seguintes:

  • Lista de Blocos de Identificação: Esta é uma lista de identificadores separada por espaço que será ignorada para todas as regras de nomenclatura.

    Por exemplo, digamos que você tem uma regra para usar obj_ como prefixo para objects, mas você quer que seu gerente e rede objects permaneça como está e não use o prefixo obj_.

    Nesse caso, você escreveria a rede de gerentes no campo Indentifier Blocklist. Feather irá ignorar aqueles objects e quaisquer outros identificadores (variáveis, enumeros, parâmetros, etc.) com os mesmos nomes.
  • <identifier> Regra de nomenclatura: Você pode definir regras de nomenclatura para cada tipo de identificador na lista, tais como assets, macros, enumeros, parâmetros de funções, variáveis, etc.

    Cada menu drop-down de regras de nomenclatura tem as seguintes opções:
    • Prefixo: Texto que deve aparecer antes do nome do identificador, que pode ser obj_ para objects, _ para variáveis locais, etc., dependendo de suas preferências.
    • Sufixo: Texto que deve aparecer após o nome do identificador.
    • Estilo de nomenclatura: Escolha o estilo de nomenclatura para o identificador. Suas opções são:
      • Sem restrições: Nenhuma formatação é forçada para este tipo de identificador.
      • UpperCamelCase: MyHealth, ObjGrappleHook, MaxInventoryItemsAllowed, etc.
      • lowerCamelCase: myHealth, objGrappleHook, maxInventoryItemsAllowed, etc.
      • ALL_UPPER: MY_HEALTH, OBJ_GRAPPLE_HOOOK, MAX_INVENTORY_ITEMS_ALLOWED, etc.
      • all_lower: my_health, obj_grapple_hook, max_inventory_items_allowed, etc.
