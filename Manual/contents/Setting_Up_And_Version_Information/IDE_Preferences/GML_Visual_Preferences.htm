O GML Visual As preferências são usadas para definir como o editor Visual se parece e se sente. As diferentes opções disponíveis estão listadas abaixo, juntamente com uma descrição do que cada uma faz.

  • Localização padrão da caixa de ferramentasVisual GML: Por padrão, a caixa de ferramentas visuais GML (onde todas as diferentes bibliotecas de ação são mantidas) é colocada à direita do Editor, mas você pode usar esta opção para tê-la por padrão à esquerda.
  • GML Modo de visibilidade geral do nóvisual: Um nó "visão geral" é o pequeno texto descritivo que aparece ao lado de certas ações na Ação Workspace, por exemplo, esta Ação de Instância de Destruição:

    Você pode optar por ativá-las sempre, ativá-las somente se a ação for colapsada (a configuração padrão), ou desativá-las completamente.
  • Resposta automática à conversão visual da GML para a linguagem GameMaker: Quando você converte umGML Visual ou script no código GML, você receberá por padrão uma mensagem avisando sobre o que está prestes a fazer e perguntando se deseja continuar. Ajustando esta opção para "OK", você simplesmente aprovará a mudança sem mostrar a mensagem.
  • Resposta automática à exclusão do nó visual GML: Quando você apaga um GML Visual Se você não tiver um nó de evento, você receberá, por padrão, uma mensagem avisando sobre o que está prestes a fazer e perguntando se deseja continuar. Configurando esta opção para "OK", você simplesmente aprovará a exclusão sem mostrar a mensagem.
  • Tamanho máximo para as miniaturas de ação da caixa de ferramentas Visual GML: Na parte inferior da caixa de ferramentas de ação você tem um controle deslizante para mudar o tamanho do GML Visual ícones. Esta configuração será usada para definir o tamanho máximo permitido (padrão 64px).
  • Tamanho mínimo para as miniaturas de ação da caixa de ferramentas Visual GML: Na parte inferior da caixa de ferramentas de ação você tem um controle deslizante para mudar o tamanho do GML Visual ícones. Esta configuração será usada para definir o tamanho mínimo permitido (padrão 16px).
  • Tamanho padrão para miniaturas de ação da caixa de ferramentas Visual GML: Esta configuração será usada para definir o tamanho padrão para todos os ícones de ação na caixa de ferramentas (padrão 32px).
  • Número máximo de ações recentemente utilizadas para exibição: Na caixa de ferramentas de ação há uma biblioteca no topo que contém as ações utilizadas mais recentemente. Você pode usar esta opção para alterar o número máximo de ações recentes que serão mostradas (padrão 5).
  • Espessura da cadeia que conecta as ações visuais da GML: Isto define a espessura do gráfico da cadeia que liga as diferentes ações em uma GML Visual bloco de código (padrão 3px).
  • Tempo de mouse para começar a arrastar os nós (ms): Ao clicar em um nó e arrastá-lo, você estará reposicionando o nó dentro do código de ação. Entretanto, se você segurar o mouse por um momento e depois mover, você estará movendo a posição do nó dentro do workspace, mas não dentro da ordem do código. Esta configuração define o número de milissegundos a ter que esperar com o botão do mouse pressionado antes que o nó possa ser movido (e não re-posicionado). O valor padrão é 350.
  • Ocultar painel de visão geral do nó visual da GML: Aqui você pode optar por ocultar o painel de visão geral do Editor de Código de Ação. Por padrão, isto é desativado e, se for habilitado, removerá do editor a lista de cadeias de ação de texto simples.
