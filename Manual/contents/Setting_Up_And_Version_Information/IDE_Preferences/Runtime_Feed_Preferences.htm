Preferências de alimentação em tempo de execução

As Preferências de Alimentação de Tempo de Execução são usadas para exibir e alterar o Tempo de Execução atualmente instalado, bem como o runtime em uso atual. O GameMaker pode ter várias ferramentas diferentes do runtime instaladas a qualquer momento e você pode definir qual delas usar a partir desta preferência. O ideal é que você queira usar o runtime mais atualizado, mas às vezes ele pode ter um problema, ou você pode ser obrigado a usar um mais antigo para comparabilidade, etc... e assim você pode mudar as ferramentas do runtime que o GameMaker usa para construir seu jogo aqui.

A seção inicial desta preferência está relacionada a onde você puxa as ferramentas runtime e exige que você insira uma URL para um feed RSS. Este feed será então escaneado para atualizações das ferramentas runtime que podem então ser baixadas e instaladas. Esta seção mostrará a URL do feed Master runtime por padrão, mas você pode clicar no botão Adicionar para criar um novo feed Runtime onde você dá o nome e a URL para o feed RSS, e então clicar em Aplicar para dizer ao GameMaker que acesse o feed. Você precisará fechar e reabrir a janela Preferências para que o novo feed seja mostrado.

Uma vez criado um alimento Runtime ele será listado nesta seção como uma subcategoria, e você deverá ter sempre pelo menos um alimento listado (por padrão, este seria o alimento Master ). As preferências de alimentação Runtime são usadas para selecionar, instalar e desinstalar as diferentes ferramentas runtime. No topo você tem as ferramentas atuais runtime sendo utilizadas, a data em que foi criada e o local em que está armazenada no disco rígido. Abaixo destes detalhes você pode encontrar uma lista de todas as runtimes disponíveis, e você pode escolher qualquer uma desta lista para mudar para ser a runtime usada para compilar seus projetos.

Para selecionar um runtime diferente para usar, basta encontrá-lo na lista e depois clicar duas vezes sobre ele. Se o runtime não tiver sido baixado e instalado, então você será solicitado a fazê-lo, e uma vez instalado será solicitado a fechar e reiniciar o IDE para que o novo runtime possa ser rubricado.
