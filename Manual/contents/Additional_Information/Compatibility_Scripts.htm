Roteiros de Compatibilidade

Para aqueles usuários provenientes de versões anteriores do GameMaker, pode ser um pouco confuso usar o GameMaker no início, já que houve uma série de mudanças feitas na forma como o projeto assets, edição e codificação são tratados. Estas mudanças significam que nem a IDE nem a linguagem GameMaker (GML) suportam os itens listados abaixo. Entretanto, isso não significa que você não pode importar jogos feitos com versões anteriores para o GameMaker, como você pode. O GameMaker converterá automaticamente todas as funções legadas em scripts de compatibilidade, que são scripts, contendo funções que convertem os métodos herdados para os métodos atualizados.

O que acontece é que o GameMaker analisará seu projeto para todas as funções e variáveis obsoletas que você utilizou, e então criará uma série de scripts usando esses nomes de funções. Estes scripts conterão ou o novo equivalente função de tempo de execução ou uma função de script curto que executa a mesma ação que a função obsoleta.

Quando você importar um projeto GameMaker pela primeira vez, você receberá um relatório de compatibilidade muito parecido com este:

O corpo principal do relatório mostra quais objects e scripts foram afetados e onde, enquanto no final você pode encontrar uma lista dos scripts que foram criados para seu projeto. Você pode clicar no botão do meio do mouse ou pressionar em qualquer um dos scripts listados no relatório para abri-lo e ver o que foi feito.

Observe que se você vir algum scripts criado com uma dupla barra inferior antes de seu nome (por exemplo: __global_object_depths), então estes são scripts que foram criados pelo IDE para ajudar na compatibilidade e não são scripts que tenham sido convertidas de funções obsoletas, portanto não devem ser adulteradas, pois são necessárias para que a outra compatibilidade scripts funcione.

Quando terminar de revisar o relatório de compatibilidade, você pode fechar o workspace, mas se precisar revisar qualquer coisa dentro do relatório, você pode encontrá-lo novamente indo para a seção Notas do Navegador de ativos.

Note que você pode continuar usando esta compatibilidade scripts assim como teria usado as funções da versão herdada de GameMaker, no entanto, devido à forma como elas funcionam, haverá um impacto de desempenho associado a isto. Recomendamos que você atualizar os projetos para usar diretamente os novos métodos e remover a compatibilidade scripts se você puder, pois isso melhorará o desempenho e também lhe dará uma visão da maneira como o GameMaker e o GameMaker Language funcionam em comparação com produtos antigos.

Observe também que há algumas funções que foram introduzidas no GameMaker Language para garantir a compatibilidade com projetos mais antigos e que aparecerão nestas compatibilidades scripts.
