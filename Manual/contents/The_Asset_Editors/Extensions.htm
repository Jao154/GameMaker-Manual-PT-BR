O Editor Extension

Um pacote de extensão pode ser usado para adicionar uma série de funções adicionais e constantes ao GameMaker Language (GML) e transferi-las facilmente entre projetos. As funções do Extension também podem ser vinculadas a SDKs externos ou código fonte escrito em outras linguagens (como Java para Android, Objective-C para iOS, etc.) que podem ser usadas para expandir a funcionalidade de seu jogo (normalmente usado para implementar APIs de terceiros, tais como para anúncios, compras emapp, e mais).

As extensões podem ser usadas para criar uma biblioteca de funções que você usa freqüentemente em todos os seus projetos, assim, em vez de ter que importar o mesmo scripts em cada projeto, você pode simplesmente adicionar um extension asset que contém uma coleção de suas funções e constantes favoritas. Tais funções também podem ser específicas para uma determinada plataforma, e você pode escolher facilmente em quais plataformas um extension pode ser usado.

Para um exemplo passo a passo da criação de um extension, veja: Criação de uma extensão

Para desativar extensions com base na plataforma alvo, veja: Como desabilitar Extensions?

Importação de extensões

Normalmente, os terceiros extensions serão do GameMaker Marketplace, e aqueles que você assinou serão então listados na janela Marketplace Library, que pode ser instalada a partir daí. Se você não obteve seu pacote extension do Marketplace, você ainda pode adicioná-lo a um projeto clicando com o botão direito do mouse em em qualquer lugar do Navegador de Ativos e selecionando Adicionar Existente, depois navegando até o local da extensão para selecioná-lo.

NOTA Se você tiver um pacote .yymps/.yymp/.gmez, você pode arrastá-lo para IDE para importá-lo, ou ir sob o menu "Ferramentas" e selecionar "Importar pacote local".

Plataformas-alvo

Uma vez que você tenha adicionado um extension ao navegador Asset, você deve dar-lhe um nome e um número de versão e também definir para quais plataformas você deseja exportá-lo. Ao desmarcar as plataformas selecionadas, você pode dizer ao GameMaker para não exportar o extension como parte do executável para essas plataformas. Você também deve assegurar que as propriedades do extension coincidam com a(s) plataforma(s) alvo selecionada(s), e selecionar alvos de exportação compatíveis para os arquivos extension com base no seguinte:

  • Todos os alvos: GML extensions. Estes são extensions feitos usando o GameMaker Language (GML) e são compatíveis com todas as plataformas alvo. (Note que este é o único tipo de extension que o alvo Opera GX suporta atualmente.)
  • Windows (Standard, UWP, Xbox): Você pode utilizar os arquivos Windows DLL (Dynamic Link Libraries). Observe que se você tiver verificado o "Use x64 Windows Runtime" nas Opções de Jogo do Windows, então todas as DLLs para o projeto devem ser compatíveis com 64 bits. As DLLs de 32 bits não funcionarão com um executável de 64 bits.
  • macOS: O alvo macOS pode usar os arquivos padrão DyLib para extensions.
  • Ubuntu (Linux): Para o alvo Ubuntu você precisa ter certeza de que seu extension usa arquivos .so.
  • HTML5: Estas plataformas podem utilizar JavaScript extensions.
  • Android: Esta plataforma requer um extension a ser escrito em Java ou um Android Framework SDK.
  • iOS / tvOS: Extensions para esta plataforma estão escritos no Objective C e requerem um arquivo *.mm e um *.h, ou se utilizando qualquer estrutura, um arquivo *.a nativo pré-compilado.
  • PlayStation: Para PlayStation seu extensions deve ser PRX arquivos, com cada alvo exigindo um PRX diferente para estar de acordo com as especificações do alvo (PS3, PS4 ou PS5). Se estiver trabalhando em uma PSVita, o extension deve ser SUPRX.

NOTA Alguns extensions são obrigados a seguir convenções de nomenclatura específicas para GameMaker para identificar corretamente a plataforma e a arquitetura da CPU para a qual foram projetados. Para mais informações, consulte Convenções de nomenclatura.

Como um extension pode ter vários arquivos associados, é possível criar um único extension para múltiplas plataformas, neste caso você deseja especificar todas as plataformas compatíveis no editor principal extension (ou seja: marque a caixa de seleção ao lado delas) e então nas propriedades individuais dos arquivos para cada parte do extension, defina a plataforma para a qual esses arquivos devem ser exportados.

Para mudar a plataforma alvo de um arquivo extension específico ou de um espaço reservado, você deve primeiro abrir as propriedades extension (clique duas vezes em no extension no navegador Asset ), e depois uma vez nas propriedades extension selecione o arquivo de recurso da direita que você deseja editar. Descomprimir na construção

A opção "Uncompress on build" pode ser usada para arquivos compactados que são destinados a uma plataforma alvo específica. Se esta opção for ativada, o conteúdo desse arquivo será extraído para o diretório Included Files do pacote construído, mas somente após ser copiado para a plataforma alvo.

Por exemplo, se você habilitar isto para um arquivo .zip em seu extension e depois iniciar uma construção de macOS a partir de Windows, esse arquivo .zip será copiado como está de Windows para macOS e depois será descomprimido durante a compilação na plataforma alvo (que, neste contexto, é macOS).

Chamando Extension Funções

Uma vez que você tenha configurado seu extension, você pode usar as funções e constantes para ele da mesma forma que você usaria as funções e constantes incorporadas no GameMaker Language. Se seu jogo é multi-plataforma (por exemplo, ele é projetado para rodar em alvos Windows e Android ), então você deve usar as constantes e funções do SO para verificar a plataforma atual antes de chamar extension funções específicas para evitar problemas, por exemplo:

switch(os_type)
{
    case os_android:
        call_android_extension_function(x, y, z);
    break;

    case os_windows:
        call_windows_extension_function(x, y, z);
    break;
}

Observe que para plataformas diferentes, seu extensions pode compartilhar nomes de funções, portanto se você tiver um extension com arquivos distintos para (por exemplo) aparecer uma mensagem em Windows e para Android, ambos os arquivos extension podem compartilhar os mesmos nomes de funções desde que ambos estejam configurados para exportar apenas para suas respectivas plataformas.

Extensões Oficiais de Terceiros

As versões antigas do GameMaker tinham muitas funcionalidades de terceiros, mas isto era ineficiente e difícil de manter, pois se alguma funcionalidade específica fosse alterada por terceiros, então todo o GameMaker IDE e Runtime teriam que ser atualizados. Assim, as versões atuais adotaram o uso de pacotes extensions para adicionar suporte de terceiros a seus jogos. Isto significa que você pode adicionar apenas aquelas funções extras que você precisa e que você pode até adaptá-las e editá-las para atender às suas necessidades específicas.

A maioria destes extensions será baixada e instalada para você quando você selecionar uma opção de jogo específica (como Facebook ou Google Mobile Ads, por exemplo), mas você pode encontrar mais informações sobre eles na seção apropriada para cada plataforma na Base de Conhecimento do GameMaker:

  • Plataformas e configuração doGameMaker

Você pode encontrar o próprio extensions na seguinte página Marketplace:

  • Oficial Marketplace Ativos

Observe que estes extensions vêm com uma demonstração e um arquivo de ajuda (no formato *.pdf ) que explicam como utilizá-los e, como tal, suas funções não são cobertas por este manual.
