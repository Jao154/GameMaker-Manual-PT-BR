Código de Comentários

Ao trabalhar em colaboração com outras pessoas, ao lidar com um grande projeto, ou mesmo apenas para sua própria referência e debugging, deixar comentários em seu código de evento e scripts é muito importante. No GameMaker você tem vários mecanismos para ajudá-lo a deixar notas e comentários sobre seções de código e até mesmo blocos inteiros de código que podem ser vistos na lista de eventos do object que você está editando atualmente. Desta forma, você pode deixar notas para si mesmo e para seus colegas, ou para explicar uma peça de código particularmente complicada, ou até mesmo apenas deixar um lembrete sobre o que algo faz, ou o que você quer fazer no futuro.

Para deixar um simples comentário de uma linha, a primeira coisa que você pode fazer é usar // antes do texto. Por exemplo:

//initialize variables
sr = 10;
hp = 100;

Você também pode deixar comentários de várias linhas, para dar crédito, omitir uma seção completa do código para debugging, ou até mesmo deixar notas sobre como você quer mudar o código no futuro. Para isso, você pode usar /* ... */ assim:

/*
Setup GIF Variables to initial values (these can be edited so you don't need to use the setter functions all the time if you are always using the same values)

IMPORTANT! Please see the individual "set" scripts for more extensive information about each of the different variables.
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

Além dos comentários, você também pode usar etiquetas especiais #region para marcar uma seção de código para o Code Folding, e cada região pode conter um comentário. Como isto funciona é que você adicione uma tag #region - junto com qualquer texto de comentário - ao topo de uma seção de código e depois adicione uma tag #endregion correspondente ao fundo da seção e então você pode "dobrar" o código, ou seja: esconda-o para que você possa cortar o número de linhas de código visíveis no editor a qualquer momento:

O código real se parece algo assim:

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

Finalmente, vale mencionar que ao trabalhar com eventos no Editor de Objetos, você pode usar um comentário especial "description" no início do evento para dizer quais são os conteúdos gerais do evento, e isto será mostrado na janela de eventos do editor Object. Isto deve ir para a primeira linha do editor de código, e é formatado da seguinte forma:

/// @description <This Is An Event Description>
