Arquivos de texto

Arquivos de texto são uma maneira útil de armazenar grandes quantidades de dados externamente de seu jogo. Ao contrário dos arquivos ini, você pode ter até 32 arquivos de texto abertos ao mesmo tempo, e você pode ler e escrever grandes pedaços de dados para eles também. Entretanto, cabe a você decidir como esses dados devem ser estruturados, já que não há uma estrutura definida de "seção" ou "chave" como nos arquivos*.ini .

Observe que para jogos com localizações é muito importante que as letras acentuadas possam ser lidas a partir de arquivos externos. Isto significa que você deve criar primeiro o arquivo txt no formato UTF8 e depois adicioná-lo ao GameMaker como um arquivo incluído para que ele seja exportado durante a execução do jogo e usado no lugar do arquivo txt padrão no formato ANSI que é criado pelas funções do arquivo GameMaker quando nenhum arquivo é fornecido anteriormente. Desta forma, você pode ler e escrever corretamente com todos os acentos e letras não romanas sendo mantidos.

Existem as seguintes funções que tratam de arquivos:

  • file_text_open_read
  • file_text_open_write
  • file_text_open_append
  • file_text_open_from_string
  • file_text_read_real
  • file_text_read_string
  • file_text_readln
  • file_text_write_real
  • file_text_write_string
  • file_text_writeln
  • file_text_eoln
  • file_text_eof
  • file_text_close
