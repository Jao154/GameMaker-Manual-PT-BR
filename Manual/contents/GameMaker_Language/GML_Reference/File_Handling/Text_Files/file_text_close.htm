file_text_close

Uma vez que você tenha terminado de trabalhar com um determinado arquivo (seja lendo ou escrevendo para ele), você deve fechar o arquivo novamente, ou então você corre o risco de perder as informações contidas dentro dele. Isto também evita vazamentos de memória e garante que você nunca ultrapasse o limite do arquivo ao ter mais de 32 arquivos abertos.

Sintaxe:

file_text_close(fileid);

ArgumentoTipo Descrição
fileid The id of the file to close.

Devoluções:

Exemplo:

var file = file_text_open_write(working_directory + "Game_Data.txt");
while (!file_text_eof(file))
{
    file_text_readln(file);
}
file_text_write_string(file, level_data);
file_text_close(file);

O código acima abre um arquivo para escrever então loops através das linhas de texto já escritas no arquivo até que ele chegue ao fim. Neste ponto, ele escreve um string e depois fecha o arquivo novamente.
