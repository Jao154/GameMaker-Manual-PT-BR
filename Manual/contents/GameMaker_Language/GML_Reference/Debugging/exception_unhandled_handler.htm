exception_unhandled_handler

Esta função runtime pode ser usada para "assumir" a mensagem de erro padrão do GameMaker quando ocorre uma exceção runtime. Você fornece um método ou função que é chamado quando a exceção ocorre, e seu primeiro argumento será um struct (conhecido como o Estruturas de Exceção) contendo informações sobre a exceção. Este struct tem o seguinte formato:

Estruturas de Exceção
Variável Tipo Descrição
message Cordão Uma pequena mensagem para a exceção
longMessage Cordão Uma mensagem mais longa para a exceção
script Cordão Um string que descreve de onde veio a exceção
line Real O número de linha de onde veio a exceção
stacktrace Array de Cordãos Um conjunto de strings que é o indicativo para o quadro quando a exceção foi gerada.

Você também pode fornecer undefined como argumento, o que removerá o manipulador de exceções personalizado (se definido) e reverterá para a mensagem de erro padrão do GameMaker.

Esta função retornará o método do manipulador de exceções previamente definido, no entanto, se nunca foi alterado ou se foi reiniciado anteriormente usando undefined, então ele simplesmente retornará undefined.

Passar um método para esta função significa que nenhuma mensagem de erro será mostrada ao usuário, e você mesmo terá permissão para lidar com a exceção. Entretanto, você não pode usar isto para "sair" da exceção para continuar seu jogo, pois o jogo ainda fechará depois que o manipulador da exceção for chamado. Você também não pode chamar funções que mantenham o jogo funcionando por mais tempo do que o quadro final (por exemplo, game_restart()), pois o jogo precisa fechar após a queda.

Observe que o contexto para executar o método não está em um evento, portanto nada pode ser apresentado à tela neste momento. Em geral, a coisa mais segura que poderia ser feita neste momento é salvar um arquivo com as informações do contexto de exceção para que ele possa ser - por exemplo - analisado e processado na próxima execução do jogo e depois tratado.

Observe que se você usar a declaração return no método fornecido, então o valor dado será convertido para um inteiro e usado como o código de saída do corredor.

Sintaxe:

exception_unhandled_handler(user_handler);

Argumento Tipo Descrição
user_handler Método The method (user-defined function) to use as the exception handler, or undefined para redefini-lo para o padrão.

Devoluções:

Method (or undefined)

Exemplo:

exception_unhandled_handler(function(ex)
{
    // Print some messages to the output log
    show_debug_message( "--------------------------------------------------------------");
    show_debug_message( "Unhandled exception " + string(ex) );
    show_debug_message( "--------------------------------------------------------------");

    // Write the exception struct to a file
    if file_exists("crash.txt") file_delete("crash.txt");
    var _f = file_text_open_write("crash.txt");
    file_text_write_string(_f, string(ex));
    file_text_close(_f);

    // Show the error message (for debug purposes only)
    show_message(ex.longMessage);

    return 0;
});

O código acima criará um método de tratamento de exceções e o utilizará para salvar um arquivo com os dados de exceção; ele também mostrará a longa mensagem de erro em uma nova janela pop-up para fins de depuração.
