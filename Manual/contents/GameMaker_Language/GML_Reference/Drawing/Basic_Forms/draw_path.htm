caminho_de_sorteio

Com esta função você pode conseguir que o GameMaker desenhe um path para a tela. O path será desenhado como uma linha simples, e pode ser relativo à instância de chamada ou na posição absoluta em que foi criado no editor path ou através de código. Esta função é extremamente útil quando o debugging dinamiza paths (por exemplo, aqueles criados para instâncias com o mp_grid_path() função).

NOTA: Se você estiver querendo desenhar uma forma usando um shader, você deve estar ciente de que a maioria shaders espera as seguintes entradas: vertex, texture, Cor. Entretanto, ao usar esta função, apenas vertex e dados de cor estão sendo passados, e assim o shader não pode desenhar nada (ou desenhar algo, mas não corretamente). Se você precisar desenhar formas desta forma, então o shader deve ser personalizado com isto em mente.

Sintaxe:

draw_path(path, x, y, absolute);

ArgumentoTipo Descrição
path The path to draw
x The x coordinate of where the path is drawn
y The y coordinate of where the path is drawn
absolute Whether the path is drawn at the absolute position (true) ou a posição relativa (false)

Devoluções:

Exemplo:

if mp_grid_path(grid, path, x, y, obj_Player.x, obj_Player.y, 1)
{
    draw_path(path, x, y, false);
}

O código acima usará a função mp_grid_path para gerar um path e armazená-lo na variável "caminho". Se o path for criado com sucesso, ele será então desenhado na tela em uma posição relativa à instância que está executando o código.
