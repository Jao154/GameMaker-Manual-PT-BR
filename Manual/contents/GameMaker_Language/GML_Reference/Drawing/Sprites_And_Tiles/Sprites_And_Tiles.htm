Sprites e Ladrilhos

Mesmo que você possa criar fundos no editor room e possa atribuir sprites a instâncias, muitas vezes isto não é suficientemente flexível para o jogo que você está fazendo. Por exemplo, se você tiver uma instância que tenha que desenhar um menu, você pode querer exibir vários botões sprites de cada vez, ou se você tiver um jogo com fundos de paralaxe rolantes, você precisará ser capaz de configurá-los através de código em uma instância controladora de algum tipo. Nestas situações, você deve usar as seguintes funções que podem desenhar sprite e fundo assets em qualquer lugar no room com algumas das funções que também permitem manipular a forma, tamanho, cor e alfa do sprite para obter efeitos especiais.

Você pode atribuir um sprite a uma instância e por padrão o GameMaker o desenhará com as propriedades dadas pelas várias variáveis sprite, entretanto, se você acrescentar algo ao evento do sorteio, você está "assumindo o controle" e deve dizer ao GameMaker exatamente o que deve sacar. Isto significa que se você tiver que usar as seguintes funções para definir exatamente o que é a instância a ser sorteada e sobre o que no room é para sorteá-la:

  • draw_self
  • draw_sprite
  • draw_sprite_ext
  • draw_sprite_general
  • draw_sprite_part
  • draw_sprite_part_ext
  • draw_sprite_stretched
  • draw_sprite_stretched_ext
  • draw_sprite_pos
  • draw_sprite_tiled
  • draw_sprite_tiled_ext

Se você estiver usando o formato SWF sprites, então há também opções adicionais para controlar o anti-aliasing usado no desenho (veja Importing Non-Bitmap Sprites para mais informações sobre estes tipos de sprites). Note que estas funções funcionam em nível global e definirão o AA em todos os vetores SWF sprites desenhados após terem sido chamados.

  • draw_enable_swf_aa
  • draw_set_swf_aa_level
  • draw_get_swf_aa_level

Além das funções listadas acima, há também duas funções especiais projetadas para uso com animações esqueléticas (veja Importing Non-Bitmap Sprites para maiores informações sobre estes tipos de sprites, e a seção Skeletal Animations para funções específicas relacionadas a estes tipos de sprites):

  • draw_skeleton
  • draw_skeleton_instance
  • draw_skeleton_collision
  • draw_skeleton_time

Os azulejos são desenhados automaticamente com base nos dados que contêm, no tile set utilizado e nas propriedades do mapa tile. Entretanto, é possível desenhar mapas individuais tiles ou tile usando as seguintes funções:

  • draw_tile
  • draw_tilemap
