orientação_parte_tipo

Esta função é usada para determinar a orientação da partícula sprite quando ela é criada e também pode ser usada para fazer a orientação da partícula aumentar ou diminuir ao longo de sua vida útil. Os valores mínimo e máximo de orientação padrão em 0, mas estes valores podem ser alterados para aleatorizar a orientação seguindo as direções padrão do GameMaker de 0 graus para a direita, 90 graus para cima, 180 graus para a esquerda e 270 graus para baixo. Se você definir o mesmo valor, as partículas serão todas criadas com a mesma orientação.

Você também pode definir um valor de incremento que adicionará (se um número positivo) ou subtrairá (se um número negativo) uma quantidade de graus à orientação ao longo de sua vida útil. Este valor pode ser um mínimo de (+/-) 0,01.

Você pode definir o fator "agitar" também, o mesmo que outras funções das partículas. Este é um valor que será adicionado ou subtraído aleatoriamente à orientação de cada etapa da vida útil das partículas. Obviamente, valores maiores são mais pronunciados do que os menores, e este valor pode até ser negativo, sendo a faixa máxima entre -20 e 20.

Finalmente, você pode optar por ter a orientação relativa ou não, o que significa que enquanto a partícula tiver uma direção (e velocidade), a partícula sprite será orientada em torno desse vetor. Tenha em mente que se você tiver a velocidade da partícula definida para reduzir e ela chegar a 0, a falta de velocidade define a direção para o valor padrão de 0° e assim uma orientação relativa fará com que a partícula sprite "pule" para um ângulo diferente.

Sintaxe:

part_type_orientation( ind, ang_min, ang_max, ang_incr, ang_wiggle, ang_relative );

Argumento Tipo Descrição
ind Identificação do tipo de partícula The index of the particle type to change.
ang_min Real The minimum starting angle of the particles.
ang_max Real The maximum starting angle of the particles.
ang_incr Real The increase (or decrease if set negative) of the angle in degrees per step.
ang_wiggle Real The amount the angle wiggles clockwise/counterclockwise per step.
ang_relative Real Whether to set its angle relative to the direction of the particle's motion (true) or not (false).

Devoluções:

N/A

Exemplo:

part_type_shape(particle2, pt_shape_spark);
part_type_size(particle2, 0.10, 0.50, 0.01, 0);
part_type_scale(particle2, 0.30, 0.30);
part_type_colour1(particle2, 8454143);
part_type_alpha1(particle2, 0.50);
part_type_speed(particle2, 4, 4, -0.07, 1);
part_type_direction(particle2, 0, 359, 0, 20);
part_type_orientation(particle2, 0, 359, 0, 20, 1);
part_type_blend(particle2, 1);
part_type_life(particle2, 1, 5);

O código acima estabelecerá vários valores de partículas, incluindo a orientação que será valor aleatório entre 0º e 359º. Também terá uma quantidade aleatória adicionada de 0 a 20º a cada passo, e a orientação é relativa à direção do movimento.
