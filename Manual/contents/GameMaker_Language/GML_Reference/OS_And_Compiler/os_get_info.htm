os_get_info

Esta função retorna um DS Map com informações detalhadas sobre o sistema operacional em que o jogo está sendo executado. As informações exatas retornadas dependerão do sistema operacional e do dispositivo, portanto é recomendável que você teste esta função em todas as plataformas que você deseja direcionar e emitir os valores retornados usando uma função como show_debug_message(). Note que o mapa DS não é automaticamente apagado da memória e você deve usar o ds_map_destroy() funcionar quando você não precisa mais dele.

Abaixo você pode encontrar alguns detalhes sobre os dados extras que serão disponibilizados para o mapa DS específicos para as seguintes plataformas, e deve ser notado que em todas as plataformas exceto HTML5 o mapa terá a chave "is64bit" que será true ou false dependendo se o corredor está rodando no modo 64-bit ou não.

Windows / UWP

Em Windows Desktop e Windows UWP, você obterá alguns dados extras do DX11 com as seguintes chaves disponíveis no mapa DS (útil para extensions):

  • udid: Um identificador que é exclusivo da máquina
  • video_d3d11_device: ponteiro para o dispositivo DX11
  • video_d3d11_context: ponteiro para o contexto do DX11
  • video_d3d11_spawnchain: ponteiro para a cadeia de troca DX11
  • video_adapter_vendorid: string com a identificação do fornecedor do adaptador
  • video_adapter_deviceid: string com a identificação do dispositivo do adaptador
  • video_adapter_subsysid: string com a identificação do sub sistema do adaptador
  • video_adapter_revision: string com o número de revisão do adaptador
  • video_adapter_description: string com a descrição do adaptador de vídeo
  • video_adapter_dedicatedvideomemory: string com o tamanho da memória de vídeo
  • video_adapter_dedicatedsystemmemory: string com o tamanho da memória do sistema (utilizado pelo adaptador DX11)
  • video_adapter_sharedsystemmemory: string com o tamanho da memória do sistema compartilhado

macOS

A função retorna algumas informações OpenGL limitadas sobre MacOS:

  • udid: Um identificador que é exclusivo da máquina
  • gl_vendor_string GL_VENDOR de OpenGL, exemplo - "Intel Inc."
  • gl_version_string: GL_VERSION de OpenGL, exemplo - "2.1 INTEL-16.2.16"
  • gl_renderer_string: GL_RENDERER de OpenGL, exemplo - "Intel(R) Iris(TM) Graphics 6100"

Ubuntu

Esta contém as mesmas informações que em macOS.

Android

Na plataforma Android você obterá alguns dados extras de várias APIs diferentes com as seguintes chaves disponíveis no mapa DS:

  • android_tv: isto retornará true se o dispositivo for detectado como sendo um dispositivo Android TV, ou false caso contrário
  • GL_VERSIONa versão do OpenGL, conforme relatado pelo motorista
  • GL_VENDORo fornecedor da OpenGL, conforme relatado pelo motorista
  • GL_RENDERERo renderizador da OpenGL, conforme relatado pelo motorista
  • GL_EXTENSIONS extensions de OpenGL que estão disponíveis conforme relatado pelo motorista
  • GL_SHADING_LANGUAGE_VERSIONa versão da GLSL que é suportada por este driver OpenGL
  • GL_MAX_TEXTURE_SIZEo tamanho máximo texture suportado por este motorista OpenGL
  • SDK_INTAndroid.os.Build.VERSION.SDK_INT (ver nota abaixo)
  • RELEASEvalor do android.os.Build.VERSION.RELEASE
  • MODELvalor do andróide.os.Build.MODELO
  • DEVICEvalor do andróide.os.Build.DEVICE
  • MANUFACTURERvalor do android.os.Build.MANUFACTURER
  • CPU_ABIvalor de android.os.Build.CPU_ABI
  • CPU_ABI2valor de android.os.Build.CPU_ABI2
  • BOOTLOADERvalor do android.os.Build.BOOTLOADER
  • BOARDvalor do androide.os.Build.BOARD
  • VERSION: valor da versão de os.version do método Android System.getProperty.
  • REGION: valores de user.region do método Android System.getProperty
  • VERSION_NAMEvalor da versãoNome deste pacote - veja aqui para mais informações.
  • PHYSICAL_KEYBOARD: se pensarmos que um teclado físico está disponível, então o string "TRUE" caso contrário "FALSE"

NOTA: Para informações sobre qualquer variável do android.os.Build, veja a documentação do Desenvolvedor Android.

iOS & tvOS

Aqui estão as chaves devolvidas em iOS e tvOS (para informações detalhadas, consulte a documentação da Apple Developer):

  • name: string como relatado por [[UIDevice currentDevice] name]
  • systemName: string como relatado por [[UIDevice currentDevice] systemName]
  • systemVersion: string como relatado por [[UIDevice currentDevice] systemVersion]
  • model: string como relatado por [[UIDevice currentDevice] model]
  • localisedModel: string como relatado por [[UIDevice currentDevice] localizedModel]
  • uiIdiom: string como relatado por [[UIDevice currentDevice] userInterfaceIdiom]
  • platform: hw.machine string reunido de sysctlbyname
  • hwModel: hw.model string reunido de sysctlbyname
  • freeDiskSpace: O espaço livre em disco
  • totalDiskSpace: O espaço total em disco
  • cpuFrequency: A freqüência da CPU
  • busFrequency: A freqüência do ônibus
  • cpuCount: A contagem da CPU
  • totalMemory: A memória total do dispositivo
  • userMemory: A memória do usuário
  • maxSocketBufferSize: O tamanho máximo da tomada buffer
  • Teclas adicionais contendo informações gráficas OpenGL

Ópera GX

As seguintes chaves extras serão incluídas no DS Map for Opera GX:

  • mobile: Isto será 1.0 se o jogo estiver rodando em um navegador móvel, e 0.0 se não estiver.

PlayStation 4

Consulte a documentação PS4 para obter informações sobre as funções aqui citadas.

A função retorna as seguintes teclas em um console PlayStation 4:

  • display_safe_area_ratio: Um valor de ponto flutuante conforme retornado por sceSystemServiceGetDisplaySafeAreaInfo().
  • is_neo_mode: Isto será 1 se o jogo estiver rodando em um Neo (PS4 Pro) e 0 se estiver rodando em um PS4 normal.
  • enter_button_assign: Um valor inteiro como retornado por sceSystemServiceParamGetInt(SCE_SYSTEM_SERVICE_PARAM_ID_ENTER_BUTTON_ASSIGN, &enterassign); um valor de 0 significa que o botão Círculo é usado como "atribuir" e 1 significa que o botão X é usado em seu lugar.

PlayStation 5

A função retorna as seguintes teclas em um console PlayStation 5:

  • display_safe_area_ratio: O mesmo que o PS4 (ver acima).
  • display_resolutionHD", "4K" ou "desconhecido".
  • display_dynamic_rangeHDR", SDR" ou "desconhecido".
  • display_refresh_rate120Hz", 60Hz" ou "Desconhecido".

Xbox One

As informações retornadas no Xbox One são em sua maioria as mesmas de Windows, entretanto as teclas video_adapter_* e udid são 0 (exceto para video_adapter_description que é um vazio string ""). Ele também não tem a chave video_d3d11_swapchain.

Esta plataforma também retorna as seguintes informações:

  • is_xbox_one_x: Este será 1 se o jogo estiver rodando em um Xbox One X e 0 se estiver rodando em um Xbox One básico.

Série Xbox X/S

As informações retornadas nos consoles da Série Xbox X/S são praticamente as mesmas do Xbox One, porém as teclas video_d3d11_* foram substituídas por video_d3d12_*.

Aqui estão algumas chaves que esta plataforma devolve, além das listadas no Xbox One:

  • video_d3d12_cmdqueue: Aponta para a fila de comando DX12
  • video_d3d12_cmdlist: Aponta para a lista de comandos DX12
  • video_d3d12_currentrt: Aponta para o alvo atual de renderização do DX12

Sintaxe:

os_get_info()

Devoluções:

DS Map ID

Exemplo:

if (os_type == os_android)
{
    var _info = os_get_info();
    
    if (_info[? "android_tv"])
    {
        global.android_tv = true;
    }
}

O código acima verifica se o sistema operacional atual é Android, recebe as informações do sistema operacional e lê o mapa retornado para verificar se o jogo está rodando em um Android TV; nesse caso, ele define a variável personalizada global.android_tv para true.
