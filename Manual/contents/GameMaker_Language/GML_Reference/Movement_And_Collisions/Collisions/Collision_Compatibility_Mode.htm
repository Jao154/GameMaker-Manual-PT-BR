Modo de compatibilidade de colisão

Nas Opções Gerais de Jogo para seu projeto, você pode habilitar o "Modo de Compatibilidade de Colisão" para switch para o sistema de colisão legado. Você só precisará fazer isso se notar um comportamento estranho com colisões em seu(s) projeto(s) existente(s) após atualizar para GameMaker 2022.1, o que mudou a forma como as colisões funcionam.

Mudanças nas colisões

No sistema legado de colisão, todas as coordenadas de caixa de delimitação eram arredondadas para inteiros e eram "exclusivas", ou seja, excluíam as bordas inferior e a mais direita de uma máscara de colisão. Por exemplo, a caixa de delimitação de uma máscara de colisão 16x16 seria gerada de (0, 0) a (15, 15) (relativo). Isto funcionaria bem para instâncias que utilizassem apenas valores inteiros para posicionamento e escalonamento, entretanto, o uso de valores de ponto flutuante causaria um comportamento inesperado e uma leve desconexão entre a renderização e a verificação de colisão, pois GameMaker sempre arredondaria os valores da caixa de delimitação para a realização de verificações de colisão.

No sistema de colisão atual, os valores da caixa de delimitação não são arredondados e são usados como estão, o que significa que as colisões são mais precisas e se comportam como seria de se esperar. As caixas de delimitação também são "inclusivas" agora, ou seja, incluem as bordas mais inferior e mais direita. Por exemplo, a caixa de delimitação para uma máscara de colisão 16x16 é agora gerada de (0,0, 0,0) a (16,0, 16,0) (relativa).

Modo de Compatibilidade

As mudanças de colisão explicadas acima podem causar a quebra de alguns projetos existentes, que você pode corrigir facilmente ao ativar o Modo de Compatibilidade de Colisão para tais projetos de modo que o sistema de colisão legado seja usado; no entanto, recomenda-se testar e atualizar seu código de colisão para trabalhar com o novo sistema, pois ele deve permitir que seu jogo tenha colisões mais precisas e lógicas.

Para projetos que são novos ou não utilizam o código de colisão existente, não há necessidade de habilitar esta opção.
