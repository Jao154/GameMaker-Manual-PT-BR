linha_de_colisão

Collision_line object esta verificação pode ser precisa ou não, mas para que colisões precisas sejam habilitadas, o object ou instância que você está verificando também deve ter colisões precisas habilitadas para seu sprite. Caso contrário, a verificação padrão é baseada em caixas de delimitação. A imagem a seguir ilustra como isto funciona:

Lembre-se, para que colisões precisas sejam consideradas tanto a object sprite como a função de colisão devem ter uma marcação precisa como em. Deve-se observar também que o valor de retorno da função pode ser o id de qualquer uma das instâncias consideradas em colisão ao longo da linha, portanto, se três instâncias se sobrepuserem à linha definida, qualquer um de seus ids pode ser o valor de retorno da função.

Note que ao invés de um índice object você pode fornecer uma instância id para verificar uma instância específica, ou as palavras-chave da instância all, ou other (dependendo do evento e do escopo do código atual).

Sintaxe:

collision_line(x1, y1, x2, y2, obj, prec, notme);

Argumento Tipo Descrição
x1 Real The x coordinate of the start of the line.
y1 Real The y coordinate of the start of the line.
x2 Real The x coordinate of the end of the line.
y2 Real The y coordinate of the end of the line.
obj Objeto Ativo ou ID da instância The object to check for instance collisions.
prec Booleano Whether the check is based on precise collisions (trueque é mais lento) ou sua caixa de delimitação em geral (false, mais rápido).
notme Booleano Whether the calling instance, if relevant, should be excluded (true) ou não (false).

Devoluções:

Instance ID or noone

Exemplo:

with (obj_Enemy)
{
    if collision_line(100, 400, 100, 600, id, false, false) instance_destroy();
}

Este código faz com que todas as instâncias de "obj_Enemy" verifiquem numa linha de 100x, 400y a 100x, 600y para uma colisão com eles mesmos, e se houver uma, então elas são destruídas.
