superfície_de_solução

Com esta função você pode escrever informações de uma superfície para uma determinada buffer. O buffer deve ter sido criado anteriormente e deve ser um buffer de 1 byte alinhado suficientemente grande para armazenar dados para a superfície que você vai escrever (se não tiver certeza, use um buffer de crescimento ou veja o exemplo no final desta página).

As informações para a superfície serão sempre escritas até o início do buffer (independentemente da posição de busca), mas você pode escolher um valor de offset (em bytes) para mover a posição de escrita desde o início do buffer.

Note que a função escreve cada pixel da superfície no buffer usando uma formatação RGBA no alvo Windows, mas em outros alvos pode ser diferente dependendo do sistema operacional ou mesmo do dispositivo.

NOTA: Esta função foi alterada a partir da atualização do GameMaker 2.3.1 em diante. As versões anteriores usavam a constante buffer_surface_copy, que agora está obsoleta.

Sintaxe:

buffer_get_surface(buffer, surface, offset);

Argumento Tipo Descrição
buffer Identificação do tampão The index of the buffer to use.
surface Identificação da superfície The index of the surface to use.
offset Real The data offset value (in bytes).

Devoluções:

N/A

Exemplo:

buffer = buffer_create(surf_width * surf_height * 4, buffer_fixed, 1);
buffer_get_surface(buffer, surface, 0);

Este código criará um código de tamanho fixo buffer que tem a capacidade de armazenar 4 bytes por pixel (para os componentes R, G, B e A) com base no tamanho exato da superfície, e então copiar os dados da superfície para dentro dele.
