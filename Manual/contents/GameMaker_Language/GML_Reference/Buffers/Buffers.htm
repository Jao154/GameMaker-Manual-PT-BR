Buffers

A buffer é uma região da memória física usada para armazenar temporariamente dados enquanto estão sendo movidos de um lugar para outro ou manipulados de alguma forma, por exemplo, você pode usar um buffer para armazenar dados de pacotes que aguardam transmissão através de uma rede ou para armazenar um pacote que tenha sido recebido através de uma rede. O GameMaker tem uma série de funções específicas relacionadas à criação, modificação e carregamento/salvamento buffers, mas antes de usá-las, observe os seguintes pontos:

  • Quando um buffer é criado, ele é automaticamente liberado e preenchido a 0.
  • O argumento do "tamanho" das funções buffer refere-se sempre ao tamanho em bytes do buffer.
  • O argumento "Alinhamento" refere-se a como os dados são armazenados dentro de um buffer. Se seu alinhamento for definido para dizer, 4, e você escrever um único pedaço de dados que seja 1 byte em tamanho, então faça um buffer_tell, você receberá um offset de 1. No entanto, se você escrever outro pedaço de dados, também 1 byte em tamanho, então faça um buffer_tell, você receberá um offset de 5, já que o alinhamento "acolchoou" os dados para aquele tamanho. O alinhamento só afeta onde as coisas são escritas, portanto, se você fizer um buffer_tell depois de escrever algo, ele retornará a posição de escrita atual que imediatamente segue os dados que você escreveu. Observe, entretanto, que se você escrever outro dado, internamente o buffer moverá a posição de escrita para o próximo múltiplo do tamanho do alinhamento antes de escrever o dado.
  • Quando se trata de "offset", este é o valor em bytes para compensar os dados por dentro do dado buffer. Portanto, se seu buffer estiver 2 bytes alinhados e você quiser pular os 4 primeiros lugares alinhados, você teria um offset de 2 * 4 bytes.
  • Algumas das funções buffer também criam um novo buffer (como buffer_load, por exemplo). Lembre-se de que estas buffers também precisarão ser removidas da memória quando não estiverem em uso, usando a função buffer_delete.
  • A memória utilizada para criar buffers é a memória do sistema, portanto, mesmo quando o jogo não tem foco, qualquer dado armazenado em um buffer deve ser seguro, entretanto, se o aplicativo for fechado ou reiniciado, ele será perdido.

  • Usando amortecedores

IMPORTANTE! Quando você cria um buffer, o valor do índice usado para identificá-lo é um valor inteiro começando em 0. Estes índices são reutilizados por GameMaker, portanto um valor de índice destruído buffer pode ser usado por um índice recém-criado posteriormente, e recomendamos que você defina qualquer variável que contenha um índice tampão para -1 após ter destruído o buffer.
