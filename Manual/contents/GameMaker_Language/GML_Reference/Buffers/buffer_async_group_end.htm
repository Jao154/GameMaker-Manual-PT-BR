buffer_async_group_end

Esta função termina a definição de um grupo assimétrico buffer. Você deve ter chamado previamente a função buffer_async_group_begin() para iniciar o grupo, depois chamar a função buffer_save_async() para cada arquivo que você deseja salvar (ou buffer_load_async() para carregar buffers). Finalmente, você chama esta função, que iniciará a gravação dos arquivos. A função retornará um valor de identificação único para a gravação, que poderá então ser usado no evento Assíncrono de Gravação/Carga para analisar os resultados do async_load Mapa DS.

Sintaxe:

buffer_async_group_end();

Devoluções:

Async Request ID

Exemplo ampliado:

A função buffer_async_group_end() pode ser chamada a partir de qualquer evento, e como é assíncrona, a ligação de retorno pode ser quase instantânea ou pode levar vários segundos. Chamar a função é simples e pareceria algo parecido com isto:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
save_id = buffer_async_group_end();

O código acima inicia um grupo buffer e depois o define para salvar 4 arquivos de forma assíncrona. A definição do grupo é então terminada (momento em que a gravação começará), armazenando o ID da chamada de função na variável "save_id". Quando a gravação estiver completa, o evento assíncrono Salvar/Carregar será acionado e você poderá analisar o mapa async_load para a identificação correta da função, desta forma:

if ds_map_find_value(async_load, "id") == saveid
{
    if ds_map_find_value(async_load, "status") == false
    {
        show_debug_message("Save failed!");
    }
}

O código acima verificará primeiro a identificação do mapa DS que foi criado, depois verificará o status da chamada de retorno, postando uma mensagem de depuração se tiver havido algum problema.
