buffer_load

Esta função é utilizada para carregar um buffer que foi salvo anteriormente usando o buffer_save() bem como quaisquer Arquivos incluídos ou arquivos carregados externamente. Ele retornará um novo ID do buffer que é criado pela função como um "grow" buffer (veja aqui) com um alinhamento de bytes de 1. Este ID deve ser armazenado em uma variável e usado em todas as chamadas de funções posteriores a este buffer. Se a carga falhar por qualquer motivo, a função retornará -1 e uma mensagem será exibida na janela de saída do compilador dizendo que a carga falhou.

NOTA É importante que você remova da memória qualquer recurso criado dinamicamente como este quando não precisar mais deles para evitar vazamentos de memória, por isso, quando você terminar com o buffer que você criou, deverá liberá-lo novamente usando buffer_delete().

Notas específicas da plataforma

  • Em HTML5, qualquer buffers carregado do armazenamento local será decodificado usando a base64 (já que eles são salvos como base64 strings), entretanto, usando esta função em Arquivos Incluídos ou quaisquer arquivos externos não usarão a decodificação da base64, já que não se espera que eles estejam em um formato base64.
  • Em HTML5, todos os Arquivos Incluídos são armazenados como texto codificado UTF-8, o que pode resultar em um aumento no tamanho do arquivo carregado buffer quando comparado ao arquivo original, já que a versão codificada UTF-8 do arquivo pode ter um tamanho diferente do seu original (devido à mudança no formato).

Sintaxe:

buffer_load(filename);

ArgumentoTipo Descrição
filename The name of the file to load from.

Devoluções:

Buffer ID

Exemplo:

player_buffer = buffer_load("Player_Save.sav");

O código acima carregará em memória um buffer previamente gravado, criando um novo buffer. O índice deste novo buffer é armazenado na variável "player_buffer".
