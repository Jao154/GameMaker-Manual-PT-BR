Preferências de Rollback

Talvez você queira permitir que cada jogador selecione um personagem, ou personalize suas propriedades. Tais propriedades podem ser configuradas como "preferências do jogador".

Como usar as preferências?

Primeiro, habilite as preferências chamando rollback_use_player_prefs() / Use Prefs. de Jogadores. Com esta função você pode opcionalmente definir as preferências padrão para todos os jogadores.

Uma vez que você chama essa função, a "partida automática" é desativada, o que significa que o jogo não começará mesmo quando todos os jogadores tiverem aderido, e você terá que iniciá-lo manualmente com rollback_start_game() / Comece o jogo.

Antes do jogo começar, você pode definir as preferências de um jogador, chamando rollback_set_player_prefs() / Definir Prefs de Jogador no cliente daquele jogador. Estes podem ser lidos posteriormente através de rollback_get_player_prefs() / Obter Prefs de jogador, ou a variável player_prefs em uma instância de jogador.

Evento de Rollback

Quando um jogador muda suas preferências, todos no jogo recebem um rollback_player_prefs Rollback Event. O jogador deve estar completamente conectado para que o evento seja enviado.

Se as preferências foram definidas antes da conexão, então ao completar a conexão, as últimas preferências definidas antes da conexão são enviadas a todos os jogadores.

Condição de início

Se você ativou as preferências, e não há preferências padrão definidas, então o jogo não começará a menos que as preferências para todos os jogadores tenham sido definidas.

Neste caso, você pode chamar rollback_start_game() repetidamente em um alarme para tentar iniciar o jogo, ou chamá-lo sempre que o evento rollback_player_prefs Rollback Event for recebido.

Que tipos posso usar?

Uma preferência pode ser de qualquer tipo, por exemplo, um número, string, array, etc., mas é recomendado o uso de uma estrutura, que é a maneira mais fácil de armazenar uma coleção de variáveis.
