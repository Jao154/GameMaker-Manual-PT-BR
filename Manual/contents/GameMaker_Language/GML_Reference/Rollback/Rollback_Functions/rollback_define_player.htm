rollback_define_player

Esta função define o object que deve ser usado para jogadores em multiplayer. Isto é opcional, e quando usado, o sistema criará automaticamente instâncias para jogadores que se conectam, e destruirá instâncias para jogadores que se desconectam.

Por padrão, isto cria instâncias de jogadores na camada "Instâncias", que devem existir em seu jogo room. Se você deseja usar outra camada, passe seu nome como segundo argumento.

Para informações sobre como criar um jogo com ou sem esta função, consulte Definindo um objeto do jogador. Se usado, esta função deve ser executada antes da função de juntar/criar.

Cada instância criada com esta função recebe as variáveis da instância listadas nesta página.

Sintaxe:

rollback_define_player(object, [layer_name]);

Argumento Tipo Descrição
object Objeto Ativo The object that will be used for creating player instances.
layer_name Cordão OPTIONAL O nome da camada onde as instâncias do jogador serão criadas. "Instâncias" por padrão.

Devoluções:

N/A

Exemplo:

rollback_define_player(obj_player);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_start_game(2, false);
}

O código acima define um jogador object, e depois tenta participar de um jogo. Se não foi juntado, ele começa um novo jogo, com um máximo de 2 jogadores e Sync Test desativado.
