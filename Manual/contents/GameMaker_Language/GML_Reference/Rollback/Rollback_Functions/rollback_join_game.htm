rollback_join_game

Esta função tenta se juntar a um jogo. Você deve estar em uma URL que contenha o ID do virtual room para participar.

Se um room foi unido, ele retorna verdadeiro, e se nenhum jogo foi unido, ele retorna falso.

Quando um novo jogador entra em um jogo no qual você já está presente, o evento Rollback Event é acionado. Consulte Eventos de Rollback para mais informações.

O argumento dry_run é opcional, e quando definido para true, permite que você execute a função sem realmente entrar em um jogo. Isto é usado para verificar se um jogo pode ser juntado sem realmente se juntar a ele.

Sintaxe:

rollback_join_game([dry_run]);

Argumento Tipo Descrição
dry_run Booleano Disabled by default (false). Quando ativada(verdadeira), a função não entra no jogo, mas simplesmente retorna verdadeira se for possível entrar no jogo, ou falsa de outra forma.

Desta forma, você pode usar esta função para verificar apenas se o jogador foi convidado.

Devoluções:

Boolean

Exemplo:

rollback_define_player(obj_player);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_create_game(2, false);
}

O código acima define um jogador object, e depois tenta participar de um jogo. Se não foi unido, ele cria um novo jogo, com um máximo de 2 jogadores e o Sync Test desativado.

Para um exemplo ampliado, veja Criar um Jogo Multiplayer.
