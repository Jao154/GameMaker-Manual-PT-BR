rollback_leave_game

Esta função deixa a atual sessão multiplayer, permitindo que o jogador entre ou crie um novo jogo novamente.

Quando o anfitrião de um room sai, a propriedade desse room é transferida para outro jogador, para que o jogo possa continuar. O jogo só termina quando todos os jogadores tiverem partido.

Evitando erros

Depois que o jogador deixa uma sessão multiplayer, GameMaker continua a executar todo o código do jogo, porém retorna ao estado de "pré-arranque" onde não é possível criar ou modificar instâncias gerenciadas.

Isto significa que você precisa destruir manualmente as instâncias de seu jogador ao sair de um jogo, e pausar qualquer outra lógica de jogo usando o rollback_game_running em uma condição, para evitar erros.

Veja: Estado antes do início do Rollback

Evento desencadeado

Quando um jogador deixa um jogo no qual você está presente, o evento Rollback Event é acionado. Consulte Eventos de Rollback para mais informações.

Sintaxe:

rollback_leave_game();

Devoluções:

N/A

Exemplo:

if (keyboard_check_pressed(vk_escape))
{
    rollback_leave_game();
}

O código acima faz o jogador abandonar o jogo quando a tecla Escape é pressionada.
