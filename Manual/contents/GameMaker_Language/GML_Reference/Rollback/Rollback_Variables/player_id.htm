player_id

Quando você usa rollback_define_player() para definir um jogador object, o sistema cria automaticamente uma instância do mesmo object para cada jogador. Cada instância desse object recebe a variável player_id, que armazena a identificação desse jogador.

Esta identificação começa em 0, que inicialmente é o próprio anfitrião. Quaisquer jogadores a partir de 1 são "pares" que se juntaram ao jogo.

Se você não usar rollback_define_player(), então esta variável não será criada para nenhuma objects. Nesse caso, as instâncias do jogador e seus IDs devem ser gerenciados manualmente, como explicado em Definindo um objeto do jogador.

Sintaxe:

player_id;

Devoluções:

Real

Exemplo:

// Create event
if (player_id == 0)
{
    x = 300;
}
else if (player_id == 1)
{
    x = 500;
}

O código acima muda a posição X da instância do jogador com base em sua identificação.

Você também pode usar este ID como um índice de array, por exemplo, para armazenar pontuações para todos os jogadores:

// Step event
var _coin = instance_place(x, y, obj_coin);
if (_coin != noone)
{
    obj_game.scores[player_id]++;
    instance_destroy(_coin);
}

// Draw event
var _my_score = obj_game.scores[player_id];
draw_text(x, y, _my_score);

O código Step event acima aumenta a pontuação do jogador em um array no obj_game quando ele colide com uma moeda. Essa instância de moeda é então destruída.

O código de evento do sorteio acima recupera a pontuação do jogador a partir da mesma matriz, e saca seu valor.
