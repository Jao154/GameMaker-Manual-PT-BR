xboxlive_pad_para_utilizador

Os usuários do Xbox podem ter vários blocos atribuídos a sua ID de usuário, com cada um deles preenchendo um "slot". Estes slots são numerados a partir de 0, e podem conter qualquer índice de pad, portanto, o slot 0 poderia ser atribuído ao pad 3 e o slot 1 poderia ser atribuído ao pad 2. Esta função pode ser usada para obter o pad que é atribuído a qualquer slot, e é particularmente útil caso um usuário deixe um gamepad sem entrada por um certo período de tempo, pois ele mesmo switch desligará, assim você pode verificar qualquer ID de usuário usando esta função para ter certeza de que eles têm um gamepad ativo, e se eles não reagirem de acordo (por exemplo) pausando seu jogo. Se o pad estiver ativo, o seu valor de índice será retornado, mas se não estiver, a função retornará -1.

Observe que se uma almofada em qualquer slot ficar inativa, os slots posteriores "deslizarão" para a posição vaga, portanto, se você tiver três almofadas ativas atribuídas aos slots 0, 1 e 2, e a almofada no slot 0 estiver inativa, os slots 1 e para agora se tornarem slots 0 e 1, portanto, o slot 0 deverá sempre conter uma almofada ativa a menos que todas as almofadas estejam desligadas (nesse caso, ela retornará -1).

Sintaxe:

xboxlive_pad_for_user(user_id, slot);

Argumento Tipo Descrição
user_id ID do usuário do Xbox The User ID (a pointer) to check
slot Real The index (an integer) of the gamepad slot to check

Devoluções:

Xbox Pad ID

Exemplo:

if (xboxlive_pad_for_user(user_id, 0) == -1)
{
    for(var i = 0; i < gamepad_get_device_count(); i++;)
    {
        var u_id = xboxlive_user_for_pad(i);
        if (u_id == -1)
        {
            async_ID = xboxlive_pad_for_user(i, u_id);
        }
    }
}

O código acima verifica o slot 0 para o ponteiro de identificação do usuário dado para ver se algum campo de jogo foi atribuído, e se não foi, então um é atribuído.
