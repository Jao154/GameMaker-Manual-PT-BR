physics_world_update_speed_speed

O GameMaker atualiza as coisas com base na velocidade room, de modo que, se você definir para, digamos, 30, o GameMaker executará 30 passos no decorrer de um segundo. Entretanto, para as funções físicas que podem não ser suficientes e você pode querer que as coisas sejam atualizadas a uma velocidade um pouco mais rápida para aumentar a estabilidade ou precisão. Para isso, usamos a função physics_world_update_speed() que define a velocidade de atualização para o sistema de física independentemente da velocidade room. Isto significa que você poderia ter uma velocidade room de 30, mas ajustar a física para 60, efetivamente dobrando a velocidade na qual o sistema de física se atualiza e realiza seus cálculos em comparação com a velocidade na qual a etapa é atualizada.

NOTA: atualmente não é possível definir isto para menos de room velocidade, mas atualizações futuras podem mudar isto.

Sintaxe:

physics_world_update_speed(speed)

Argumento Tipo Descrição
speed Real The number of times per second that the physics system updates

Devoluções:

N/A

Exemplo:

physics_world_update_speed(room_speed * 2);

O código acima define a velocidade de atualização do sistema físico para ser o dobro da velocidade do room.
