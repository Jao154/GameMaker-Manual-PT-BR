Economia de nuvens

Ao fazer jogos, muitas vezes é necessário armazenar informações sobre o estado do jogo em um arquivo de algum tipo, mas armazenar essas informações no dispositivo nem sempre é a melhor opção, pois, se o jogador excluiu o jogo e depois reinstalou, essas informações podem ser perdidas. Para evitar isto, você pode usar vários serviços diferentes de nuvem, que oferecem armazenamento de dados através da Internet para recuperação e modificação a qualquer momento. Você deve observar que esta função é limitada a um único "blob" de dados por jogo, portanto, toda vez que você enviar um novo salvamento para o serviço de nuvem, seja ele um string ou um arquivo, ele irá sobrescrever quaisquer dados previamente armazenados.

NOTA Atualmente apenas o alvo Android usa estas funções, e você precisa marcar a caixa de seleção Ativar o Google Cloud Saving na seção Social da Android Opções de Jogo. Isto o levará a baixar o Google Services extension, que é necessário para salvar a nuvem em Android para funcionar. Para mais informações sobre o Cloud Saving, consulte o seguinte artigo no helpdesk: Economia de nuvens Android

O GameMaker suporta isto com algumas funções simples que funcionam em conjunto com o evento Asynchronous Cloud Event. Estas funções são explicadas nas páginas seguintes:

  • cloud_synchronise
  • cloud_string_save
  • cloud_file_save
