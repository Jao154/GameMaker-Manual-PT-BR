push_get_first_local_notification

Esta função irá preencher um mapa DS pré-fabricado com uma série de pares chave/valor para a primeira notificação local push no queue a ser mostrado. Ela retorna -1 se não houver notificações em fila de espera, ou um valor real representando de outra forma a identificação para a notificação. Esta identificação pode então ser usada para cancelar a notificação usando a função push_cancel_local_notification().

NOTA: Esta função é limitada aos módulos alvo iOS e Android.

O mapa DS conterá os seguintes pares chave/valor:

  • "título" - O título da notificação (só estará disponível na plataforma Android, pois iOS não armazena as informações do título)
  • "mensagem" - O texto do corpo da mensagem
  • "data" - A seqüência de pacotes de dados

Sintaxe:

push_get_first_local_notification(map);

ArgumentoTipo Descrição
map The DS Map ID para usar.

Devoluções:

Push Notification ID or -1

Exemplo :

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
{
    var data = ds_map_find_value(map, "data");
    if data == "Daily_Reward"
    {
        push_cancel_local_notification(ntf);
    }
    ntf = push_get_next_local_notification(map);
}
ds_map_destroy(map);

O código acima cria um DS Map e depois solicita dados para a primeira notificação disponível no queue. Se houver uma notificação, o mapa é preenchido e o código então verifica a chave "data" para um string específico. Se o string for encontrado, a notificação é cancelada e passa a verificar a próxima notificação até que tudo no queue tenha sido verificado.
