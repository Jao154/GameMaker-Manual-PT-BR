Funções assíncronas

No contexto do GameMaker, uma função assíncrona é aquela que envia para uma web server, ou pede a entrada do usuário, ou mesmo transmite dados, enquanto deixa o GameMaker continuar a funcionar sem bloquear o jogo sendo executado. Quando a informação é recebida , é disparado um Evento Async especial que, por sua vez, permite que qualquer instância com um Evento Async atribuído a ele execute outras funções (embora este não tenha que ser o caso). Portanto, basicamente, é uma forma de comunicação com algum recurso externo (server, usuário ou dispositivo) sem que o jogo ou projeto pare enquanto se espera por uma resposta.

Por que isso é importante? Bem, já que permite que o GameMaker continue funcionando enquanto envia ou recebe dados, e isto significa que você pode fazer muitas coisas sem bloquear o jogo loop, como por exemplo:

  • transmita dados do dispositivo para seu jogo sem que o jogador espere que as coisas sejam carregadas
  • ter eventos de chamada de retorno para fazer as coisas somente quando as informações corretas forem recebidas
  • comunicar e trocar dados com um servidor web

Isto faz com que tudo funcione de uma maneira muito mais fluida e discreta. Deve-se notar que os Eventos Async serão acionados para todas as instâncias que os tiverem, de modo que você pode usar uma função assimétrica em uma instância, e ter o Evento Assíncrono que trata da resposta em outra, ou mesmo várias outras instâncias. Você pode ir aqui para mais informações sobre como funciona o evento assíncrono - Eventos assíncronos.
