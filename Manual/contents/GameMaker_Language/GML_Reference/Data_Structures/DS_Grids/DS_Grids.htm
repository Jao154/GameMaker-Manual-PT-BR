Grades DS

Uma grade DS é basicamente um tipo de matriz com 2 dimensões. Você define a grade definindo sua largura e altura inteiras, que então define o número total de "células" dentro da grade (a largura e a altura multiplicadas) e estas células são então usadas para manter diferentes valores de dados (números reais ou strings).

Este tipo de estrutura permite definir e recuperar o valor das células da grade dando o índice x e y da mesma (que começa com 0 para ambos os eixos x e y e vai até a largura - 1 e a altura -1). Mas você também pode definir o valor em regiões específicas da grade, adicionar valores individuais e recuperar a soma, máximo, mínimo e valor médio sobre uma determinada região, tornando esta estrutura de dados a mais versátil de todas aquelas disponíveis para você.

Ao acessar estruturas de dados da grade DS, você deve sempre tentar usar valores inteiros para a posição da célula, e todos os índices não inteiros serão nivelados pelo GameMaker se você não o fizer. Se não for isso que você precisa, então você mesmo terá que fazer o arredondamento antes de passar o índice que você deseja verificar.

NOTA Como acontece com todos os recursos dinâmicos, as estruturas de dados ocupam a memória e assim devem ser sempre destruídas quando não são mais necessárias para evitar vazamentos de memória, o que irá retardar e, eventualmente, quebrar seu jogo.

NOTA se você precisar verificar se existe uma estrutura de dados, você pode usar o ds_exists() função.

Além das funções específicas listadas abaixo, todos vocês podem usar uma expressão (chamada acessor) para adicionar ou modificar o conteúdo de sua grade DS. Este acessor se parece com um array 2D com a seguinte sintaxe:

grid_index[# xpos, ypos]

Você pode encontrar mais informações, além de exemplos, na página GML Visão geral sobre Accessors.
