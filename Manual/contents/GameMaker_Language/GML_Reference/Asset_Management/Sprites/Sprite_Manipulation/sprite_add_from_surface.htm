sprite_add_from_surface

Esta função funciona exatamente da mesma forma que sprite_create_from_surface() apenas em vez de criar um novo sprite a partir da área da superfície indexada que você selecionar, ele adiciona a área definida da superfície como uma nova subimagem a um sprite previamente criado (isso significa que você não pode adicioná-lo diretamente a um sprite a partir da árvore de recursos, mas apenas a um criado anteriormente a partir de uma superfície, ou um que tenha sido duplicado a partir de um recurso sprite usando sprite_duplicate()).

Deve-se notar que a memória utilizada ao adicionar um sprite desta forma será maior do que você pode esperar. Isto porque o GameMaker irá armazenar o sprite como uma página texture e ele também será armazenado na memória GPU, portanto a memória total será maior do que seria esperado para um arquivo de imagem do mesmo sprite.

Sintaxe:

sprite_add_from_surface(index, surface, x, y, w, h, removeback, smooth);

Argumento Tipo Descrição
index Ativo Sprite The index of the sprite to add the new image to.
surface Identificação da superfície The index of the surface from which we get the image.
x Real The x position to copy from.
y Real The y position to copy from.
w Real The width of the area to be copied (from the x position).
h Real The height of the area to be copied (from the y position).
removeback Booleano Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth Booleano Indicates whether to smooth the edges.

Devoluções:

N/A

Exemplo:

spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
var i;
for (i = 1; i < 8; i += 1)
{
    sprite_add_from_surface(spr_custom, surf, i, 0, 32, 32, false, false);
}

O código acima cria um sprite a partir da superfície indexada na variável "surf", atribuindo seu índice à variável "spr_Custom", e então usa um for loop para se mover através da superfície e capturar várias seções que são adicionadas no sprite como sub-imagens.
