sprite_add

Com esta função você pode adicionar uma imagem como um sprite, carregando-a de uma fonte externa onde o arquivo de imagem a ser carregado deve estar sempre no formato *.png, *.gif, *.jpg/jpeg ou *.json (*json arquivos são usados para carregar sprites de animação de esqueleto feitos com Spine). A função retorna o novo índice sprite que deve então ser usado em todos os outros códigos relacionados ao sprite. Se você usar esta função com HTML5 ou estiver obtendo uma imagem de uma URL, esta função também gerará um evento assíncrono de imagem carregada. Consulte esta página para mais informações.

Todas as imagens a serem transformadas em animadas sprites - exceto *.gif (veja abaixo) - devem ter o formato "strip" (veja a imagem abaixo) e serão divididas no número de subimagens especificadas seguindo a regra sprite width = largura de strip / sub images.

Como você pode ver na imagem acima, o sprite foi colocado sobre um fundo roxo escuro, e isto pode ser removido colocando o argumento "removeback" em true. Isto funciona verificando o pixel inferior esquerdo do sprite para a cor lá e depois usa isso como a cor a ser removida. Por exemplo, na imagem acima, se tivéssemos a cor do píxel inferior esquerdo como verde, todas as partes verdes do sprite teriam sido removidas e o resto do fundo roxo teria sido ignorado.

Se você escolher a opção "remover", você também pode querer que o GameMaker suavize as bordas do sprite, definindo o argumento "suavizar" para true. Tudo o que isso faz é criar uma borda semi-transparente ao redor das bordas do sprite após ter tido seu fundo removido.

Finalmente, você também pode especificar a origem x e y para o sprite. Este é o ponto onde o sprite é "fixo" na instância que o utiliza, e é sempre calculado como relativo ao canto superior esquerdo 0,0 de uma subimagem sprite. Assim, por exemplo, um sprite que é 32 x 32 pixels com estes valores definidos para (16,16) terá sua origem no centro.

Se você estiver adicionando uma imagem no formato *.gif, então você só precisa especificar o nome do arquivo e o número da imagem (que deve ser 1 para o primeiro quadro) e pode deixar todos os outros argumentos como 0, observando que somente a primeira imagem do GIF será carregada. Para *.json sprites, o número da imagem deve ser 1, e as configurações "removeback" e "smooth" serão ignoradas.

Por padrão, todos os novos sprites têm suas caixas delimitadoras calculadas automaticamente (a caixa delimitadora exata dependerá do tamanho e da transparência do sprite), entretanto você pode querer personalizar isso, caso em que você também deve usar a função sprite_collision_mask().

Deve-se notar que a memória utilizada ao adicionar um sprite desta forma será maior do que você pode esperar. Isto porque o GameMaker irá armazenar o sprite como uma página texture e ele também será armazenado na memória GPU, portanto a memória total será maior do que seria esperado para um arquivo de imagem do mesmo sprite.

NOTA: Quando você carrega um sprite no GameMaker, você deve se lembrar de removê-lo novamente (com sprite_delete()) quando não for mais necessário, caso contrário há o risco de um vazamento de memória que irá retardar e, eventualmente, quebrar seu jogo.

NOTA: No iOS, se você estiver carregando um arquivo incluído de uma pasta, ou seja: com um path algo como "Backgrounds/background1.png", você NÃO precisa incluir a pasta como parte do path e simplesmente fornecer o nome do arquivo é suficiente. Em todas as outras plataformas, você deve fornecer a(s) pasta(s) como parte do path para o arquivo.

NOTA: Em HTML5, se você estiver carregando um sprite a partir de um server seguro, você pode precisar definir o tipo de origem cruzada e usar um path relativo em vez de um path absoluto. Veja . http_set_request_crossorigin() para mais detalhes.

Sintaxe:

sprite_add(fname, imgnumb, removeback, smooth, xorig, yorig);

Argumento Tipo Descrição
fname Cordão The name (a string file path) of the file to add.
imgnum Real Use to indicate the number of sub-images (1 for a single image or for a *.gif).
removeback Booleano Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth Booleano Indicates whether to smooth the edges if transparent.
xorig Real Indicate the x position of the origin in the sprite.
yorig Real Indicate the y position of the origin in the sprite.

Devoluções:

Sprite Asset

Exemplo:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);

O código acima carrega um sprite no jogo e armazena seu índice na variável "spr".
