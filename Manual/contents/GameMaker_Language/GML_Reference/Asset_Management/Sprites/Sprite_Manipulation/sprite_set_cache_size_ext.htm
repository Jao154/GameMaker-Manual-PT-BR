sprite_set_cache_size_ext

Ao realizar a mistura de imagens, o HTML5 não pode fazê-lo dinamicamente da mesma forma que um executável poderia ser executado. Portanto, o GameMaker tem que salvar temporariamente uma cópia misturada da imagem e carregá-la. Esta função define quantas cópias mescladas do dado sprite podem ser colocadas em cache antes que as antigas sejam sobrescritas. O padrão é 4. Isto é aplicado a uma única subimagem dada do sprite.

Sintaxe:

sprite_set_cache_size_ext(ind, index, max);

Argumento Tipo Descrição
ind Ativo Sprite The index of the sprite to change the cache size of.
index Real The sub-image of ind to change the cache size of.
max Real The maximum number of cached copies of the sprite that can be stored.

Devoluções

N/A

Exemplo:

sprite_set_cache_size_ext(sprite0, 0, 2);

Isto colocará o sprite cache da primeira subimagem do sprite0 em 2 cópias.
