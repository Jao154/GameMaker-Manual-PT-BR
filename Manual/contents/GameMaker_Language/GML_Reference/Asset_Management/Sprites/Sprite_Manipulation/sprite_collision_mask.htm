sprite_collision_mask

Com esta função, você pode definir as propriedades da máscara de colisão que um sprite deve ter. Se você selecionar automático (0) ou imagem completa (1) como o modo caixa de delimitação, então os valores individuais da caixa de delimitação podem ser definidos como 0. Entretanto, para uma máscara definida pelo usuário (2), você terá que definir estes valores. Os diferentes valores da caixa de delimitação são sempre relativos ao canto superior esquerdo do sprite (independentemente da origem x e y) que é considerada a posição (0, 0).

A definição do tipo de máscara define a forma geral para a própria máscara, mas note que qualquer coisa que não seja uma máscara retangular exigirá mais poder de processamento ao resolver colisões, com uma queda subseqüente no desempenho. Em geral, você só deve usar tipos de máscara que não sejam retangulares quando absolutamente necessário.

NOTA: Esta função não permite o tipo de máscara de colisão retangular girada.

O tipo de máscara que pode ser ajustada será uma das constantes a seguir:

Tipo de caixa de delimitação (Forma) Constante
Constante Descrição
bboxkind_rectangular Uma forma retangular (não rotativa) de máscara de colisão retangular
bboxkind_ellipse Uma forma de máscara de colisão elíptica
bboxkind_diamond Uma forma de máscara de colisão com diamante
bboxkind_precise Uma máscara de colisão precisa, onde a máscara estará de acordo com os pixels não transparentes do sprite, com base no valor de tolerância dado (ver abaixo))


Finalmente, a tolerância é usada para definir a precisão da máscara (quando usada com uma máscara de "imagem completa", isto não terá efeito), com uma tolerância de 0, o que significa que a máscara seguirá cada pixel que tiver uma transparência acima de 0, enquanto outros valores deslocarão o perímetro da máscara de colisão dependendo da transparência dos pixels.

NOTA: Esta função só é útil para o bitmap sprites e não funcionará com SWF ou JSON (Spine) sprites.

NOTA: Esta função só funcionará em sprites adicionado ou duplicado sprites e não diretamente em recursos pré-fabricados. Você pode duplicar sprites usando a função sprite_duplicate().

Sintaxe:

sprite_collision_mask(ind, sepmasks, bboxmode, bbleft, bbtop, bbright, bbbottom, kind, tolerance);

Argumento Tipo Descrição
ind Ativo Sprite The index of the sprite to set the bounding box of.
sepmasks Booleano Whether to create collision masks for each sub-image of the sprite (true), ou uma máscara para todos (false).
bboxmode Real What kind of bounding box to use. 0 = automatic, 1 = full image, 2 = user defined.
bbleft Real The maximum left position of the bounding box.
bbtop Real The maximum top position of the bounding box.
bbright Real The maximum right position of the bounding box.
bbbottom Real The maximum bottom position of the bounding box.
kind Tipo de caixa de delimitação (Forma) Constante The kind of mask, a constant (see the table in the description).
tolerance Real Indicates the tolerance in the transparency value (0=no tolerance, 255=full tolerance).

Devoluções

N/A

Exemplo:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);
sprite_collision_mask(spr, true, 1, 0, 0, 0, 0, 0, 0);

O código acima carrega um sprite de uma fonte externa e armazena o novo índice na variável "spr". O código então define o novo sprite para ter uma máscara de colisão precisa para cada uma de suas subimagens.
