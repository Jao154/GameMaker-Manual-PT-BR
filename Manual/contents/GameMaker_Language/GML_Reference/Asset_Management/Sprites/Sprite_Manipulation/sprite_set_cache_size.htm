sprite_set_cache_size_s

Ao realizar a mistura de imagens, o HTML5 não pode fazê-lo dinamicamente da mesma forma que um executável poderia ser executado. Portanto, o GameMaker tem que salvar temporariamente uma cópia misturada da imagem e carregá-la. Esta função define quantas cópias mescladas do dado sprite pode ser armazenado em cache antes que os antigos sejam sobregravados. O padrão é 4. Isto é aplicado a todas as sub-imagens do sprite.

Sintaxe:

sprite_set_cache_size(ind, max);

ArgumentoTipo Descrição
ind The index of the sprite to change the cache size of.
max The maximum number of cached copies of the sprite that can be stored.

Devoluções

Exemplo:

sprite_set_cache_size(sprite0, 2);

Isto colocará o sprite cache de todas as sub-imagens de sprite0 em 2 cópias.
