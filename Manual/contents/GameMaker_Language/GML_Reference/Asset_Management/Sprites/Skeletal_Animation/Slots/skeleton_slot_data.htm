dados_esqueleto_dados_de_squadra

Com esta função você pode preencher uma lista DS (pré-criada) com dados para cada um dos slots de anexo disponíveis que uma animação contém. Estes dados são retornados no fórum de um mapa DS que contém os seguintes pares chave/valor:

  • "nome": O nome do espaço.
  • "osso": O nome do "osso".
  • "anexo": O nome do anexo se for usado ou "(nenhum)" se não houver um atribuído.

Os valores para cada chave serão strings e podem ser usados nas outras funções de anexação de esqueleto para estes tipos de sprite. Note que os mapas DS criados não são destruídos, então você precisará loop através da lista DS e destruir cada um dos mapas criados você mesmo.

Sintaxe:

skeleton_slot_data(sprite, list);

Argumento Tipo Descrição
sprite Ativo Sprite The sprite index of the Spine skeletal animation to get the data from.
list DS List ID The ID of the DS list to populate with the DS maps.

Devoluções:

N/A

Exemplo:

var list = ds_list_create();
var open = true;
slot_name = "";
skeleton_slot_data(sprite_index, list);

for (var i = 0; i < ds_list_size(list); i++)
{
    var map = list[| i];
    if open
    {
        if map[? "attachment"] == "(none)"
        {
            open = false;
            slot_name = map[? "name"];
        }
    }

    ds_map_destroy(map);
}

ds_list_destroy(list);

O código acima cria uma lista DS e depois a preenche com os dados do slot para o exemplo sprite. Estes dados são então analisados para extrair os mapas DS individuais com os dados do slot. Isto é então verificado para ver se há um slot vazio, e se assim for, a variável "slot_name" recebe o nome do slot vazio antes que o mapa DS seja destruído. Finalmente, destruímos a lista de DS, pois ela não é mais necessária.
