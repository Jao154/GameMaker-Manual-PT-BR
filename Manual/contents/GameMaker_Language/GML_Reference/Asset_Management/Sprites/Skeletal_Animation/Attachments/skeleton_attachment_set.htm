conjunto_esqueleto_de_encaixe

Uma animação esquelética sprite pode ter outras imagens adicionadas como anexos, com essas imagens sendo adicionadas a um slot nomeado (o nome é dado quando você cria o slot de anexo em seu programa de animação) e elas serão desenhadas junto com a animação do atual sprite. Com esta função, você pode definir um anexo para um determinado slot, onde é necessário dar os nomes (como strings) do slot e do anexo. Esses nomes são definidos pelo programa de animação utilizado, ou (no caso do anexo) quando você chama skeleton_attachment_create().

Note que você também pode passar um sprite_index in como anexo, e que sprite será usado, ou você pode usar -1 para remover o anexo do slot. Quando você passar um índice sprite como argumento, ele criará um slot de anexo usando o nome do sprite como o string para nomear o slot (assim usando spr_sword, por exemplo, criará um slot de anexo "spr_sword"), e o slot usará o primeiro índice de imagem (0) do sprite, assim como seu offset de origem x e y, com uma escala de (1,1) e uma rotação de 0.

Sintaxe:

skeleton_attachment_set(slot, attachment);

Argumento Tipo Descrição
slot Cordão The slot name (a string) to get the attachment of.
attachment Ativo deCordas ou Sprite The name (as a string or a sprite_index) of the attachment image.

Devoluções:

String

Exemplo:

if skeleton_attachment_get("slot_leftHand") == ""
{
    skeleton_attachment_set("slot_leftHand", choose("sword", "spear", "knife"));
}

O código acima verificaria o nome do anexo atualmente atribuído para o slot chamado "slot_leftHand" e se um string vazio for devolvido, um novo sprite será anexado.
