asset_add_tags

Com esta função você pode adicionar uma ou mais tags strings a qualquer asset a partir do navegador asset. Você fornece ou o nome asset (como string) ou seu índice asset, assim como uma única tag string ou uma array onde cada item é uma única tag string. Se você fornecer um valor de índice asset, então você precisará fornecer o argumento opcional do tipo asset (uma constante), pois assets de diferentes tipos pode ter o mesmo índice, mesmo que eles não possam ter o mesmo nome. Os tipos disponíveis asset estão listados na tabela abaixo:

Constante Descrição
asset_object O nome dado refere-se a um objeto.
asset_sprite O nome dado refere-se a um duende.
asset_sound O nome dado refere-se a um som.
asset_room O nome dado refere-se a um room.
asset_tiles O nome dado refere-se a um conjunto de azulejos.
asset_path O nome dado refere-se a um caminho.
asset_script O nome dado refere-se a um script.
asset_font O nome dado refere-se a uma fonte.
asset_timeline O nome dado refere-se a uma linha de tempo.
asset_shader O nome dado refere-se a um sombreador.
asset_animationcurve O nome dado refere-se a uma Curva de Animação.
asset_sequence O nome dado refere-se a uma sequência.

Se a função conseguir adicionar a(s) etiqueta(s), ela retornará true caso contrário, retornará false.

Sintaxe:

asset_add_tags(name_or_index, tags, [asset_type]);

Argumento Tipo Descrição
name_or_index Cordão ou Ativo The name of the asset (a string) or its index value (an integer).
tags Cordão ou matriz de cordas A single asset tag string or an array with various asset tags.
[asset_type] Tipo de ativo Constante OPTIONAL O tipo de ativo no qual as etiquetas devem ser colocadas, usado apenas quando se fornece um índice para o primeiro argumento.

Devoluções:

Boolean

Exemplo:

var _a = array_create(3);
_a[0] = "enemy";
_a[1] = "all_levels";
_a[2] = "boss";
asset_add_tags(obj_Enemy_Boss_Parent, _a, asset_object);

O código acima irá criar uma série de tags e depois adicioná-las ao dado object.
