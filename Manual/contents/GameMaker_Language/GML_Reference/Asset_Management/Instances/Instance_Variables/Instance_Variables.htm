Variáveis de instância

Quando você criar um novo object, ele virá com certas variáveis já inicializadas com valores padrão. Estas variáveis chamamos variáveis de instância, já que serão criadas para cada instância que você colocar do object em seu jogo rooms, e uma vez criada uma instância, os valores destas variáveis são exclusivos para aquela instância e somente para aquela instância. Algumas destas variáveis você usará muito, como x e y, enquanto outras são requeridas menos dependendo do que você quer fazer, mas em geral são muito úteis e onde possível você deve procurar usá-las em vez de criar suas próprias variáveis personalizadas.

A maioria das variáveis de instância podem ser definidas, bem como lidas, permitindo que você altere as propriedades e o comportamento de uma instância simplesmente ajustando o valor de uma determinada variável - você pode evitar que uma instância desenhe, por exemplo, simplesmente ajustando a variável incorporada visible para false. Abaixo você pode encontrar as diferentes variáveis que são inicializadas para todas as instâncias de todas as objects em seu jogo.

IMPORTANTE Se uma instância estiver em um sequence então algumas destas variáveis - como image_xscale / image_yscale / image_angle / y / x / - serão sobrescritas quando o sequence atualizar cada passo após começar a ser jogado.

Estas variáveis tratam de propriedades de instância geral:

  • id
  • visible
  • solid
  • persistent
  • depth
  • layer
  • alarm

Estas variáveis tratam da posição e do movimento da instância:

  • direction
  • friction
  • gravity
  • gravity_direction
  • hspeed
  • vspeed
  • speed
  • xstart
  • ystart
  • x
  • y
  • xprevious
  • yprevious

Propriedades do objeto

A variável a seguir contém o ID do índice único do object a partir do qual a instância foi criada:

  • object_index

Todas estas variáveis estão relacionadas ao sprite atribuído à instância e podem ser usadas para mudar o que é desenhado e como:

  • sprite_index
  • sprite_width
  • sprite_height
  • sprite_xoffset
  • sprite_yoffset
  • image_alpha
  • image_angle
  • image_blend
  • image_index
  • image_number
  • image_speed
  • image_xscale
  • image_yscale

Estas variáveis lidam com a máscara de colisão:

  • mask_index
  • bbox_bottom
  • bbox_left
  • bbox_right
  • bbox_top

Estas variáveis estão relacionadas a paths e como a instância interage com uma, se designada:

  • path_index
  • path_position
  • path_positionprevious
  • path_speed
  • path_scale
  • path_orientation
  • path_endaction

Estas variáveis servem para definir uma instância para usar uma linha do tempo:

  • timeline_index
  • timeline_running
  • timeline_speed
  • timeline_position
  • timeline_loop

Sequências

As seguintes variáveis armazenam informações relativas ao Sequence que podem estar controlando a instância a qualquer momento:

  • in_sequence
  • sequence_instance

Há um grande número de variáveis incorporadas para uso com as funções físicas do GameMaker, e para manter as coisas mais claras, elas podem ser encontradas na seção do manual que cobre tudo relacionado à simulação física:

  • Física
