persistente

Esta variável pode ser lida para descobrir se a instância está marcada como persistent ou não, ou pode ser usada para definir persistência para true (persistente) ou false (não persistente) para a instância. Uma instância persistente é aquela que será "transferida" de room para room, significando (por exemplo) que ela só tem que ser criada uma vez no início do jogo e estará presente em todos os outros rooms. Deve-se tomar cuidado com persistência, pois é fácil perder de vista as instâncias persistentes que podem levar a problemas mais tarde no desenvolvimento do jogo.

Quando você cria uma instância com persistência habilitada em room, esta variável será definida para true e lhe será atribuída uma camada ou uma profundidade (dependendo de qual função você usou para criar a instância do object). Quando a room for alterada, e se a seguinte room não tiver uma camada com o mesmo nome ou profundidade que a atribuída, então será criada uma nova camada para a instância que persiste em todo o rooms. Se você deu uma camada nomeada quando a instância foi criada, então a nova camada também terá o mesmo nome que a camada original, enquanto que se você atribuiu uma profundidade à instância, então a nova camada será nomeada "_layer_XXX", onde "XXX" é um valor hexadecimal usado para dar à camada um nome único. Estas camadas serão removidas do site room em Room End. Outra coisa a se notar sobre camadas e instâncias persistentes é que se você tiver atribuído à instância uma camada nomeada na criação, e houver outra camada no seguinte rooms com o mesmo nome, então a instância persistente será atribuída à camada com o mesmo nome , independentemente da profundidade da camada. Finalmente, se uma instância persistente se mover para um room com uma camada na mesma profundidade em que a instância foi criada, ela não será atribuída a esta camada, mas uma nova camada será criada na mesma profundidade (seguindo a convenção de nomenclatura explicada anteriormente).

Um persistente object ainda terá seus eventos Início do jogo, Fim do jogo, Início da sala e Fim da sala acionados, porém se você reiniciar o jogo (com, por exemplo, a função game_restart() ) todos os persistentes objects serão removidos e só existirão quando criados novamente pelo jogo. Observe também que se você desativar um object persistente, ele não passará mais de um room para outro a menos que seja reativado antes que o evento Fim do Room seja acionado.

Sintaxe:

persistent;

Devoluções:

Exemplo:

if !persistent
{
    persistent = true;
}

O código acima verificará a instância para ver se a persistência está marcada como false e se a instância NÃO é persistente a persistência flag está marcada como true.
