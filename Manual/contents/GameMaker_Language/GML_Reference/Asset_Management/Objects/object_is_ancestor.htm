object_is_ancestor

Esta função pode ser usada para verificar um object e ver se é um antepassado ou não de outro object. Portanto, ambos os argumentos precisam ser índices object e não os ids de instância, e o primeiro é sempre o object que você quer descobrir se é um filho do segundo argumento, que é sempre o object que quer verificar como o antepassado (pai).

Sintaxe:

object_is_ancestor(obj, par);

Argumento Tipo Descrição
obj Objeto Ativo The object that is being checked as the child.
par Objeto Ativo The object that is being checked as the ancestor (parent).

Devoluções:

Boolean

Exemplo:

if object_is_ancestor(object_index, obj_Enemy)
{
    instance_destroy();
}

O código acima verifica se a instância que executa o código é um filho do "obj_Enemy" object, e se assim for, é destruído.
