evento_erecido

Esta função chamará o evento atual do pai object da instância. Normalmente, quando uma instância tem um pai object, ela automaticamente inherits todos os mesmos eventos que o pai, mas se (por exemplo) seu pai object tiver um evento de criação e você adicionar um a seu filho object, todas as instâncias da criança object executarão o novo evento de criação que você adicionou e não o que está no pai object. Caso você precise usar tanto o evento do pai object quanto o evento do filho object do mesmo tipo, você deve usar esta função, pois ele executará o evento do pai object antes de continuar com o resto do código ou ações que o evento do filho contém.

Sintaxe:

event_inherited();

Devoluções:

Exemplo:

event_inherited();
switch (sprite_index)
{
    case spr_Enemy_1: dmg += 2; break;
    case spr_Enemy_4: dmg -= 1; break;
    case spr_Enemy_10: dmg +=10; break;
}

O código acima chama o evento pai herdado (no qual inicializamos a variável "dmg") e em seguida modificamos a variável "dmg". Se não houver um pai especificado para a instância executando este código, receberíamos um erro de "variável desconhecida", pois dmg não foi definido.
