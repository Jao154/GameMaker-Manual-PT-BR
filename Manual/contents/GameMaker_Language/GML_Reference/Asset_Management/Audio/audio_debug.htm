audio_debug

Esta função pode ser usada para exibir informações de depuração sobre o sistema de áudio, com true ligando-o e false desligando-o para switch. Quando você estiver debugging áudio desta forma, a sobreposição mostrará as seguintes informações:

  • Cabo - O cabo de fonte OpenAL (único para cada som tocado)
  • Cabo de dados - O cabo OpenAL buffer (mostra o mesmo valor para múltiplos sons com o mesmo cabo de fonte)
  • Buffers processados - quantos buffers já foram jogados
  • Buffers enfileirados - quantos buffers foram enfileirados (e ainda estão esperando para jogar)
  • Ganho - O valor de ganho para o áudio (0 - 65536)
  • Play Offset - a posição da amostra (posição de jogo atual) no buffer atual
  • Enfileirado Buffers Barras - um pequeno diagrama de caixa do processado/fileirado buffers com o offset de jogo (posição atual) desenhado como uma linha azul

Para grupos sincronizados de áudio, você também tem a função audio_sync_group_debug().

NOTA Esta função não funciona na plataforma alvo HTML5.

Sintaxe:

audio_debug(enable);

ArgumentoTipo Descrição
enable Enable (set to true) ou desabilitar (false) áudio debugging.

Devoluções:

Exemplo:

if (debug_mode)
{
    audio_debug(true);
}
else
{
    audio_debug(false);
}

O código acima irá switch ligado ou desligado da camada de depuração de áudio, dependendo se o jogo está ou não rodando em modo de depuração.
