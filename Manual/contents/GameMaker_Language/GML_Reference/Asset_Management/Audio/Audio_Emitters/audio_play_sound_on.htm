audio_play_sound_on

Com esta função você pode tocar qualquer recurso sonoro através de um emissor, com qualquer mudança no ganho do emissor, posição, tom ou velocidade afetando a forma como o usuário ouve o som final sendo tocado. Você fornece o índice do emissor a ser utilizado, o índice do som a ser reproduzido e então especifica se o som é para loop ou não. Em seguida, você pode atribuir uma prioridade, que é então usada para determinar como os sons são tratados quando o número de sons tocados está acima do limite estabelecido pela função audio_channel_num(). Sons de prioridade mais baixa serão interrompidos em favor de sons de prioridade mais alta, e o valor de prioridade pode ser qualquer número real (o valor real é arbitrário, e pode ser de 0 a 1 ou 0 a 100, pois o GameMaker os priorizará da mesma forma). Observe que ao lidar com prioridade, quanto maior o número, maior será a prioridade, de modo que um som com prioridade 100 será favorecido em relação a um som com prioridade 1.

Esta função também retornará um número de índice único para o som que está sendo reproduzido, que pode então ser armazenado em uma variável para que você possa pausá-lo ou pará-lo com as funções apropriadas. Isto significa que se você tiver múltiplas instâncias do mesmo som sendo reproduzido a qualquer momento, você pode direcionar apenas uma instância desse som para lidar com o uso das funções de áudio.

Sintaxe:

audio_play_sound_on(emitter, sound, loop, priority, [gain], [offset], [pitch], [listener_mask]);

Argumento Tipo Descrição
emitter ID do emissor de áudio The index of the emitter to use.
sound Bens sólidos The index of the sound to use.
loop Booleano Flags the sound as looping or not.
priority Real Set the channel priority for the sound.
gain Real OPTIONAL Valor para o ganho.
offset Real OPTIONAL O tempo (em segundos) para definir o ponto de partida. Valores maiores do que o comprimento do som dado são ignorados.
pitch Real OPTIONAL O multiplicador de passo (padrão 1).
listener_mask Real OPTIONAL Os dados de bitmask a serem definidos para o som. No alvo HTML5 isto não terá efeito, pois o alvo não suporta mais que um ouvinte.

Devoluções:

Sound Instance ID

Exemplo 1:

if global.SFX
{
    audio_play_sound_on(s_emit, snd_Explode, false, 1);
}

O código acima verifica a variável global "SFX" e se ela retornar true então o som indexado na variável "snd_Explode" será reproduzido através do emissor indexado na variável "s_emit" com uma prioridade baixa e sem looping.

Exemplo 2:

if hit == true
{
    audio_play_sound_on(s_emit, snd_Hit, false, 1, 1.3);
}

O código acima verifica se a instância que executou o código foi atingida. Se true tocar o som "snd_Hit" através do emissor "s_emit" com um ganho ligeiramente maior de 1,3. O ganho definido aqui é multiplicado pelo ganho do emissor para obter o ganho final no qual o som é tocado.
