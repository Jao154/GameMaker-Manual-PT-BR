audio_play_sound_at

Com esta função você pode reproduzir qualquer recurso sonoro em uma determinada posição dentro do espaço de áudio. Você fornece o índice de som e então atribui a ele uma posição dentro do espaço 3D. A posição padrão do ouvinte é (0, 0, 0), então isto significa que se o ouvinte não tiver sido movido e você quiser que o som venha da esquerda (por exemplo), você deve definir a posição x para um valor negativo (para mais informações sobre como definir a posição do ouvinte, veja audio_listener_position()).

Você também pode definir uma distância de queda (0 fará o som silencioso, o padrão é 100) o que fará o som desvanecer à medida que se afasta da posição do ouvinte. A forma como o som em si será ouvido dependerá da referência de queda (que é a distância sob a qual o volume da fonte normalmente cairia pela metade) e do fator de queda (que afeta somente o som que ultrapassa a distância de referência da queda). O fator padrão é normalmente 1, e o efeito dos diferentes valores de falloff dependerá do modelo escolhido (para um guia completo dos diferentes modelos de falloff e como estes valores são usados, por favor veja a função audio_falloff_set_model()).

Os dois argumentos seguintes são para definir o som é para loop ou não e para atribuir uma prioridade ao som. Esta prioridade é então usada para determinar como os sons são tratados quando o número de sons tocados está acima do limite estabelecido pela função audio_channel_num(). Sons de prioridade mais baixa serão interrompidos em favor de sons de prioridade mais alta, e o valor de prioridade pode ser qualquer número real (o valor real é arbitrário, e pode ser de 0 a 1 ou 0 a 100, pois o GameMaker os priorizará da mesma forma). Observe que ao lidar com prioridade, quanto maior o número, maior será a prioridade, de modo que um som com prioridade 100 será favorecido em relação a um som com prioridade 1.

Esta função retornará um número de índice único para o som que está sendo tocado, que pode então ser armazenado em uma variável para que você possa pausá-lo ou pará-lo com as funções apropriadas. Isto significa que se você tiver múltiplas instâncias do mesmo som sendo reproduzido a qualquer momento, você pode direcionar apenas uma instância desse som para lidar com o uso das funções de áudio.

Sintaxe:

audio_play_sound_at(index, x, y, z, falloff_ref, falloff_max, falloff_factor, loop, priority, [gain], [offset], [pitch], [listener_mask]);

Argumento Tipo Descrição
index Bens sólidos The index of the sound to play.
x Real The x position.
y Real The y position.
z Real The z position.
falloff_ref Real The falloff reference relative to the listener (clamp).
falloff_max Real The maximum falloff distance relative to the listener.
falloff_factor Real The falloff factor (default 1).
loop Booleano Flags the sound as looping or not.
priority Real Set the channel priority for the sound.
gain Real OPTIONAL Valor para o ganho.
offset Real OPTIONAL O tempo (em segundos) para definir o ponto de partida. Valores maiores do que o comprimento do som dado são ignorados.
pitch Real OPTIONAL O multiplicador de passo (padrão 1).
listener_mask Real OPTIONAL Os dados de bitmask a serem definidos para o som. No alvo HTML5 isto não terá efeito, pois o alvo não suporta mais do que um ouvinte.

Devoluções:

Sound Instance ID

Exemplo 1:

if global.SFX
{
    audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1);
}

O código acima verifica a variável global "SFX" e se ela retornar true então o som indexado na variável "snd_Waterfall" será laçado em sua posição room, com uma referência de queda de 100, uma distância de queda de 300, um fator de queda de 1 e uma baixa prioridade.

Exemplo 2:

if global.SFX
{
    audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1, 1, 2);
}

O código acima verifica a variável global "SFX" e se ela retornar true então o som indexado na variável "snd_Waterfall" será laçado em sua posição room usando as configurações de falloff dadas e nenhuma mudança no ganho. O som começa imediatamente a tocar a partir do segundo 2.
