audio_free_play_queue

Esta função é utilizada para liberar a memória associada ao áudio dado queue. O índice queue é o valor retornado quando você criou o queue usando a função audio_create_play_queue()e esta função deve ser chamada quando o queue não for mais necessário para evitar vazamentos de memória. A liberação do queue irá parar qualquer som que esteja tocando, e não é possível apagar o buffer do qual um som está sendo transmitido até que o queue seja liberado. Esta função desencadeará um Evento Assíncrono de Reprodução de Áudio, e neste evento será criado um mapa DS especial na variável async_load com os seguintes pares de chaves/valores:

  • "queue_id" - o queue que terminou de jogar dados
  • "buffer_id" - o buffer que não está mais sendo jogado de
  • "queue_shutdown" - isto é ajustado para 0 durante a reprodução normal e 1 quando o evento é recebido porque você liberou o queue com esta função. Quando esta função é definida em 1, você não deseja acessar mais dados no queue.

Sintaxe:

audio_free_play_queue(queueIndex);

Argumento Tipo Descrição
queueIndex Áudio Queue ID The index of the queue to free.

Devoluções:

N/A

Exemplo:

var q_id = async_load[? "queue_id"];
if (q_id == audio_queue)
{
    audio_free_play_queue(audio_queue);
    buffer_delete(audio_buffer);
}

O código acima iria no evento assíncrono de reprodução de áudio e verifica se o áudio que terminou de tocar é da "audio_queue" e se é ele libera o queue e o buffer associado a ele.
