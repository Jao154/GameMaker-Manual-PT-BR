audio_get_recorder_info

Esta função retornará um Mapa DS com informações sobre o índice de origem do registrador em questão. Você pode descobrir quantas fontes de gravadores estão disponíveis usando a função audio_get_recorder_count()e o mapa retornado conterá os seguintes pares chave/valor:

  • "nome" - um nome para descrever o dispositivo
  • "índice" - o índice a ser utilizado para registrar
  • "data_format" - os dados de formato serão devolvidos em (atualmente é sempre buffer_s16, mas outros formatos podem ser suportados no futuro)
  • "taxa_amostra" - a taxa de amostragem (em Hz) dos dados retornados (atualmente fixada em 16000hz, mas isso pode mudar no futuro)
  • "canais" - a constante audio_mono (outras constantes para estéreo e 3D podem ser suportadas no futuro)

Note que embora a função crie um mapa DS para você, ela não o remove novamente mais tarde e, portanto, você mesmo deve estar destruindo o mapa quando ele não for mais necessário para evitar qualquer vazamento de memória.

NOTA A maioria das plataformas suporta a gravação de áudio de alguma forma, mas isso não significa que todos os dispositivos o permitam, mesmo que a plataforma o permita, então você deve sempre verificar se o audio_get_recorder_count() retorna um valor maior que 0 para verificar se os dispositivos de gravação estão disponíveis antes de utilizar o restante das funções de gravação.

Sintaxe:

audio_get_recorder_info(recorder_index);

Argumento Tipo Descrição
recorder_index Real The index of the recorder source to get the information from.

Devoluções:

DS Map ID

Exemplo:

len = async_load[? "data_len"];
audio_buff = buffer_create(len, buffer_fast, 1);
buffer_copy(async_load[? "buffer_id"], 0, len, buff, 0);
audio_queue_sound(audio_queue, audio_buff, 0, len);
audio_play_sound(audio_queue, 10, 0);

O código acima seria chamado no evento assíncrono de gravação de áudio e atribui alguns dados gravados a um buffer, que é então adicionado a um áudio queue. Isto é então reproduzido.
