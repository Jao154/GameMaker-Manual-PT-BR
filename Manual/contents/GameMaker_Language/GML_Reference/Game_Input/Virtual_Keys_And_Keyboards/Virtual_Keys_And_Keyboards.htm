Chaves virtuais e teclados virtuais

Ao desenvolver jogos para dispositivos touchscreen, muitas vezes é importante obter a entrada do teclado por parte do usuário. Ao utilizar o GameMaker, isto pode ser feito de uma de duas maneiras:

  • Crie suas próprias chaves virtuais na tela para receber entrada
  • Abrir o teclado virtual do sistema operacional e obter informações a partir dele.

O que você usa dependerá muito do jogo e do que você precisa, mas em geral você usaria teclas virtuais quando quiser ter botões na tela para controle do jogador ou para menus, enquanto você usaria o teclado virtual quando precisar da entrada do jogador, como obter um nome ou definir um valor.

Para que seus jogos funcionem corretamente em dispositivos móveis com tela sensível ao toque, talvez seja necessário atribuir áreas da tela para responder a certas coisas quando elas são tocadas pelo usuário. Isto é feito atribuindo uma "chave virtual" a uma chave de teclado padrão mapeada internamente e depois usando os eventos de teclado que eles geram para controlar sua aplicação. Vale notar que estas "teclas virtuais" são mapeadas para a posição da tela em vez da posição room e, portanto, os valores x/y são absolutos, com base no tamanho da tela. Isto significa que você não precisa se preocupar com o uso das vistas ou das coordenadas relativas room, e pode simplesmente definir as teclas no Create Event of an object (você só precisa definir uma tecla virtual uma vez por sala), depois desenhe sua tecla sprites no Draw GUI Event, dimensionando a camada GUI para ser a mesma que a tela.

NOTA Estas funções não funcionarão no Windows, Ubuntu (Linux), nem no MacOS, plataformas alvo usando uma tela sensível ao toque, embora recebam cliques de mouse como "toques".

Estas funções são suportadas nos alvos HTML5 e Opera GX.

As funções-chave virtuais disponíveis são:

  • virtual_key_add
  • virtual_key_show
  • virtual_key_hide
  • virtual_key_delete

Se você precisar de texto ou entrada numérica em seu jogo em um dispositivo touchscreen, então você pode usar as funções de teclado virtual fornecidas para usar o teclado do SO do dispositivo. Estas funções chamarão ou ocultarão o teclado do SO e aceitarão a entrada dele, porém ele não acionará os eventos regulares do teclado, mas atualizará a variável keyboard_string com a entrada do usuário (isto se deve ao fato de que o teclado virtual pode inserir palavras inteiras e não apenas caracteres individuais).

A abertura ou fechamento do teclado virtual irá gerar Eventos Assíncronos do Sistema, nos quais o async_load O mapa DS será preenchido com os seguintes pares chave/valor:

  • "event_type" - o tipo de evento do sistema recebido, que será o "status do teclado virtual" para teclados virtuais.
  • "screen_height" - a altura do teclado virtual (em pixels). Esta será 0 se o teclado for invisível.
  • "keyboard_status" - o status atual do teclado, retornado como uma das seguintes seqüências:
    • "escondido".
    • "oculto".
    • "mostrando"
    • "visível".

As funções de teclado virtual disponíveis são:

NOTA Estas funções são válidas apenas para as plataformas alvo Android (incluindo AndroidTV), iOS (incluindo tvOS), e UWP .

  • keyboard_virtual_show
  • keyboard_virtual_hide
  • keyboard_virtual_status
  • keyboard_virtual_height
