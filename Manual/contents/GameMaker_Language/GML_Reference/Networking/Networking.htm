Trabalho em rede

As funções de rede GameMaker são baseadas em soquetes, que é um padrão da indústria em todas as plataformas. Com os soquetes, você tem a capacidade de criar tanto clientes como servers usando o simples GML que permite criar até mesmo um jogo de um único jogador usando o modelo cliente/servidor (algo que permitirá que o multiplayer seja adicionado facilmente mais tarde).

Basicamente, uma tomada é um object que pode enviar, receber, conectar e escutar as portas da rede. Estas funções tratam apenas de conexões TCP/IP, que é no que a Internet se baseia (IPv4, para ser mais preciso). Em essência, isto permite conectar um socket usando um endereço IP (digamos 192.168.1.10) a outro socket em outro endereço IP (digamos 192.168.1.11). Uma vez conectados, estes dois soquetes podem enviar dados para frente e para trás usando as funções network_send_* (como network_send_packet()) e o evento assíncrono dedicado que é desencadeado por eles. Este evento assíncrono gerará um mapa DS contendo os dados recebidos, bem como outros detalhes, e você pode encontrar os detalhes completos na seção sobre o Evento Async da Rede.

Em vez de programas que precisam ler e lidar com cada pedaço de tráfego de rede que entra em uma máquina, os endereços IP também lidam com portas, que são numeradas de 0 a 65535. Isto permite que cada programa receba apenas os pacotes de dados em que está interessado, e não tudo o que foi enviado por cada programa. Isto também economiza tempo considerável da CPU, pois reduz seriamente o processamento de dados, de modo que com o GameMaker você também pode "ouvir" as portas. Assim, um server criará um soquete, e depois o ligará a uma porta específica para ouvir. Ele será então notificado das conexões, desconexões e transferências de dados a partir desse soquete.

Os próprios "pacotes" de dados são compostos de dados binários retirados de um buffer. Este buffer pode ser criado e manipulado usando as funções do GameMaker Buffer.

Vale notar que se você estiver criando um projeto para o alvo HTML5, então você não poderá usar o server criar funções devido a restrições do navegador e terá que escrever seu próprio server para esta plataforma usando algo como Electron ou NodeJS. Observe também que se seu jogo for uma plataforma cruzada e você antecipar a conexão a uma instância HTML5 do jogo a partir de um server rodando em um dispositivo móvel ou desktop, então seu server só poderá se comunicar com a instância HTML5 usando o protocolo Web Socket (isto é feito usando a constante network_socket_ws ou network_socket_wss - veja network_create_server() para mais informações). Isto significa que seu server pode ter que criar dois soquetes em portas diferentes, um para conectar a alvos não HTML5 (Android, Windows, etc...) e outro para conectar ao alvo HTML5.

As seguintes funções existem para que você possa utilizar em rede:

  • network_create_server
  • rede_criar_servidor_servidor_desenhar
  • network_create_socket
  • network_create_socket_ext
  • network_connect
  • network_connect_async
  • network_connect_raw
  • network_connect_async_raw
  • network_resolve
  • network_set_config
  • network_set_timeout
  • network_send_broadcast
  • network_send_packet
  • network_send_raw
  • network_send_udp
  • network_send_udp_raw
  • network_destroy
