Guia de Início Rápido

Bem-vindo ao Guia de Início Rápido! Esta seção do manual foi projetada para familiarizá-lo com os aspectos mais importantes do GameMaker e lhe mostrará o básico para que você faça jogos o mais rápido possível. O guia cobre a funcionalidade essencial IDE e também fornece uma breve introdução à programação, para que você possa aproveitar ao máximo o produto o mais rápido possível, quer você use o GML Code ou o GML Visual.

Para ilustrar o fluxo de trabalho do GameMaker e fazer com que você faça algo, preparamos este guia para levá-lo do básico do que é GameMaker e do que é um programa, através da criação do geral assets e até como codificar coisas básicas como movimento e pontuação.

Fazer jogos com o GameMaker é, conceitualmente, um assunto bastante simples e direto, seguindo estas etapas gerais:

  • Criar recursos de imagem (sprites, conjuntos de azulejos, fontes) e adicioná-los ao Asset Browser.
  • Crie ativos objetos para representar as coisas que você quer usar em seu jogo e atribua a eles a imagem exigida assets.
  • Programe estes objects para fazer coisas em resposta a eventos específicos.
  • Adicione instâncias destes objects em uma sala onde eles executarão seus programas.

Nas páginas seguintes, entraremos nas especificidades de cada um dos itens listados acima, mas esse é o fluxo geral de ações necessárias para fazer um jogo. O importante aqui é observar que seus principais "blocos de construção" para o jogo serão sempre sprites, objects e rooms, embora haja uma série de subsidiárias assets que também podem ser usadas para adicionar funcionalidade ou comportamentos extras a estes itens (como shadersou linhas do tempo).

Agora, antes de continuarmos a explicar cada uma dessas coisas com mais detalhes, rapidamente esboçaremos como o GameMaker transformará os blocos de construção acima em seu jogo. Basicamente, tudo acontece no que é chamado de Game Loop, que tem uma velocidade medida em quadros por segundo, e um "quadro" é chamado de Step in GameMaker. O frame padrão por segundo (ou FPS) é 60, o que significa que para cada segundo que passar o GameMaker executará o jogo loop 60 vezes, e cada passo (loop/frame) um número de eventos ocorrerá e o jogo progredirá.

Para ajudar a entender isso, vamos imaginar que você tenha um personagem que queira se mover pelo jogo room. Você criaria um duende para esse personagem, depois criaria um objeto e lhe atribuiria o sprite, e no object você adicionaria um evento chamado Step Event. Neste evento, você diria a ele para se movimentar corretamente por 2 pixels. Você então colocaria uma instância desse object no jogo room, e quando você executasse o jogo, a instância começaria a se mover para a direita por 2 pixels a cada etapa (o Evento de Etapa é executado uma vez a cada etapa do jogo), de modo que em 1 segundo a instância se moveu 120 pixels para a direita.

No exemplo acima, falamos em dizer ao object para ir para a direita, mas como fazer isso? Bem, isso é feito usando um programa que escrevemos usando ou GML Visual ou GML Code. Qual deles você usa depende de você e da maneira que você quer trabalhar, mas ao usá-los, você está programando seu jogo. A próxima seção trata disso com mais detalhes.
