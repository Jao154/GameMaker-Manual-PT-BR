Conflitos

Nesta seção vamos analisar a resolução de conflitos entre um repositório local clonado e um repositório principal. Supondo que você esteja usando agora um repositório local, precisaremos editar um arquivo e depois causar algum tipo de conflito, então abra o teste script que temos usado e mude a mensagem "Hello World" para algo como isto:

show_debug_message("Hello World, How Are You?");

Agora você deve salvar o projeto e comprometê-lo com o repositório local. Agora, como estamos ligados a um repositório mestre remoto (mesmo que seja um que tenhamos clonado no mesmo disco), precisamos fazer um Push para que as mudanças sejam feitas no master. Para isso, vá para Controle de Fonte > Push Changes, que nos dará uma atualização de status na guia Output Source Control:

Agora precisamos abrir nosso projeto original (não o clonado), então navegue até onde você o salvou em disco - ou simplesmente verifique os Projetos Recentes no menu Arquivo, como deve ser diretamente abaixo do atual - e carregue-o no GameMaker. Uma maneira fácil de verificar se você tem o projeto certo é simplesmente verificar o script que você tem e se a mensagem é "Olá Mundo" então você tem o certo, ou você pode abrir a janela Ver Histórico e verificar dessa maneira.

Agora você precisa editar novamente o script, então faça com que ele leia algo como isto:

show_debug_message("This will cause a conflict.");

Isto pode ser salvo em disco, e as mudanças comprometidas, mas se você tentar empurrar estas mudanças, você receberá a seguinte mensagem:

Alguém já fez mudanças que você ainda não fez! Isto significa que você precisa fazer um puxão no repositório e ver qual é exatamente o problema, então vá para Controle de Fonte > Puxar Mudanças. Isto mostrará duas janelas, a primeira é um aviso sobre arquivos alterados (como nas ações anteriores, você quer escolher Recarregar aqui e não Salvar), e a outra é a janela Conflitos :

Esta janela mostrará os arquivos em conflito em uma lista à esquerda, e você pode selecionar os arquivos dessa lista usando o botão esquerdo do mouse (ou selecionar vários arquivos usando / ou junto com ). Você tem quatro opções para lidar com os arquivos em conflito, disponíveis a partir dos botões à direita:

  • Use o deles - Isto irá sobregravar as mudanças que você fez com o do repositório remoto.
  • Use Mine - Isto irá anular as mudanças do repositório remoto com as que você acabou de fazer.
  • Fusão - Isto tentará abrir uma Ferramenta de Fusão Externa para lidar com o conflito.
  • Escolha a Ferramenta de Integração - Isto lhe permitirá configurar a Ferramenta de Integração (veja a seção sobre Ferramentas de Integração/Diff Externas para mais detalhes).

Agora, você pode usar os botões para resolver o conflito, mas antes disso, vamos apenas abrir o arquivo em conflito para ver exatamente qual é a questão. Se você abrir o script que está em conflito, agora ele se parecerá com isto:

Isto mostra que "Esta Vontade Causa um Conflito" está na CABEÇA (isto é o que temos empurrado para o repositório local) e que "Olá Mundo, Como você está?" está sendo puxado do mestre, o repositório remoto. Você pode voltar para a janela Conflitos agora e clicar em Usar o deles para puxar o arquivo alterado e sobrescrever o arquivo HEAD atual. O script agora será marcado como alterado e você deve fazer um novo commit e push antes de continuar.

Note que podemos realmente corrigir isso no GameMaker IDE. Se removermos as linhas 1, 2, 3 e 5 para que fiquemos apenas com show_debug_message("Hello World, How Are You?") e o salvarmos, o conflito será resolvido, pois Git assumirá que você sabe o que está fazendo com seu próprio arquivo. Entretanto, isto é propenso a erros, por isso é recomendável instalar uma ferramenta de fusão, depois configurar e usar isso ao invés de tentar fundir manualmente.
