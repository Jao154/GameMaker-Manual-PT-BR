Ferramenta de fusão

Quando você trabalha em um projeto com outras pessoas, sempre há uma chance de haver um conflito, pois várias pessoas editam o mesmo arquivo e a maioria dos sistemas source control pode fazer uma boa tentativa de fundir qualquer arquivo em conflito, mas quando há mudanças no mesmo local, é necessário um pouco de ajuda para saber o que fazer - é aqui que entra uma ferramenta de fusão para mostrar as mudanças, e deixar você decidir.

Como cada ferramenta de fusão é diferente, há quatro macros que podem ser usadas para ajudar a especificar arquivos que podem ser configurados a partir da janela Conflitos , clicando no botão Escolher ferramenta de fusão:

Ao clicar nesse botão, a janela Preferências será aberta no plugin Git:

Aqui você fornece um path para a ferramenta de fusão escolhida e depois uma macro (ou macros) para usar ao decidir o que fazer, com as seguintes opções disponíveis:

  • ${scm_mine}: path para sua versão
  • ${scm_theirs}: path para sua versão
  • ${scm_base}: path para a versão antes de ocorrerem as mudanças conflitantes
  • ${scm_merged}: path para onde a ferramenta de fusão deve economizar para

Como uma ajuda para a criação, quando você dá um path a certas ferramentas de fusão, o GameMaker irá preencher alguns padrões para começar, dependendo da ferramenta escolhida:

  • KDiff3: "${scm_theirs} "${scm_mine}" -o "${scm_merged}"
  • Meld: "${scm_mine}" "${scm_merged}" "${scm_theirs}"
  • Bloco de notas++: ${scm_merged}
  • TortoiseMerge: /base:"${scm_base}" /theirs:"${scm_theirs}" /mine:"${scm_mine}" /merged:"${scm_merged}"
  • DiffMerge: "${scm_mine}" "${scm_merged}" "${scm_theirs}"

Observe que se você não tiver (ou não quiser usar) uma ferramenta de fusão, então você pode abrir o script no IDE e tentar consertá-lo a partir daí.

Então, voltando ao nosso exemplo Conflito, como indicado na seção sobre Conflitos... Fizemos uma atração do mestre e um dos arquivos está em conflito. Isto abre a janela Source Control Conflicts , onde você pode clicar no conflituoso script, e agora clique em Merge, já que você configurou sua ferramenta Merge. Clicar em Merge abrirá agora a ferramenta onde você pode lidar com o conflito e depois usar a(s) macro(s) fornecida(s) para fazer as mudanças necessárias. Uma vez que você tenha terminado e fechado a ferramenta de fusão, o GameMaker IDE mostrará a seguinte janela de confirmação:

Basta responder "Sim" a esta solicitação, pois você mesmo acabou de fazê-lo usando a ferramenta de fusão, e então você precisa comprometer esta mudança para significar o fim da operação de fusão(Controle de Fonte > Comprometer Mudanças). A janela Commit agora mostrará que uma fusão está em andamento, assim como mostrará alguns arquivos não escalonados:

Estes arquivos não encadeados são nosso arquivo script com a adição de .base, .ours e .theirs. Podemos excluir todos eles com segurança, então clique no superior e depois + no inferior para selecioná-los todos. Em seguida, clicamos com o botão direito do mouse nos arquivos selecionados e escolhemos Eliminar caminho para removê-los. Há também uma mensagem de compromisso pré-preenchida para nós, afirmando que se tratava de uma fusão e que havia um conflito (podemos acrescentar ou editar como quisermos).

Podemos clicar em Comprometer agora.

Devido ao fato de que você pode ter problemas com outros arquivos, isso significa que seu fluxo de trabalho ao utilizar repositórios remotos ou compartilhados deve ser o seguinte:

Make Changes > Save > Commit > Pull > Merge > Push

Ferramenta Diff

Você também deve criar uma Ferramenta Diff para uso quando fizer compromissos com um repositório. Esta é configurada a partir das Preferências, o mesmo que para a ferramenta Fundir:

A ferramenta Diff é configurada de forma muito semelhante à ferramenta Merge, mas utiliza apenas as opções ${scm_base} e ${scm_theirs} - neste caso, ${scm_base} representa o arquivo não modificado, enquanto ${scm_theirs} representa o estado atual do arquivo. As Opções da Ferramenta devem ser pré-populadas com as opções padrão para Meld, KDiff3 e TortoiseMerge, assim como foram para a ferramenta Merge. Note que o GameMaker não suporta a saída de patches de diff de arquivo único, portanto não há um padrão para o Notepad++.

Para visualizar as mudanças entre as revisões atuais e as anteriores usando a ferramenta Diff, basta abrir a janela Comprometer e depois clicar duas vezes em um arquivo. Se a ferramenta tiver sido configurada corretamente, ela será lançada e você poderá ver as mudanças entre os arquivos.
