O Perfilador

O Debugger permite que você passe por seu código e encontre quaisquer problemas com a base de código, mas às vezes você precisa verificar detalhes mais sutis a fim de otimizar adequadamente seus jogos. A opção Perfil ajuda a fazer exatamente isso, dando-lhe uma visão geral do desempenho, mostrando o tempo que leva para chamar funções, scripts, e até mesmo quanto tempo levou para completar um evento específico.

Além da janela principal mostrando todos os diferentes eventos, scripts e funções sendo chamadas, você também tem as seguintes opções para usar o Profiler:

Habilitar / Desabilitar

Este é o botão para ativar ou desativar o Profiler. Quando ativado, o profiler será iniciado e você começará a ver eventos, scripts e as funções serão listadas na janela principal juntamente com diferentes valores de dados para cada um deles. Como estes dados são exibidos dependerá do Mostrador de Tempo e do Modo de Visualização (ambos explicados com mais profundidade abaixo).


Exibição do tempo

Esta caixa de seleção permite a switch entre o tempo médio e o tempo absoluto para o perfilador. Quando não for verificado, os valores mostrados na janela principal serão o número total de chamadas feitas para o script ou evento e o tempo total levado (os tempos são em microssegundos) ao longo do perfil. Entretanto, se você habilitar isto, então estas colunas mostram o tempo médio gasto por etapa para cada função, script ou evento, e o número médio de vezes que foi chamado.


Modo de visualização

Este menu permite que você escolha o modo de visualização, que pode ser Top Down ou Bottom Up.

Selecionando o modo de visualização de cima para baixo mostrará o perfil de cima para baixo (mais ou menos o mesmo que a hierarquia de calltack) para que ele mostre o evento, depois o script e depois quaisquer funções utilizadas. Note que você pode clicar duas vezes em em qualquer uma das entradas para que ele seja aberto na janela de depuração Source , se você ter um. Se você vir um ícone ao lado de um nome, então há mais script ou chamadas de função dentro dele. e clicar na seção irá expandir a árvore para mostrá-los.

Selecionando o modo de vista de baixo para cima, você poderá ver todas as funções de baixo para cima e script chamadas individualmente. Ao usar este modo de visualização, clicando no expandirá a função ou script para mostrar o que realmente chamou.


Ver Alvo

O View Target permite que você escolha entre visualizar o projeto GML (dados de evento e código), os processos do motor de fundo ou ambos. Se você escolher "GML", então você terá os eventos, funções e scripts sendo chamado cada passo enquanto a visão "Motor" mostrará apenas as chamadas do motor exigidas pelo projeto, permitindo que você veja quaisquer gargalos na maneira como seu projeto está lidando com as coisas. Se você selecionar "Ambos", então ambos os conjuntos de dados ser exibido, mas note que quando combinado com a visão "Bottom Up", os diferentes script e chamadas de função serão aninhados dentro das chamadas do motor e você precisará clicar no para expandi-los para vê-los.



Os dados do perfil são mostrados em quatro colunas e, clicando em qualquer uma delas, os dados serão ordenados de acordo com o critério da coluna.

  • Nome: Este é o nome do evento, script ou função sendo chamada, e pode ter um ao lado para denotar mais chamadas para ele ou dele (dependendo do Modo de Visualização que você selecionou). Você pode clicar duas vezes em em qualquer um dos nomes para que ele abra o bloco de código ou script dentro da janela Fonte. Clicando no topo desta coluna, os nomes serão ordenados em ordem alfabética.
  • Tempo: Isto mostrará ou o tempo total que uma função (ou script ou evento) levou durante a criação do perfil, ou o tempo médio que levou cada passo a ser chamado (dependendo da opção de exibição de tempo selecionada) e é exibido em microssegundos.
  • Chamadas: Aqui você pode ver o número total de chamadas para um determinado evento, função ou script, ou o número médio de chamadas para ele em uma única etapa (novamente, dependendo da opção de exibição de tempo selecionada).
  • Passo%: O valor mostrado aqui é a porcentagem geral de um único passo que a chamada deu.
