Compilar erros

O segundo tipo de erro que pode ser relatado para seu jogo é o erro de compilação. Um erro de compilador acontece quando seu jogo encontra algum tipo de erro que o verificador de sintaxe pode não ter sido capaz de detectar - é muito fácil esquecer de declarar uma variável ou digitar erroneamente uma string como real, por exemplo - ou quando um erro está relacionado a como você configurou as ferramentas de compilação a partir das Preferências de Plataforma para a plataforma alvo. Estes erros serão pegos pelo GameMaker enquanto o jogo estiver sendo compilado e esta informação também será mostrada na Saída de Erro do Compilador, como mostrado abaixo:

NOTA Estes erros normalmente não aparecem em uma janela e simplesmente mostrarão "Build Failed" na saída do compilador. Se uma janela aparecer é normalmente um erro de corredor, que é explicado aqui.

As mensagens de erro do compilador seguirão todas o mesmo formato:

[object] - [event] - [Line Number]: [error string]

Se o erro for encontrado em um script então ele será simplesmente:

[script] - [Line Number]: [error string]

Você pode então clicar duas vezes em em qualquer uma das entradas de erro do compilador para abrir o dado asset na posição marcada como dando o erro, e então você pode usar as informações contidas na mensagem para apontar exatamente onde no object ou script o erro ocorreu. No entanto, às vezes esses erros podem parecer um pouco crípticos, portanto, abaixo você pode encontrar uma lista completa de todos os erros e uma breve explicação do que eles significam:
