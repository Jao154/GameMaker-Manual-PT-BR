Preferências dos editores de texto

As preferências dos editores de texto são usadas para definir como os diferentes editores de código/script são vistos e sentidos. A página principal tem as seguintes opções:

  • Atraso na verificação da sintaxe (ms): O tempo de atraso (em milissegundos) antes da execução do verificador de sintaxe. Este valor padrão é de 2000ms.
  • Atraso de Conclusão de Código (ms): O tempo de atraso (em milissegundos)
  • Abra scripts no editor de tela cheia: Ao marcar esta opção, os recursos script serão abertos automaticamente em um novo workspace. Se esta opção não for verificada, eles serão abertos em uma janela dentro do atual workspace. Esta opção é desmarcada por padrão.
  • Abra shaders no editor de tela cheia: Ao marcar esta opção, os recursos shader serão abertos automaticamente em um novo workspace. Se esta opção não for verificada, eles serão abertos em uma janela dentro do atual workspace. Esta opção é desmarcada por padrão.
  • Abra object eventos em editor de tela cheia: Ao marcar esta opção, os editores de código de eventos object abrirão automaticamente em um novo workspace. Se não for verificado, eles abrirão em uma janela acorrentada ao editor de eventos object. Esta opção é desmarcada por padrão.
  • Abra o evento scripts como abas na mesma janela: Ao adicionar código ou GML Visual a um evento, a janela de código é acorrentada à janela do evento, e quando esta opção for marcada, quaisquer outros eventos que forem abertos para edição serão adicionados à mesma janela como abas ao longo da parte superior. Ao desmarcar esta opção, cada evento será aberto como sua própria janela encadeada única. Esta opção é marcada por padrão.
  • Abra o link scripts como abas na mesma janela: Quando você tiver vários links scripts e abri-los, por padrão eles serão agrupados na mesma janela que as abas. Desmarcando esta opção, eles serão forçados a abrir cada uma delas em sua própria janela/espaço de trabalho. Esta opção é marcada por padrão.
  • Editor de texto panorâmico em foco: Quando você abre um editor de texto de um evento object, selecionando esta opção, o workspace será automaticamente visualizado para que o editor fique dentro dos limites. Esta opção está desligada por padrão.
  • Largura da janela de código padrão: Defina a largura padrão (em pixels) para a janela de código a ser aberta. O valor padrão é 900.
  • Altura padrão da janela de código: Defina a altura padrão (em pixels) para a janela de código a ser aberta. O valor padrão é 600.
  • Largura mínima da janela de código: Defina a largura mínima (em pixels) permitida pela janela de código. O valor padrão é 300.
  • Altura mínima da janela de código: Definir a altura mínima (em pixels) permitida pela janela de código. O valor padrão é 200.

Além das opções acima, há duas subcategorias, Editor de Código e Editor Visual, descritas abaixo.

Editor de código

As preferências de cores são usadas para mudar a forma como o código e o editor script exibem diferentes textos. Cada entrada tem um número de subentradas para permitir que você defina exatamente como cada pedaço de texto separado deve ficar, como mostrado na imagem de exemplo acima. Em geral, você pode definir a cor, font, tamanho e estilo para cada uma das diferentes opções. Mais abaixo, você também pode definir um estilo mais geral para coisas como seleção de linhas de colchetes, etc...

Sob as preferências de cor, você tem as diferentes opções de estilo. Estas preferências são usadas para definir o comportamento do editor de código/script, com as seguintes opções disponíveis:

  • Fique de olho: Se verificado, isto manterá a tecla de tabulação pressionada como caracteres de tabulação - ' \t' - no editor, enquanto desmarcada significa que as tabulações são convertidas em espaços. O padrão está desligado.
  • Tamanho da aba: O número de espaços de caracteres que ao pressionar a tecla "Tab" deve indentar o código. O padrão é 4.
  • Mostrar o espaço para brancos: Mostrar " ." para mostrar o espaço em branco (ver imagem abaixo). O padrão está desligado.
  • Mostrar números de linha: Mostrar os números das linhas à esquerda ou não. O default está ativado.
  • Mostrar linhas de aba: Mostrar " ...." para mostrar o espaço das abas. O padrão está desligado.
  • Habilitar o código de cores: Habilitar ou desabilitar a codificação por cores para o editor. Se desativado, o código não será estilizado, enquanto que quando ativado será estilizado usando a formatação dada acima na seção Cores. O padrão está ativado.
  • Limite de linha de documento: Defina o número máximo de linhas para qualquer documento do editor. O valor padrão é 0 (0 ou menos é essencialmente infinito) e qualquer outro valor limitará as linhas permitidas.
  • Traço de partida em novas linhas: Ativar ou desativar a auto-indentação para seu código. O padrão está ativado, o que manterá qualquer tabulação definida em uma linha quando uma nova linha for adicionada.
  • Auto travessão após a cinta: Ativar ou desativar a auto-indentação para seu código ao adicionar as chaves {}. O default está ativado, o qual irá tabular a próxima linha de código quando uma nova brace { e uma nova linha forem adicionadas.
  • Habilitar o preenchimento do código: Habilitar ou desabilitar a janela de conclusão de código. Ao escrever o código e isto estiver ativado, você terá uma janela popup mostrando possíveis funções que você pode usar com base no texto de entrada atual. O padrão está ativado.
  • Colunas de funções autoajustáveis: Ativar ou desativar a adição automática de colchetes () quando você usar o autocompletar. Ao escrever o código e isto for ativado, o IDE adicionará automaticamente os dois colchetes () que uma função requer, colocando o cursor dentro deles se a função levar argumentos, ou colocando o cursor depois deles se não levar argumentos. Este recurso só funciona quando a função tiver sido selecionada a partir do autocompletar. O padrão está ativado.
  • Desfazer / Redo Limite de pilha: Aqui você pode definir o limite de desfazer /Redo Limite de pilha. Qualquer coisa abaixo de 0 é essencialmente infinito (limitado pela memória disponível) enquanto valores positivos limitarão a pilha a esse número, removendo qualquer ação na cauda da pilha se o limite for atingido. O valor padrão é -1.
  • Destaque para os jogos de seleção: Isto destacará levemente qualquer seção de código que seja duplicada em outro lugar. Portanto, se você selecionar uma função, por exemplo, todas as instâncias desta função na janela de código atual também serão destacadas. A predefinição está ativada.
  • Codificação/conclusão de código colorido para funções obsoletas: Isto destacará qualquer seção de código que esteja obsoleta. O padrão está em vigor.

Editor visual

As preferências dos comentários controlam as diferentes opções de cor e formatação para quaisquer comentários que tenham sido adicionados ao seu GML Visual scripts. Essas opções são:

  • Cor: A cor padrão para o texto sendo escrito na janela (padrão ARGB #FF53E270).
  • Nome da fonte: O nome do font a usar (disponível fonts dependerá dos instalados em seu sistema).
  • Estilo: O estilo font a ser usado, como negrito ou itálico (note que nem todas as janelas permitirão os mesmos estilos e também dependerá do font usado).
  • Tamanho: O tamanho para definir o font para a janela de saída dada.
  • Anti-Aliasing: Aqui você pode configurar o modo anti-aliasing para as janelas de saída font renderização. Você pode escolher deixar o IDE decidir selecionando Usar Padrão, ou você pode forçá-lo a Ligar ou Desligar selecionando uma dessas opções.
