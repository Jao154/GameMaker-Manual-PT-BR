Drag And Drop™ Preferências

As preferências Drag e Drop™ são usadas para definir a aparência do editor Drag e Drop™ script. As diferentes opções disponíveis estão listadas abaixo, juntamente com uma descrição do que cada uma faz.

  • DnD™ localização padrão da caixa de ferramentas: Por padrão a caixa de ferramentas DnD™ (onde todas as diferentes bibliotecas de ação são mantidas) é colocada à direita do Editor, mas você pode usar esta opção para tê-la por padrão à esquerda.
  • DnD™ modo de visibilidade geral do nó: Uma "visão geral" do nó é o pequeno texto descritivo que aparece ao lado de certas ações na Ação Workspace, por exemplo, esta ação de Instância de Destruição:

    Você pode optar por ativá-las sempre, habilitá-las somente se a ação for colapsada (a configuração padrão), ou desativá-los completamente.
  • Resposta automática a DnD™ conversão para a linguagem GameMaker: Quando você converte um evento Drag and Drop ou script em código GML, você receberá por padrão uma mensagem avisando sobre o que está prestes a fazer e perguntando se deseja continuar. Configurando esta opção para "OK", você simplesmente aprovará a mudança sem mostrar a mensagem.
  • Resposta automática à eliminação do nó DnD™: Ao excluir um nó de evento Drag and Drop, você receberá, por padrão, uma mensagem avisando sobre o que você está prestes a fazer e perguntando se deseja continuar. Configurando esta opção para "OK". simplesmente aprovará a eliminação sem mostrar a mensagem.
  • Tamanho máximo para as miniaturas de ação da caixa de ferramentas DnD™: Na parte inferior da caixa de ferramentas de ação você tem um controle deslizante para mudar o tamanho dos ícones Drag and Drop. Esta configuração será usada para definir o tamanho máximo permitido (padrão 64px).
  • Tamanho mínimo para as miniaturas de ação da caixa de ferramentas DnD™: Na parte inferior da caixa de ferramentas de ação você tem um controle deslizante para mudar o tamanho dos ícones Drag and Drop. Esta configuração será usada para definir o tamanho mínimo permitido (padrão 16px).
  • Tamanho padrão para as miniaturas de ação da caixa de ferramentas DnD™: Esta configuração será usada para definir o tamanho padrão para todos os ícones de ação na caixa de ferramentas (padrão 32px).
  • Número máximo de ações recentemente utilizadas para exibição: Na caixa de ferramentas de ação há uma biblioteca no topo que contém as ações que você utilizou mais recentemente. Você pode usar esta opção para alterar o número máximo de ações recentes que será mostrado (padrão 5).
  • Espessura da cadeia que conecta DnD™ ações: Isto define a espessura do gráfico da cadeia que liga as diferentes ações em um bloco de código DnD (padrão 3px).
  • Tempo de mouse para começar a arrastar os nós (ms): Ao clicar em um nó e arrastá-lo, você estará reposicionando o nó dentro do código de ação. Entretanto, se você segurar o mouse por um momento e depois se mover, você estará movendo o posição do nó dentro do workspace, mas não dentro da ordem do código. Esta configuração define o número de milissegundos a ter que esperar com o botão do mouse pressionado antes que o nó possa ser movido (e não posicionado novamente). O valor padrão é 350.
  • Ocultar DnD™ painel de visão geral do nó: Aqui você pode optar por ocultar o painel de visão geral do Editor de Código de Ação. Por padrão, isto é desativado e, se for habilitado, removerá do editor a lista de cadeias de ação de texto simples.

Há também um sub-menu para comentários em DnD™:

As preferências dos Comentários controlam as diferentes opções de cor e formatação para quaisquer comentários que tenham foi adicionado ao seu Drag e Drop™ ação scripts. Estes optons são:

  • Cor: A cor padrão para o texto sendo escrito na janela (padrão ARGB #FF53E270).
  • Nome da fonte: O nome do font a ser usado (disponível fonts dependerá daqueles instalados em seu sistema).
  • Estilo: O estilo font a ser usado, como negrito ou itálico (note que nem todas as janelas permitirão os mesmos estilos e também dependerá do font usado).
  • Tamanho: O tamanho para definir o font para a janela de saída dada.
  • Anti-Aliasing: Aqui você pode definir o modo anti-aliasing para as janelas de saída font renderização. Você pode escolher deixar o IDE decidir selecionando Use Default, ou você pode forçá-lo a ligar ou desligar , selecionando uma dessas opções.
