Web e HTML5

O GameMaker oferece várias funções para interagir com a Internet ou com um server, e podem ser usados em todas as plataformas alvo disponíveis. Estas funções são:

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_request
  • http_get
  • http_get_file
  • http_post_string

Além das funções gerais listadas acima, o alvo HTML5 tem algumas funções adicionais na web que são especificamente projetadas para serem usadas pelos jogos que o visam:

  • browser_height
  • browser_width
  • browser_input_capture
  • webgl_enabled
  • clickable_exists
  • clickable_add
  • clickable_add_ext
  • clickable_change
  • clickable_change_ext
  • clickable_set_style
  • clickable_delete
  • analytics_event
  • analytics_event_ext
  • os_browser
  • http_get_request_crossorigin
  • http_set_request_crossorigin

Além destas funções para o módulo de destino HTML5, há também uma variável especial JavaScript que pode ser usada naquelas raras ocasiões em que o conteúdo da pasta html5game é armazenado em um lugar completamente diferente (URL). Normalmente este path é codificado no arquivo JavaScript e é relativo ao local de onde o JavaScript é chamado, mas você pode definir a variável g_GameMakerHTML5Dir antes da chamada para o jogo script e o jogo será executado a partir do local recém-definido. Por exemplo, o jogo será executado a partir do novo local definido:

<script>var g_GameMakerHTML5Dir = "http://mygamealternatedirectory.com/";</script>

Também vale a pena notar que você pode flag script funções usando o prefixo gmcallback_ o que significa que elas não serão ofuscadas quando você compilar o projeto para HTML5. Isto é muito uisqueiro ao usar as funções clickable_* listadas acima (para mais informações, por favor, veja aqui).
