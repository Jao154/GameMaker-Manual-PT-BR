gpu_set_cullmode

Esta função pode ser usada para definir o modo de abate de retaguarda. Um polígono (primitivo) tem um lado frontal e um lado posterior. Diz-se que o lado da frente é o lado onde os vértices são definidos em ordem anti-horária. Normalmente ambos os lados são desenhados, mas se você fizer uma forma fechada (como uma pirâmide) então isto é um desperdício de poder de processamento porque o lado de trás do triângulo nunca pode ser visto. É nestes casos que você pode switch sobre o corte da face traseira, pois isso economiza cerca da metade do tempo de desenho, embora isso signifique que você tem a tarefa de definir seus polígonos da maneira correta para evitar problemas.

O valor do modo de abate utilizado deve ser uma das seguintes constantes (o valor padrão é cull_noculling):

Constante Descrição
cull_noculling Nenhum abate será feito
cull_clockwise Todos os triângulos no sentido horário serão abatidos
cull_counterclockwise Todos os triângulos no sentido anti-horário serão abatidos

Sintaxe:

gpu_set_cullmode(cullmode);

Argumento Tipo Descrição
cullmode Constante do modo de abate The cull mode to use (see list above)

Devoluções:

N/A

Exemplo:

if gpu_get_cullmode() != cull_clockwise
{
    gpu_set_cullmode(cull_clockwise);
}

O código acima obtém o modo de abate atual e, se não for cull_clockwise, é ajustado para essa constante.
