Colisões

Ao planejar moções ou decidir sobre certas ações, muitas vezes é importante ver se há colisões com outros objects em certos lugares do mundo do jogo, e muitas vezes escolher a colisão certa para o trabalho é a tarefa mais importante de todas. O GameMaker tem várias funções incorporadas para ajudá-lo a lidar com as colisões corretamente e de acordo com as necessidades de seu projeto.

NOTA As funções de colisão usavam valores inteiros nas versões anteriores de GameMaker, porém agora usam valores de ponto flutuante, o que significa que as funções de colisão não mais arredondarão argumentos não-inteiros. Para mais informações, por favor, leia Modo de Compatibilidade de Colisão.

As seguintes funções tratam das várias maneiras pelas quais você pode ter uma instância de verificação de colisões com outra instância:

  • place_empty
  • place_free
  • place_meeting
  • position_empty
  • position_meeting
  • position_change
  • position_destroy
  • instance_place
  • instance_place_list
  • instance_position
  • instance_position_list

Verificação avançada de colisão

As seguintes rotinas podem ser usadas para isto (assim como outras funções detalhadas nas seções relativas a Movimentação e Instâncias). Estas funções de colisão funcionam todas de maneira ligeiramente diferente, mas mantêm três argumentos comuns que explicaremos aqui:

  • obj: Aqui é onde especificamos o object com o qual a função tem que verificar uma colisão. Pode ser uma identificação de instância, um tipo object, ou a palavra-chave especial GM all. Vale notar também que se o tipo object escolhido for o pai designado de outro objects, então todos os filhos desse object também serão incluídos nessa verificação.
  • pré: Isto pode ser true ou false e está relacionado com a precisão da verificação de colisão. Se ajustado para true, então as colisões precisas devem ser calculadas (mas somente se o object a ser verificado tiver marcado "preciso" para sua detecção de colisão em suas propriedades de máscara), o que é uma tarefa um pouco mais complexa para a CPU e, portanto, deve ser usado com cautela. Se false, então as colisões serão calculadas com base na caixa de delimitação do object a ser verificado.
  • não me: Você pode precisar verificar se há colisões com um object que tem o mesmo índice que o object que executa o código, então para evitar que o código encontre uma colisão com ele mesmo, você pode configurar isso para true. Se você estiver procurando especificamente o exemplo do object que executa o código, ou não é importante, ele pode ser deixado como false.

As funções básicas retornarão ou a identificação de uma instância encontrada em colisão, ou a palavra-chave especial noone quando não houver colisão, enquanto as funções de lista retornarão o número de instâncias que estão em colisão e preencherão uma lista DS pré-fabricada com suas identificações. Observe que se houver múltiplas colisões com as áreas definidas por estas funções e instâncias do dado object, apenas um ID de instância é retornado, e pode ser qualquer uma das instâncias na colisão.

Existem as seguintes funções que lidam com colisões avançadas.

  • collision_circle
  • collision_circle_list
  • collision_ellipse
  • collision_ellipse_list
  • collision_line
  • collision_line_list
  • collision_point
  • collision_point_list
  • collision_rectangle
  • collision_rectangle_list

Verificação de colisão sem máscara

Todas as funções acima estão relacionadas às instâncias de verificação de colisão e, como tal, dependem da máscara de colisão que é definida para a instância. Entretanto, há muitos momentos em que você precisa verificar "colisões" com um ponto ou uma área, especialmente quando sua instância não tem um sprite atribuído, ou quando você está trabalhando com o mouse, etc... Portanto, o GameMaker também fornece as seguintes funções para ajudá-lo nestas situações:

  • point_in_rectangle
  • point_in_triangle
  • point_in_circle
  • rectangle_in_rectangle
  • rectangle_in_triangle
  • rectangle_in_circle

Colisões Físicas

Ao utilizar a física embutida, as funções de colisão acima não têm garantia de funcionar para instâncias habilitadas pela física. Isto se deve principalmente ao fato de que estas instâncias não usam mais a maioria das variáveis regulares embutidas (em vez disso, as instâncias habilitadas para física têm seu próprio conjunto de variáveis) e também não usam a máscara de colisão ou a caixa de delimitação, já que usam dispositivos em seu lugar. Entretanto, existe uma função especial para testar colisões com a física ativada que verifica se há sobreposição de fixações:

  • physics_test_overlap
