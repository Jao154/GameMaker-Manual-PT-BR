mp_potencial_caminho

Esta função computa um path por exemplo de sua posição atual e orientação para a posição especificada pelos argumentos xgoal,ygoal. Ela usa o tamanho do passo indicado enquanto tenta evitar a colisão com todas as instâncias ou apenas aqueles assinalados como sólidos. Como na função mp_potential_step() utiliza etapas potenciais de campo para realizar isso e a maneira pela qual o faz pode ser modificada através da função mp_potential_settings(). O indicado path já deve existir e será sobrescrito pelo novo path. A função retornará se um path foi encontrado (verdadeiro) ou não (falso).

Para evitar que a função continue a ser calculada para sempre, é necessário fornecer um fator maior que 1 - a função irá parar e relatar falha se não encontrar um path menor que este fator vezes a distância entre o início e a meta. Um fator de 4 é normalmente bom o suficiente, mas se você espera desvios longos, você pode torná-lo mais longo. Se falhar, ainda é criado um path que corre na direção aproximada do objetivo, mas não o alcançará.

NOTA: Esta função não move a instância. Ela define apenas um path, e você deve usar as funções de Caminho para se movimentar.

Sintaxe:

mp_potential_path(path, xgoal, ygoal, stepsize, factor, checkall)

ArgumentoTipo Descrição
path index of the path that is to be used by the function
xgoal The target x position.
ygoal The target y position.
stepsize The speed the instance moves in pixels per step.
factor This number is used to prevent an infinite loop. It should be larger than 1. See the function description for more information.
checkall Whether to check all instances (true) or just solid ones (false).

Devoluções:

Exemplo:

path = path_add();
mp_potential_path(path, obj_Player.x, obj_Player.y, 3, 4, 0);
path_start(path, 3, 0, 0);

O código acima cria um novo recurso path e armazena seu índice na variável "caminho". Ele então gera um path usando mp_potential_path entre o object com este código e o object "obj_Player", verificando se há colisões com todas as instâncias no room. Finalmente começa o object ao longo desse path, mesmo que a função não tenha conseguido devolver uma rota completa para o "obj_Player".
