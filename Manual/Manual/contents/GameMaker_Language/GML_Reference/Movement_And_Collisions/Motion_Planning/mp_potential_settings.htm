mp_potencial_settings

As funções mp_potential_ fazem seu trabalho usando uma série de parâmetros que podem ser alterados usando esta função. Globalmente o método funciona da seguinte forma: primeiro tenta se mover diretamente em direção ao objetivo, olhando uma série de passos à frente que podem ser definidos com o parâmetro "à frente" (o valor padrão é 3). Reduzir este valor significa que a instância começará a mudar de direção mais tarde, pois não "vê" tão à frente, e aumentá-lo significa que começará a mudar de direção mais cedo. Se esta verificação leva a uma colisão, então começa a olhar mais para a direção à esquerda e à direita da direção ideal, em linha reta. Ele faz isto em etapas de tamanho "passo-a-passo" (o valor padrão é 10), e reduzindo isto dá à instância mais possibilidades de movimento, mas será mais lento, pois precisa de mais poder de processamento.

O parâmetro "maxrot" é um pouco mais difícil de explicar! A instância tem uma direção de movimento de corrente, e maxrot (valor padrão é 30) indica o quanto é permitido variar a direção de cada lado dessa direção de corrente em um passo. Portanto, mesmo que possa se mover, por exemplo, diretamente para o objetivo, só o fará se não violar esta mudança máxima de direção. Se você fizer do maxrot um grande valor, então a instância pode mudar muito de direção em cada passo, e isto tornará mais fácil encontrar um curto path mas o path será mais feio e menos natural. Se você fizer o valor menor, então o path será mais suave, mas pode levar desvios mais longos (e às vezes até falhar em encontrar o objetivo). Finalmente, quando nenhum passo pode ser dado, o comportamento depende do valor do parâmetro "no local". Se no local for verdadeiro (o valor padrão), então a instância girará no local pela quantidade indicada com "maxrot" e se for falso, não se moverá de forma alguma. Defini-lo como falso é útil, por exemplo, para carros, mas reduz a chance de a instância encontrar um path.

Sintaxe:

mp_potential_settings(maxrot, rotstep, ahead, onspot)

Argumento Tipo Descrição
maxrot Real The number of degrees either side of the current direction that the instance can rotate in a step
rotstep Real The number of degrees either side of the current direction that the instance can check for a collision
ahead Real The number of steps ahead that the instance can check for a collision. Larger values are slower than lower ones.
onspot Booleano Use this to allow the instance to rotate on the spot when no path is found (true) or not (false)

Devoluções:

N/A

Exemplo:

mp_potential_settings(45, 5, 5, 0);

O código acima irá definir as funções mp_potential_ para usar as seguintes configurações: a instância pode olhar para frente cinco passos, mudar a direção 45 graus cada passo, olhar 5 graus para cada lado da direção atual se houver um obstáculo e não girar no local de uma colisão.
