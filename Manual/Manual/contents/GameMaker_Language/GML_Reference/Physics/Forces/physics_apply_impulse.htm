física_aplicar_impulso

Você não só pode aplicar força e gravidade a um object com a física no GameMaker, mas também pode aplicar um impulso. Isto é ligeiramente diferente de uma força na medida em que quando ela é aplicada afetará imediatamente a velocidade, e, potencialmente o torque (ou "spin") do object, particularmente se o ponto escolhido para aplicar o impulso não estiver alinhado com o centro de massa (nota: o centro de massa não é necessariamente o mesmo que a origem!). Aqui está uma ilustração:

Como você pode ver, o jogador clicou na bola e isto lhe deu um impulso para se mover para a parte superior direita, girando à medida que ela vai girando. O impulso exato é definido pelo vetor que obtemos dos componentes ximpulso/yimpulso em relação às coordenadas xpos/ypos - o que significa simplesmente que o impulso é calculado como a distância de xpos/ypos a ximpulso/yimpulso em Newtons, e a direção é o ângulo que obtemos de xpos/ypos a ximpulso/yimpulso.

NOTA: O mundo da física usa medidores para todos os seus cálculos, e a proporção de pixels por metro é definida no editor room ou usando physics_world_create().

Sintaxe:

physics_apply_impulse(xpos, ypos, ximpulse, yimpulse)

ArgumentoTipo Descrição
xpos The x coordinate in the room where the impulse will be applied
ypos The y coordinate in the room where the impulse will be applied
ximpulse the x component of the impulse vector
yimpulse the y component of the impulse vector

Devoluções:

Exemplo:

if mouse_check_button_pressed(mb_left)
{
    with (place_meeting(mouse_x, mouse_y, all))
    {
        physics_apply_impulse(mouse_x, mouse_y, -10 + irandom(20), -10 + irandom(20));
    }
}

O código acima aplicará um impulso com um vetor aleatório a uma instância que está na posição do mouse quando o botão esquerdo é pressionado.
