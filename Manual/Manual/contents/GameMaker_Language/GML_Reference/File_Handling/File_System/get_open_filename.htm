get_open_filename

Esta função abre um diálogo e pede ao jogador um nome de arquivo para abrir com o filtro dado. O filtro tem a forma "name1|mask1|name2|mask2|... ", onde a máscara pode conter as diferentes opções com um ponto-e-vírgula entre elas e você também pode usar um "*" para significar qualquer string. Por exemplo: "bitmaps|*.bmp;*.wmf", pediria ao usuário que selecionasse o nome de um arquivo bitmap do sistema, mas somente aqueles que têm os dois especificados extensions (bmp e png). esta função não abre o arquivo em si! Ela só retorna um string com o nome completo e path do arquivo. Se o usuário pressionar "Cancelar" um string vazio "" é retornado.

Uma coisa importante a se notar ao usar esta função é que ela concede a você certas permissões para aquele arquivo, pela duração de seu jogo. Assim, uma vez que você obtenha o arquivo path do usuário, você poderá acessá-lo repetidamente sem ter que pedir. Entretanto, como esta função é para leitura de um arquivo, as permissões concedidas são apenas para leitura e economia não serão permitidas pelo sistema operacional. Caso você precise ler e escrever no arquivo, você pode obter a permissão usando a função get_save_filename().

NOTA: Esta função só é válida nas metas Windows e macOS.

Sintaxe:

get_open_filename(filter, fname);

ArgumentoTipo Descrição
filter The file type filter and/or name.
fname The suggested filename to use.

Devoluções:

Exemplo:

var file;
file = get_open_filename("text file|*.txt", "");
if file != ""
{
    file_text_open_read(file);
}

O código acima pedirá ao usuário que selecione um arquivo de texto para abrir e então verifique o nome do arquivo retornado e path, e se ele existir abrirá o arquivo para leitura.
