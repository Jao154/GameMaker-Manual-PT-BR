rollback_sync_on_frame

Esta função verifica se todos os jogadores estão sincronizados no quadro atual, o que significa que seus estados não se baseiam em previsões, mas nos dados reais recebidos deles. Em tal caso, ela retorna verdadeiro.

Se os jogadores não estiverem sincronizados no quadro atual, ele retorna falso, e tenta sincronizar todos os jogadores.

A tentativa de sincronizar os jogadores com esta função pode causar um congelamento. Você deve garantir que esta função não seja chamada continuamente em um evento Step.

Em vez disso, use-o como uma chave múltipla. Veja o exemplo no final da página.

A variável rollback_confirmed_frame é atualizada automaticamente quando todos os jogadores estão sincronizados.

Sintaxe:

rollback_sync_on_frame();

Devoluções:

Boolean

Exemplo:

if (obj_game.winning_player == -1 && points >= obj_game.points_required_for_win)
{
    if (rollback_sync_on_frame())
    {
        obj_game.winning_player = player_id;
    }
}

Este código é executado no evento Step de um objeto jogador.

Ele verifica se não há nenhum jogador vencedor, e se o jogador atual tem os pontos necessários para vencer.

Se ambos forem verdadeiros, isso significa que o jogador deve vencer. Entretanto, antes de fazer isso, ele verifica se todos os jogadores estão sincronizados, para garantir que o estado esteja correto.

Se todos os jogadores estiverem sincronizados, ele define a variável winning_player. Isto faz com que a primeira condição se torne falsa, garantindo que o rollback_sync_on_frame() não seja chamado novamente.

Isto significa que a função é usada como um alternador, em vez de correr continuamente cada quadro.
