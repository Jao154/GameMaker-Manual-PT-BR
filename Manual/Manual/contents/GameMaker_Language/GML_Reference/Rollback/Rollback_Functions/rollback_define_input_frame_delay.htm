rollback_define_input_frame_delay

Esta função define o atraso de frame fixo para as entradas do player local.

Por padrão, seu jogo utilizará um atraso de entrada adaptável, que é baseado na latência entre os jogadores. Isto funcionará para quase todos os jogos lá fora, entretanto, alguns tipos específicos de jogos (por exemplo, jogos de luta) podem preferir usar um atraso de entrada fixo, ao invés de um atraso adaptativo que pode mudar a qualquer momento.

Não é recomendado mudar isso a menos que você saiba exatamente o que está fazendo. Você pode chamar esta função a qualquer momento antes e durante um jogo multiplayer.

O valor do atraso especificado deve estar em número de quadros. Se você especificar -1, o jogo voltará a usar um atraso adaptativo. O atraso máximo que você pode definir é de 10 quadros.

Sintaxe:

rollback_define_input_frame_delay(frames);

Argumento Tipo Descrição
frames Real The number of frames to set as the fixed input delay, or -1 para redefini-lo para o padrão.

Devoluções:

N/A

Exemplo:

// 'Options' script
global.frame_delay = 5;

// obj_game: Create event
rollback_define_input_frame_delay(global.frame_delay);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_start_game(2, false);
}

Você pode definir seu valor de atraso de quadro como uma variável global, e usar isso ao chamar rollback_define_input_frame_delay() antes que um jogo de rollback seja iniciado.
