Erros de Rollback

Ao trabalhar com o sistema Rollback, há certos erros que podem ocorrer. Estes erros são erros de corredor e, portanto, são mostrados enquanto o jogo está em andamento:

A tabela a seguir lista os possíveis erros que você pode obter ao trabalhar com o sistema Rollback e uma descrição. Ela também fornece uma solução para cada mensagem de erro específica.

Erro Descrição
Não foi possível conseguir que a camada criasse jogadores. Você deve passar o nome da camada para a função rollback_define_player ou criar a camada Instances. Se o player object foi definido com a função rollback_define_player, o sistema de rollback tentará criar o player object na camada, da qual o nome pode opcionalmente ser passado como um segundo parâmetro (ou na camada com o nome "Instâncias" se não estiver definido). Se a camada não puder ser encontrada por seu nome, este erro é disparado. A solução é ou definir explicitamente o nome da camada para criar os jogadores ou criar uma camada com o nome "Instâncias" se o nome da camada não for definido explicitamente.
O jogador objects deve ser gerenciado ao executar jogos multiplayer. Se o player object foi definido com a função rollback_define_player e o player object não está sinalizado como gerenciado este erro é disparado. A solução é marcar o player object como gerenciado no Editor de Objetos.
Falha em obter informações de usuário para o player_id N. Se o sistema de rollback não foi capaz de obter informações do server sobre o jogador, este erro é disparado. Verifique o console do desenvolvedor para ver se algum pedido foi bloqueado.
Não era possível chamar o rollback_chat antes de estar totalmente conectado ao jogo. Tente esperar pelo evento rollback_game_info. A função de bate-papo não está disponível até que o jogador tenha aderido ao room. Solução: Desabilitar o bate-papo até o evento rollback_game_info recebido (Veja Eventos Rollback).
rollback_use_manual_start deve ser chamado antes de rollback_create_game e rollback_join_game. Estes erros são disparados se uma chamada de função violar as restrições internas da máquina do estado. A mensagem de erro sugere uma solução.
rollback_use_player_prefs devem ser chamados antes de rollback_create_game e rollback_join_game.
rollback_use_player_prefs devem ser chamados antes do rollback_set_player_prefs.
rollback_set_player_prefs tem que ser chamado antes do jogo rollback_start_game.
rollback_define_player deve ser chamado antes de rollback_create_game e rollback_join_game.
rollback_define_input deve ser chamado antes de rollback_create_game e rollback_join_game.
rollback_define_mock_input deve ser chamado antes de rollback_create_game e rollback_join_game.
Não leia as informações do jogador antes do jogo ter começado. Chamada para rollback_get_info().
Não leia a entrada do jogador antes do jogo ter começado. Chamada para rollback_get_input().
rollback_define_input_frame_delay deve ser chamado antes de rollback_create_game e rollback_join_game
rollback_define_input_frame_delay deve ser chamado antes de rollback_create_game e rollback_join_game.
rollback_set_player_prefs falhou em definir as preferências O server rejeitou os prefs do jogador. Normalmente isto acontece nos seguintes casos: 1. Um usuário tenta reiniciar os prefs do jogador enquanto o outro usuário já iniciou o jogo. Neste caso o jogo deve usar o último prefs definido com sucesso, garantimos que cada jogador deve definir alguns prefs antes que o jogo possa ser iniciado. 2. O tamanho dos prefs seriados excede o tamanho máximo dos prefs do jogador (atualmente 800 bytes).
rollback_create_game, número de jogadores definido para N mas deve estar na faixa de 1 a 4 A função rollback_create_game foi chamada com um número errado de jogadores definido. A solução é fixar o primeiro parâmetro de rollback_create_game para estar na faixa de 1 a 4.
rollback_create_game não conseguiu definir as preferências padrão do jogador. Veja a descrição para "rollback_set_player_prefs falhou em definir as preferências".
O rollback multiplayer só é suportado no alvo operagx. A função rollback_create_game é chamada e a meta não está definida para OperaGX. Atualmente, o multiplayer é suportado apenas na plataforma OperaGX. A solução é definir o alvo para o OperaGX.
Falha em criar um jogo (tipo de sessão desconhecida). Este é um erro interno do sistema de retrocesso. Relatar um problema para a equipe GameMaker.
Não foi possível encontrar player_id neste caso, você precisa passá-lo para rollback_get_input(). A funçãorollback_get_input pode aceitar a identificação do jogador como parâmetro. Se nenhum parâmetro for fornecido para a função, será necessário o campo player_id de um player object para obter entradas. Se o parâmetro player_id não for definido, este erro é disparado. Se o player object foi criado com rollback_define_player, ele deve ter player_id definido. Solução: 1. Passar explicitamente o parâmetro player_id para a função 2. Criar player com rollback_define_player.
O Info player_id N está fora de alcance. rollback_get_info é chamado com player_id errado (ou player object tem valor errado no campo player_id). Solução: Fixar o parâmetro player_id para estar na faixa de 1 a 4.
Não foi possível encontrar a instância <id> de object <id> (<nome>) referenciada nos dados de rollback. Erro interno do sistema de retrocesso. Relatar um problemapara a equipe GameMaker.
Nenhuma idéia de como criar OBJECT_KIND=<kind>.
RefID <id> não tem object (entrada malformada?).
Inválido ReferenceKind <id> (entrada malformada?).
O mapeamento de instâncias está habilitado, mas tentamos deserializar um índice fora do intervalo. <id> não está em [0, <id>].
Valor de RV inválido na deserialização.
Tentando ressuscitar uma instância que foi apagada!
Não se pode serrar RValue com kind=<id> (<nome>).
Definição de entrada vazia. rollback_define_input foi chamado com uma entrada vazia.
A definição de entrada contém constante <código chave> duas vezes, o que não é permitido. Um código chave só pode ser mapeado para uma única entrada. Verifique se algum código chave é definido mais de uma vez na definição da entrada.
A definição de entrada com etiqueta <label> não é um valor real ou uma matriz de reais. Todos os valores que descrevem os valores de entrada devem ter o tipo Real.
Definição de entrada falsa vazia para o player_id N. Ver "Definição de entrada vazia".
A definição simulada contém um valor não real. Ver "Definição de entrada com etiqueta <label> não é um valor real ou uma matriz de reais".
