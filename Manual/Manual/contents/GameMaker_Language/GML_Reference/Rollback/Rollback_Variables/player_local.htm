player_local

Quando você usa rollback_define_player() para definir um jogador object, o sistema cria automaticamente uma instância do mesmo object para cada jogador. Cada instância daquele object recebe a variável player_local, que informa se aquela instância do jogador é local ou não.

Se você não usar rollback_define_player(), então esta variável não será criada para nenhuma objects.

Você pode usar esta variável para mostrar qual jogador é local. Não é recomendado usar esta variável de forma que afete sua lógica de jogo, que deve ser a mesma para jogadores locais e remotos.

Sintaxe:

player_local;

Devoluções:

Boolean

Exemplo:

// Draw event
if (player_local)
{
    draw_sprite(spr_arrow, 0, x, y - 60);
}

O código acima é aplicado no evento do sorteio, e verifica se a instância do jogador é local. Nesse caso, ele desenha uma seta sprite acima da instância, dizendo ao jogador qual instância eles estão controlando.
