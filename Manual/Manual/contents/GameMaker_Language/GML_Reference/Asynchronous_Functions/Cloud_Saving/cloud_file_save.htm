cloud_file_save

Esta função irá comprometer um arquivo para o serviço de nuvem escolhido para armazenamento. A função retornará um valor de identificação único que deverá ser usado no evento assíncrono apropriado para identificar o mapa DS que é retornado como uma "chamada de retorno" do serviço de nuvem. O arquivo deve conter todas as informações que você precisa salvar para seu jogo, pois você só pode armazenar um único "blob de dados" para a nuvem, e executar esta função novamente irá sobrescrever quaisquer valores previamente armazenados (assim como usar a função cloud_string_save() função). A descrição deve ser uma breve string de informações que descrevam a gravação, por exemplo: "Level2, Stage2".

Para maiores informações sobre os dados assíncronos retornados, por favor, veja a função cloud_synchronise().

Sintaxe:

cloud_file_save(file, description);

ArgumentoTipo Descrição
string The file to be uploaded (as a string).
description A brief description of the data being stored.

Devoluções:

Async Request ID

Exemplo:

var t_str = "";
for (var i = 0; i < 10; i++;)
{
    t_str += string(global.Highscore[i]) + "|"
}
var file = file_text_open_write("Highscores.txt");
file_text_write_string(file, t_str);
file_text_close(file);
save_check = cloud_file_save("Highscores.txt", "Current Highscores");

O código acima cria um string a partir dos valores armazenados na matriz global "Highscores" e depois escreve este string em um arquivo para armazenamento local. O arquivo é então enviado para o serviço de nuvem para armazenamento.
