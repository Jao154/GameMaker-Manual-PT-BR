font_add_sprite_ext

Com esta função, você pode usar uma "faixa sprite" (o próprio sprite deve ser um sprite asset do navegador Asset, ou um sprite que você adicionou aos recursos do jogo usando sprite_add) para criar um novo font asset, onde cada subimagem seria um símbolo ou letra individual. Ao contrário do normal, a font_add_sprite() que tem uma ordem específica para as sub-imagens do sprite, esta função mapeará as sub-imagens do sprite com base no argumento "string_map" da função. Este argumento é um string que você pode usar para dizer ao GameMaker qual é a ordem das subimagens do sprite font e ele as mapeará automaticamente ao escrever o texto. Assim, por exemplo, se você tiver um "string-map" de "AaBbCcDdEeFfGgHh", seu sprite font deve ter as subimagens ordenadas desta forma. Você pode definir "espaço" como sendo qualquer caractere que você queira, por exemplo, uma única linha do tamanho que você quer que o espaço seja, e todos os espaços no texto serão renderizados nessa largura (mas a imagem escolhida nunca será renderizada), ou se você não fornecer um sprite para espaço, então a largura do caractere mais largo no sprite font será usada em seu lugar.

Você também pode definir o espaçamento para que o font seja proporcional (true) ou não (false), onde um font proporcional é espaçado com base na largura real das letras (assim a letra "i" leva menos room do que a letra "w", por exemplo), enquanto um font não proporcional será espaçado com base na largura da subimagem, como um mono espaçamento font. Finalmente, você pode definir o espaço a ser deixado entre cada letra ao escrever, e este pode ser qualquer valor inteiro, sendo que 0 não tem espaço (as letras "tocarão" se forem proporcionais).

A função retorna um valor de índice que deve ser armazenado em uma variável, pois isso será necessário em todos os outros códigos que se referem a este font, ou retornará -1 se a função falhar por qualquer razão.

Vale a pena notar que para as funções de alinhamento font funcionar (como draw_set_halign()) então o font sprite deve ter sua origem definida para o canto superior esquerdo. Se você usar outros valores, então você precisará levar em consideração a origem offset ao desenhar o texto usando o font.

NOTA Quando você carrega um font no GameMaker, você deve se lembrar de removê-lo novamente (com font_delete()) quando não for mais necessário, caso contrário existe o risco de um vazamento de memória que irá retardar e, eventualmente, quebrar seu jogo.

Sintaxe:

font_add_sprite_ext(spr, string_map, prop, sep);

Argumento Tipo Descrição
spr Ativo Sprite The sprite to add a font based on.
string_map Cordão String from which sprite sub-image order is taken.
prop Booleano Set as proportional font or not.
sep Real The space to leave between each letter.

Devoluções:

Font Asset

Exemplo:

global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2);

O código acima criará um novo font asset do sprite indexado na variável "spr_CalcFont" e armazenará o índice do novo font na variável "global.Font". As sub-imagens do sprite font serão mapeadas para o string especificado.
