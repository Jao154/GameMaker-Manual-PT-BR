exemplo_destroy

Você chama esta função sempre que desejar "destruir" uma instância, normalmente desencadeando um Evento de Destruição e também um Evento de Limpeza. Isto o removerá do room até que o room seja reiniciado (a menos que o room seja persistente). Chamar a função sem argumentos simplesmente destruirá a instância que está atualmente em escopo e executando o código, mas você pode fornecer um argumento opcional "id" e direcionar uma instância específica usando o valor da instância id, ou você pode direcionar todas as instâncias de um determinado object usando um object_index. Por exemplo:

instance_destroy(other);      // destroy the "other" instance in a Collision Event
instance_destroy(obj_Bullet); // destroy ALL instances of the object "obj_Bullet"

O segundo opcional flag permite que você "desligue" o Evento Destrutivo, por exemplo, sendo destruído. Por padrão o Evento Destroy será sempre realizado, mas se você definir este flag para false, então você pode fazer com que a instância se destrua e pular a realização desse evento.

IMPORTANTE! Isto irá pular o Evento Destroy, mas o evento Clean Up ainda será chamado.

Vale notar que quando se destrói uma instância, seu evento de destruição é chamado imediatamente após o código ou ação que chama a função de destruição. Além disso, embora o evento de destruição seja realizado, a instância não é imediatamente retirada do jogo, e continuará a executar o código contido no evento atual. Somente quando o evento atual terminar, ela será removida do jogo.

Portanto, se você tem, por exemplo, este código:

if (hp <= 0) instance_destroy();
score += 10;

A variável "pontuação" será incrementada mesmo que a função instance_destroy tenha sido chamada, e a instância será finalmente removida de seu jogo no final do evento. Tenha cuidado com isso, como se você tivesse (por exemplo) criado um recurso dinâmico para a instância, como uma estrutura de dados, e depois tivesse destruído esse recurso no evento de destruição, se houver alguma referência a ele após a função ou ação de destruição ter sido realizada, então você receberá um erro de "recurso desconhecido", pois o evento de destruição o removeu do jogo.

Sintaxe:

instance_destroy([id, execute_event_flag]);

Argumento Tipo Descrição
id Identificação da instância ou do bem objeto OPTIONAL A instância ID para destruir, ou o object asset para destruir instâncias de (opcional, padrão é a instância de chamada)
execute_event_flag Booleano OPTIONAL Configurar para true ou false para realizar ou não o evento Destroy (opcional, o padrão é true)

Devoluções:

N/A

Exemplo:

if (bbox_right < 0 || bbox_left > room_width || bbox_bottom < 0 || bbox_top > room_height)
{
    instance_destroy(id, false);
}

O código acima destruirá a instância executando o código sem chamar o Evento de Destruição se a instância estiver fora dos limites do room.
