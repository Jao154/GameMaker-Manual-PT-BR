esqueleto_mix_animação

Você pode switch animações facilmente usando o skeleton_animation_set() mas isso pode causar um salto ou gagueira, já que uma animação é trocada por outra. Para evitar isso, você pode definir o valor da mistura entre dois conjuntos de animação e o sprite irá interpolar entre eles. Normalmente, você gostaria de fazer isso no Create Event da instância com a animação esquelética, pois ela só precisa ser definida uma vez, e o GameMaker interpolará todas as outras mudanças no sprite usando os conjuntos de animação naquela instância. Note que o valor de duração é de 0 a 1, onde um valor de 0,5 teria uma interpolação "metade e metade" de um conjunto para o outro.

NOTA A espinha 4.0 muda como a animação rotacional funciona, já que não interpola mais a rotação usando a rota mais curta possível, mas usa os valores absolutos em seu keyframes para interpolação. Isso pode causar um comportamento inesperado se você estiver atualizando sua animação Spine a partir de uma versão mais antiga, e mudanças dentro de seu projeto Spine podem ser necessárias para que as rotações se comportem conforme necessário.

Sintaxe:

skeleton_animation_mix(animfrom, animto, duration);

Argumento Tipo Descrição
animfrom Cordão The name (a string) of the first animation set to interpolate from.
animto Cordão The name (a string) of the second animation set to interpolate to.
duration Real The duration of the interpolation (from 0 to 1)

Devoluções:

N/A

Exemplo:

skeleton_animation_set("walk");
skeleton_animation_mix("walk", "jump", 0.2);
skeleton_animation_mix("jump", "walk", 0.4);

O código acima iria no Create Event of an instance com uma animação esquelética sprite e define a duração da mistura de animação para interpolar entre os dois conjuntos de animações "walk" e "jump".
