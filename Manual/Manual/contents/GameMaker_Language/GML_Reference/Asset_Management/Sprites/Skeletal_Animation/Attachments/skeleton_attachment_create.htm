esqueleto_criar_aplicação

Quando você cria sua animação esquelética sprite, você pode atribuir ranhuras de fixação e anexos para ir nelas. Estas são simplesmente imagens que estão separadas da animação, mas quando anexadas se moverão junto com o osso ao qual estão anexadas. Normalmente você atribuiria estes anexos em seu programa de animação (Spine), mas com esta função você pode criar seus próprios em tempo de execução usando um sprite asset de seu jogo. a função requer que você dê um nome ao anexo (como um string) e então configure o sprite_index e image_index para usar, assim como a origem x e y (que pode ser diferente daquela definida pelo sprite nas propriedades do sprite ), e você também pode configurar qualquer transformação que deseje que seja aplicada à imagem quando anexada. Se o anexo foi criado com sucesso, a função retornará 1 e se não foi (você forneceu um índice sprite inválido, ou a base sprite não é um Spine sprite), então ele retornará -1.

Sintaxe:

skeleton_attachment_create(name, sprite, ind, xorigin, yorigin, xscale, yscale, rot);

Argumento Tipo Descrição
name Cordão The name (as a string) of the attachment to create.
sprite Ativo Sprite The sprite_index to get the attachment image from.
ind Real The image_index to get the attachment image from.
xorigin Real The x origin for the image being used.
yorigin Real The y origin for the image being used.
xscale Real The horizontal scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
yscale Real The vertical scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
rot Real The rotation of the image. 0=normal, 90=turned 90 degrees counter-clockwise etc.

Devoluções:

Real (1 if successful, -1 if not)

Exemplo:

skeleton_attachment_create("sword", spr_Weapons, 0, 0, 80, 1, 1, 90);
skeleton_attachment_create("knife", spr_Weapons, 1, 0, 45, 1, 1, 90);
skeleton_attachment_create("crossbow", spr_Weapons, 0, 10, 30, 1, 1, 0);
skeleton_attachment_set("slot_leftHand", choose("sword", "knife", "crossbow"));

O código acima verificaria o nome do anexo atualmente atribuído para o slot chamado "slot_leftHand" e se um string vazio for devolvido, um novo sprite será anexado.
