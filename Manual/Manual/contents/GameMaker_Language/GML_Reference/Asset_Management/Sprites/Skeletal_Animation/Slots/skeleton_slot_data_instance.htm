esqueleto_dados_esqueleto_de_instância

Com esta função você pode preencher uma lista DS (pré-criada) com dados para cada um dos slots de anexo disponíveis para o Spine sprite associado à instância atual (incluindo quaisquer modificações de anexo). Estes dados são retornados no fórum de um mapa DS que contém os seguintes pares chave/valor:

  • "nome": O nome do espaço.
  • "osso": O nome do "osso".
  • "anexo": O nome do anexo se for usado ou "(nenhum)" se não houver um atribuído.

Os valores para cada chave serão strings e poderão então ser usados nas outras funções de anexação de esqueleto para estes tipos de sprite. Note que o DS Map criado não é destruído, então você precisará loop através da lista DS e destruir cada um dos mapas criados você mesmo.

Sintaxe:

skeleton_slot_data_instance(list);

Argumento Tipo Descrição
list DS List ID The ID of the DS list to populate with the DS maps.

Devoluções:

N/A

Exemplo:

var list = ds_list_create();
var open = true;
slot_name = "";
skeleton_slot_data_instance(list);
for (var i = 0; i < ds_list_size(list); i++)
{
    var map = list[| i];
    if open
    {
        if map[? "attachment"] == "(none)"
        {
            open = false;
            slot_name = map[? "name"];
        }
    }

    ds_map_destroy(map);
}

ds_list_destroy(list);

O código acima cria uma lista DS e depois a preenche com os dados do slot para o exemplo sprite. Estes dados são então analisados para extrair os mapas DS individuais com os dados do slot. Isto é então verificado para ver se há um slot vazio, e se assim for, a variável "slot_name" recebe o nome do slot vazio antes que o mapa DS seja destruído. Finalmente, destruímos a lista de DS, pois ela não é mais necessária.
