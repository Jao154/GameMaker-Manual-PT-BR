object_set_persistent

Com esta função você pode definir a persistência de um object específico. Um object persistente é "transportado" de room para room e mantém suas propriedades o tempo todo (para mais informações sobre persistência veja a seção sobre o Editor de Objetos). Definir a persistência do object significa que todas as instâncias deste object que são criadas depois que a persistência foi alterada serão criadas persistentes, enquanto as instâncias que já estão no room podem não ser afetadas.

Quando você cria uma instância com persistência habilitada em room, lhe será atribuída uma camada ou uma profundidade (dependendo de qual função você usou para criar a instância do object). Quando o room for alterado, e se o seguinte room não tiver uma camada com o mesmo nome ou profundidade que o atribuído, então uma nova camada será criada para a instância que persiste através do rooms. Se você deu uma camada nomeada quando a instância foi criada, então a nova camada também terá o mesmo nome da camada original, enquanto que se você atribuiu uma profundidade à instância, então a nova camada será nomeada "_layer_XXX", onde "XXX" é um valor hexadecimal usado para dar à camada um nome único. Estas camadas serão removidas do site room em Room End. Outra coisa a se notar sobre camadas e instâncias persistentes é que se você tiver atribuído à instância uma camada nomeada na criação, e houver outra camada no seguinte rooms com o mesmo nome, então a instância persistente será atribuída à camada com o mesmo nome , independentemente da profundidade da camada. Finalmente, se uma instância persistente se mover para um room com uma camada na mesma profundidade em que a instância foi criada, ela não será atribuída a esta camada, mas em vez disso uma nova camada será criada na mesma profundidade (seguindo a convenção de nomenclatura explicada anteriormente).

Por favor, note que esta não é uma função de instância! Você pode definir o persistente flag de instâncias individuais usando o persistent e, portanto, todas as dez instâncias foram marcadas como persistentes mesmo que o object em questão tenha o flag definido como falso no IDE, e mesmo que você mude o persistente flag do object para verdadeiro usando esta função, todas as instâncias que atualmente no room permanecerão como estavam, e somente as instâncias criadas após a chamada da função começarão com persistente marcadas como verdadeiras.

Sintaxe:

object_set_persistent(index, pers);

Argumento Tipo Descrição
index Objeto Ativo The index of the object to change.
pers Booleano Whether the object should be flagged as persistent (true) ou não (false).

Devoluções:

N/A

Exemplo:

if (room == rm_final)
{
    object_set_persistent(obj_Player, false);
}
instance_create_layer(32, 32, "Instances", obj_Player);

O código acima verifica em que room a instância está atualmente e se está na "rm_final" especificada, então define a persistência do object "obj_Player" como falsa e então cria a instância.
