Objetos

No GameMaker você tem objects e você tem instâncias. Objects são essencialmente o modelo base para uma instância, e como tal nunca estão presentes em um room diretamente... somente instâncias do object são colocadas no room. Este significa que se você quiser mudar algo para todas as instâncias que vai criar, então você pode fazê-lo mudando o objeto.

IMPORTANTE! Mudar qualquer coisa em um object não mudará nenhuma instância presente atualmente no room, somente aquelas que são criadas após a mudança.

Você também pode acessar informações sobre uma base object e usá-las para ditar comportamentos ou mudanças no jogo. Por exemplo, você pode fazer uma verificação para um ID do pai object, e no seguinte código, se for encontrado um exemplo com esse pai, você pode então verificar seu object_index para decidir o que deve ser feito.

O GameMaker oferece uma série de funções que permitem obter vários detalhes sobre um object. Nota, um object não é uma instância (uma entidade dentro do jogo), é simplesmente um recurso a partir do qual todas as instâncias são criadas. Isto significa que que os valores de retorno destas funções podem ser diferentes dos valores reais mostrados pelas instâncias no room, pois uma vez criada uma instância, ela pode ser alterada através de código e GML Ações visuais.

  • object_index
  • object_exists
  • object_get_name
  • object_get_mask
  • object_get_parent
  • object_get_persistent
  • object_get_solid
  • object_get_sprite
  • object_get_visible
  • object_get_physics
  • object_is_ancestor

Há também uma série de funções que permitem definir as propriedades de um object. Deve-se notar que quaisquer instâncias deste object que já existam no room podem não ser afetadas por estas funções, mas todas as novas instâncias deste objeto criado no room será, portanto é recomendável que você nunca altere um objects propriedades quando instâncias desse object estão presentes no atual room.

  • object_set_mask
  • object_set_persistent
  • object_set_solid
  • object_set_sprite
  • object_set_visible
