path_reverse

Com esta função você pode inverter a ordem na qual os pontos individuais path são numerados, assim, por exemplo, se o path tiver 5 pontos, o ponto 0 se tornaria o ponto 4, o ponto 1 seria o ponto 3 e o ponto 2 não seria alterado. A posição real de os pontos permanecem os mesmos, apenas a ordem em que são processados é alterada. Esta função muda o ativo real path , e assim afetará permanentemente como o path é usado por todas as instâncias no jogo a partir do momento em que o é utilizada até o final do jogo. Se isto não for o que você precisa, então você deve usar uma função como path_duplicate() para criar uma cópia do path primeiro, então chame esta função no duplicado asset (não esqueça de ligar para path_delete() no asset quando não for mais necessário).

Sintaxe:

path_reverse(index);

ArgumentoTipo Descrição
index The index of the path to change.

Devoluções:

Exemplo:

path_reverse(pth_AI);

Isto inverteria a ordem na qual todos os pontos do caminho dado são processados.
