audio_create_play_queue

Esta função prepara um buffer queue para áudio. Você passa o formato de dados (somente buffer_u8 ou buffer_s16 são suportados atualmente), a taxa de amostragem (que pode estar entre 1000hz e 48000hz) e os canais que o áudio requer de uma das constantes listadas abaixo:

Constante Descrição
audio_mono Áudio mono (canal único).
audio_stereo Áudio estéreo (canal duplo).
audio_3D áudio 3D (5.1).


Note que você deve liberar qualquer áudio queues criado com esta função quando não for mais necessário para evitar vazamentos de memória usando a função audio_free_play_queue().

Esta função desencadeará um Evento Assíncrono de Reprodução de Áudio quando o queue chegar ao fim, e neste evento será criado um mapa DS especial na variável async_load com os seguintes pares de chaves/valores:

  • "queue_id" - o queue que terminou de jogar dados
  • "buffer_id" - o buffer que não está mais sendo jogado de
  • "queue_shutdown" - isto é ajustado para 0 durante a reprodução normal e 1 quando o evento é recebido porque audio_free_play_queue() foi chamado. Quando isto for ajustado para 1, você não quer queue mais dados.

Sintaxe:

audio_create_play_queue(queueFormat, queueRate, queueChannels);

Argumento Tipo Descrição
queueFormat Constante do tipo de dados tampão The format of the buffer data to use (buffer_u8 ou buffer_s16).
queueRate Real The sample rate of the data in the buffer queue.
queueChannels Tipo de canal de áudio The channels to use from one of the constants listed above.

Devoluções:

Audio Queue ID

Exemplo:

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono);

O código acima cria um novo áudio queue e atribui sua identificação a uma variável para referência futura.
