audio_listener_velocity

Esta função pode ser usada para dar ao ouvinte efeitos Doppler e simular movimentos de áudio com base no vetor que é resolvido a partir das posições relativas x, y e z. Se o ouvinte em si não vai se mover, ou o movimento não é um movimento constante, você normalmente não precisaria definir estes valores, mas, por exemplo, se você estiver fazendo um jogo de rolagem onde o jogador tem um movimento constante de baixo para cima e os inimigos um movimento constante de cima para baixo, você definiria as velocidades do ouvinte e do emissor (para emissores você usaria audio_emitter_velocity()) para os vetores apropriados para simular o efeito Doppler correto à medida que eles passam pela instância do jogador.

NOTA se você tem vários ouvintes, você deve usar a função audio_listener_set_velocity().

A imagem abaixo mostra como isto poderia ser configurado para o jogo de exemplo dado acima:

Sintaxe:

audio_listener_velocity(vx, vy, vz);

Argumento Tipo Descrição
vx Real The x velocity component of the listener (default 0).
vy Real The y velocity component of the listener (default 0).
vz Real The z velocity component of the listener (default 0).

Devoluções:

N/A

Exemplo:

if speed > 0
{
    audio_listener_velocity(abs(hspeed), abs(vspeed), 0);
}

O código acima verifica se a velocidade da instância do jogador é superior a 0 e se é, então, usa os componentes absolutos apropriados hspeed e vspeed para definir a velocidade do ouvinte.
