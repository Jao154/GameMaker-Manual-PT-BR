camera_apply

Esta função aplicará as configurações da câmera em questão à vista atual da câmera que está sendo apresentada imediatamente. Enquanto você pode definir qualquer câmera de visualização usando as funções específicas camera_set_*, as novas configurações não serão usadas para renderização até o próximo quadro do jogo, mas com esta função você pode fazer estas mudanças imediatamente. Se você estiver usando várias visões, então a função deve ser usada no Evento de Sorteio e você pode usar a variável view_current para aplicar seletivamente a câmera em questão apenas à vista da câmera que está sendo apresentada atualmente. Note que isto NÃO altera o valor de ID da câmera, apenas as configurações para aquela câmera, e que se você usar esta função na câmera padrão (ou seja: a câmera interna que o GameMaker usa quando nenhuma câmera de visualização está ativa) as mudanças serão sobrescritas no próximo quadro do jogo pelas configurações internas para desenhar o room.

Sintaxe:

camera_apply(camera_id)

ArgumentoTipo Descrição
camera_id The unique camera ID value returned when you created the camera.

Devoluções:

Exemplo:

if view_current == 0
{
    camera_apply(cutscene_cam);
}

O código acima (em um Evento de Desenho) verifica qual visão de câmera está sendo apresentada no momento e se é visão de câmera[0] então as configurações para a câmera referenciada pela variável "cutscene_cam" são aplicadas a ela.
