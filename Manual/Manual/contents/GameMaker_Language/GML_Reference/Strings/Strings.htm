Cordas

Em algum momento, ao fazer seu jogo, você precisará usar texto. O texto nos jogos é tratado usando as funções de string (um string é apenas outra forma de dizer uma linha de texto) e o GameMaker tem um conjunto completo de funções que permitem manipular strings de muitas maneiras, incluindo a inserção de um string em outro, a cópia de strings e a capacidade de analisar strings para os dígitos ou as letras que eles contêm. Em geral, um string só pode ser criado adicionando texto dentro de aspas duplas " " e citação única strings não são aceitos, nem se pode dividir o string em várias linhas e esperar que o GameMaker o torne como se as quebras de linha fossem novas linhas (a menos que um identificador literal @ seja usado, como explicado abaixo).

Vale notar que existem certas convenções que você pode usar ao criar strings, a maioria preocupada com o uso de caracteres de escape. Estes são caracteres que são precedidos por um símbolo "\". Assim, por exemplo, se você quisesse colocar aspas dentro de um string você teria algo parecido com isto:

str = "Hello\"World\"";

O GameMaker também tem suporte completo para caracteres Unicode de quatro bytes de largura, permitindo que você decodifique e codifique caracteres Unicode nos limites superiores do padrão (incluindo - mas não limitado a - emoji). Para lidar com caracteres Unicode, você pode usar o "\" para preceder qualquer Unicode literal - dígitos de hexadecimais precedendo um "u", por exemplo "\u00e2" para "á"- onde os dígitos são o número do caracter Unicode. Ao trabalhar com Unicode desta forma, você precisa estar ciente do fato de que o GameMaker interpretará todos os dígitos seguindo o "u", portanto, se você quisesse escrever "áa", por exemplo, você deveria usar:

"\u00e2\a"

ou

"\u00e2\u61"

ou

"\u00e2" + "a"

já que o simples uso de "\u00e2a" resultaria na realidade no personagem Unicode "ส" (tornando-se essencialmente "\ue2a").

O GameMaker também pode lidar com qualquer literal hexadecimal - normalmente escrito como dígitos hexadecimais seguindo "0x", por exemplo "0xff", onde os dígitos são o número do personagem a ser utilizado. No GameMaker, estes são escritos usando "\x" e depois o valor hexadecimal. Estes e outros caracteres de escape pré-definidos estão listados na tabela abaixo:

Constante Descrição
\n Newline
\r Retorno do transporte
\b Backspace (0x08)
\f Alimentação de formulário (0x0c)
\t Aba Horizontal (0x09)
\v Aba vertical (0x0b)
\\ Contrabarra em si (0x5c)
\a Alerta (0x07)
\u[Hex Digits] Inserir caractere Unicode
\x[Hex Digits] Inserir caráter literal hexadecimal
\[Octal Digits] Inserir octal Unicode caractere

OBSERVAÇÃO: Strings suporta alimentação de formulário, aba vertical etc... mas isso não significa que a renderização sim, e ao desenhar strings esses caracteres podem ser ignorados.

Você também pode criar literais de corda literalmente precedendo o conjunto string com o caracter @:

var test = @"
Line breaks
over multiple
lines
";

O código acima renderá o string em várias linhas como se houvesse um caractere de fuga de linha incluído. Um literal literal string é semelhante ao literal anterior GameMaker versão string, mas eles também usam aspas duplas ou simples e devem ser prefixados por um símbolo @, podem ser quebrados sobre várias linhas no arquivo de código e NÃO suportam caracteres de escape, ou seja, @"Hello\World" não tentará escapar do W on World e será armazenado literalmente. Observe que ao usar string literalmente como este, você precisará quebrar o string se desejar incluir aspas como parte do string, ou seja:

var test = @"Hello " + "\"" + @"World" + "\""

Outra coisa a se notar é que o caractere Unicode 9647 (▯) é usado para substituir quaisquer glifos que você possa ter no seu designado font ao renderizá-lo no evento do sorteio. Portanto, se seu font não tiver, por exemplo, o símbolo °, então escrever 90° produzirá de fato 90▯.
