O Editor Tile Set

Um conjunto de azulejos é um recurso gráfico para desenhar níveis e outros componentes estáticos do seu jogo. Um tile set é composto de uma única imagem que é então dividida em diferentes "células" (tiles), e cada tile pode então ser colocado no editor room para criar uma imagem completa. Abaixo você pode ver dois exemplos sprites que podem ser usados como conjuntos de azulejos:

Imagens de Lost Garden

Estas são as formas mais comuns que as pessoas criam tile sets - ou todas encravadas juntas ou com pequenos espaços que as separam, e o GameMaker pode lidar com ambos os tipos. Essencialmente todos tile sets devem ser compostos de imagens que cabem dentro de uma grade claramente definida, sendo que cada célula da grade tem exatamente o mesmo tamanho (note que enquanto as imagens mostradas acima são todas feitas do quadrado tiles, um tile set pode ser feito também do retangular tiles ).

Ao criar seu tile sets, tenha em mente que a célula superior esquerda da grade deve estar sempre vazia, pois esta é a tile que o GameMaker usará para "esvaziar" tiles em um room e para apagar tiles existente (e mesmo se você tiver pixels nessa parte da imagem eles serão ignorados).

Quando você criar um recurso tile set pela primeira vez, a janela do editor Tile Set será aberta com as seguintes opções:

Nome

Aqui você pode dar um nome ao seu tile set. Os nomes só podem ser alfanuméricos e só podem usar o símbolo abaixo da barra "_", e não podem começar com um número.

Sprite

Aqui você pode selecionar o sprite asset a partir do qual poderá criar nosso tile set. Clicando no botão principal abrirá o Asset Explorer onde você pode escolher o sprite que deseja utilizar para o conjunto de azulejos:

Você também pode clicar no botão New Sprite para criar um novo sprite asset (que abrirá o Editor Sprite para o novo asset dentro do workspace para edição), ou você pode editar o sprite atualmente selecionado usando o botão Edit Sprite , ou mesmo optar por editar a própria imagem sprite no Editor de Imagens pressionando o botão .

Por padrão quando você abre um tile set a janela Propriedades do Tile Set também será mostrada (veja abaixo), mas se você fechá-la, então você pode clicar no botão aqui para reabri-la novamente.

Imóveis

A janela Propriedades do Tile Set é para designar o tamanho da base tile (o padrão é 16px quadrado), a quantidade de offset e a separação entre tiles (estas duas últimas opções são para uso com sprites como a imagem de exemplo mais acima nesta página onde há lacunas entre as partes individuais do tiles). Você também pode definir os valores da Borda de Saída para a imagem tile set, que controla como a imagem será armazenada no disco e adicionar pixels extras ao redor de cada tile conforme necessário.

Devido à forma como o tiles é apresentado, eles precisam estar preparados de antemão com uma borda ao redor de cada um para "manchar" as bordas. A imagem mostrada abaixo ilustra como o tiles é geralmente mapeado pelos drivers gráficos do dispositivo, com a linha vermelha ao redor do tile mostrando para onde as coordenadas U,V são mapeadas (As letras "U" e "V" são usadas para descrever a malha 2D texture porque "X", "Y" e "Z" já são usadas para descrever room e object coordenadas espaciais):

Como você pode ver, escolhe o centro do texel para mapear diretamente ao desenhar a imagem no room. Agora a uma escala de 1:1, tudo isso está bem e bom, mas o que acontecerá quando escalarmos este tile para baixo? A questão com a escala para baixo é que quando você diminui a imagem, os UVs também diminuem e tentam caber em um espaço mais comprimido, mas isso obviamente não pode acontecer, então o driver gráfico escolhe quais textos desenhar com base nos centros de pixels disponíveis.

Então, se estamos "apenas" nos movendo para um novo pixel de tela então, porque os drivers gráficos usam o centro do pixel para decidir com o que desenhar, ele escolherá um texel ao redor do tile, e não um texel no próprio tile.

Mostrado acima está um tile sendo desenhado em 6 pixels da tela, e isto mostra que quando você se sobrepõe um pouco no próximo pixel da tela, o driver gráfico tem que fazer uma escolha entre deixar cair aquele texel, ou tentar preenchê-lo com o que ele pensa ser um texel válido. Digamos apenas que, neste caso, ele decide desenhar o texel sobreposto no pixel da tela e, como agora está claramente fora de nosso tile, ele recebe este pixel do próximo tile, ou dos pequenos pedaços de espaço que você forneceu ao redor do tiles, o que significa que quando ele encolhe, você terá rachaduras entre tiles "falhas" na forma como eles são desenhados e é aqui que surge muita confusão... Cada driver gráfico e dispositivo realmente faz isso à sua maneira, o que significa que os resultados podem ser imprevisíveis e parecer terríveis... mas se você for cuidadoso e planejar com antecedência, você pode trabalhar em torno disso.

Com jogos de ponta se você desenhar de um único texture (não em uma página texture ou texture atlas), você tem a opção de usar um modo texture chamado clamp e o que isto faz é repetir a última fileira de textos infinitamente fora na distância, permitindo que você escale sem problemas, pois obriga o hardware a obter a última fileira de pixels, não importa o que aconteça. Isto é o que as configurações de saída de fronteira aqui fazem por você:

Mostrado acima é um "fixo" tile sprite e você pode ver que agora tem uma seção repetida ao redor de cada tile, o que significa que quando o hardware for ultrapassado, ele sempre escolherá o último texel do tile em vez de um dos espaços ou o tile depois disso. Portanto, ao criar um tile set, tenha em mente que se você for ampliar ou diminuir a visão do jogo ou se for renderizar o tiles em escala, talvez você precise ter uma largura ou altura de borda de saída maior do que o valor padrão de 2.

A seguir, você pode escolher o grupo de textura ao qual você deseja que o recurso de azulejos pertença. Basicamente, um grupo texture (previamente definido no Gerente do Grupo de Textura) é algo que você pode configurar para que toda a imagem assets que você precisa para rooms ou níveis específicos de seu jogo possam ser armazenados juntos. Assim, por exemplo, você pode ter todas as suas imagens de nível 1 em um grupo texture, todas as imagens de nível 2 em outro etc... e o GameMaker tentará colocar todos esses recursos agrupados na mesma página de textura para reduzir a troca de páginas texture enquanto seu jogo estiver rodando na plataforma alvo escolhida.

NOTA: Isto pode não ser sempre necessário e o aumento de desempenho deste método dependerá se o dispositivo alvo está vinculado à CPU ou à GPU.

A opção final nas propriedades tile set é a opção Disable Source Sprite Export. Isto é verificado por padrão, e o que faz é flags a fonte sprite asset a ser omitida quando a página texture para seu projeto estiver sendo gerada. Isto funciona sob a suposição de que um sprite usado como tile set não será usado para mais nada, e portanto só precisa ser exportado uma vez. Entretanto, se você estiver usando o recurso base sprite para qualquer outra coisa além de tiles, então você gostaria de desmarcá-lo para que a base sprite e o tile set sejam ambos adicionados à página texture em compilação.

Editores

Cada um dos botões na seção Editores abrirá um editor diferente para funcionalidades específicas tile set. Você pode criar tile set escovas baseadas em múltiplos tiles juntos, azulejos animados, usando diferentes tiles como molduras individuais, e você também pode criar azulejos automotivos que "se unirão" com base no tiles que os rodeia. Cada um destes editores é explicado em detalhes nas seções ligadas no final desta página.

Caixa de ferramentas de pré-visualização

Estas são as ferramentas que você pode usar para pré-visualizar o sprite selecionado para o tile set. Estes botões controlam o nível atual de zoom de pré-visualização e você pode usá-los para aumentar ou diminuir o zoom e clicar no botão irá redefinir a pré-visualização para ser 1:1 com o tile set sprite sendo editado. Você também pode clicar no botão Window Fit para fazer todo o tile set caber dentro da janela de pré-visualização atual (isto fará com que o zoom in/out seja o apropriado para fazê-lo caber). Observe que você também pode aumentar e diminuir o zoom usando o / e a Roda do Rato , e pressionando / + a tela será 1:1 com o tile set sprite.

Outra característica é que você pode switch a visualização da grade, ligando ou desligando, clicando no botão , bem como definir a cor da grade a ser utilizada (clicando no botão ). Quando a grade estiver ativa, ela mostrará um contorno colorido para cada um dos tiles da grade, respeitando as configurações de separação, etc...

Pré-visualização

Esta seção é a janela de pré-visualização tile set. Para navegar ao redor da visualização, você pode usar os mesmos controles básicos que para o geral Workspaces, ou seja: Use o botão central do mouse e arraste para girar a visualização (ou, alternativamente, use a combinação + ), e você pode rolar horizontalmente com a roda do mouse ou aumentar e diminuir o zoom usando a tecla / e a roda do mouse . Você também pode usar os diferentes botões da Caixa de ferramentas de visualização prévia para mudar o zoom e ativar/desativar a sobreposição de células da grade.

Como mencionado na seção sobre Editores acima, há outras ferramentas disponíveis para editar tile sets e gerar pincéis especiais (combinações de tiles que podem ser pintadas em um room), azulejos animados e azulejos automotivos (que são especiais tiles que se conectarão automaticamente quando colocados em um room). Cada um destes editores é explicado em profundidade a partir das páginas seguintes:

  • O construtor da escova
  • Ladrilhos Animados
  • Azulejos automotivos
