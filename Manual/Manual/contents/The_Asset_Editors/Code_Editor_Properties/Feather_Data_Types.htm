Tipos de dados de pluma

Feather usa tipos de dados para fornecer uma verificação de sintaxe inteligente ao escrever código, assegurando que você não use o tipo de dados errado para uma variável ou parâmetro de função.

Também permite especificar os tipos de dados para os parâmetros e valores de retorno de suas próprias funções de script, usando os comentários do JSDoc.

As tags @param e @return JSDoc permitem que você especifique qualquer um dos seguintes tipos:

NOTA Os tipos JSDoc são insensíveis a maiúsculas e minúsculas, o que significa que Id.DsList e id.dslist são a mesma coisa.

NOTA * A coluna Exemplos de especificação lista alguns exemplos de especificadores, mas não todos eles. Por exemplo, pode haver mais tipos possíveis sob os grupos Id, Asset, e Constant, que não estão listados aqui.

Você pode ver um tipo Any* no IDE ao usar Feather, o que indica que o tipo do identificador (que pode ser uma variável, parâmetro, valor de retorno, etc.) não pode ser discernido pelo Feather no momento. Quando mais código for adicionado para esse identificador, ele poderá assumir um tipo específico.

Especificadores

Um especificador é adicionado após o tipo de dados base usando um ponto . para especificar o tipo exato de dados nesse grupo.

Tipos tais como Id.DsList, Asset.GMObject, e Constant.Color use specifiers.

Tipos de coleta

Tipos como Array e Id.DsList, que são estruturas de dados que contêm múltiplos valores, são capazes de especificar um único tipo de dados para todos os seus conteúdos.

Isto é feito usando Tipos de Coleção, que são anexados ao tipo usando colchetes angulares <>.

Por exemplo, uma matriz contendo strings seria Array<String>, e uma Lista DS, contendo um Array, contendo Reais, seria Id.DsList<Array<Real>>>.
