Código de edição

Quando você começa a usar o editor script, é simplesmente um caso de digitação do código que você quer ter para o script asset ou para o evento, seguindo a sintaxe correta do GML. Enquanto você faz isso, você receberá ajuda da janela de auto-completar:

Esta janela irá aparecer enquanto você estiver escrevendo seu código (normalmente após uma pequena pausa, mas isto pode ser definido nas Preferências) e lhe dará uma seleção de respostas "melhor adaptação" em ordem alfabética.

Ao usar o editor de código você pode clicar em e arrastar o mouse para selecionar uma linha de código ou manter pressionado , depois clicar em e arrastar para selecionar colunas de código. Qualquer coisa selecionada desta maneira pode então ser copiada, cortada e colada usando os atalhos de teclado padrão de / + " C", / + "X" e / + "V".

Vale a pena notar que você pode comentar seu código de várias maneiras. Comentar seu código é muito importante para manter a produtividade, pois mantém as coisas claras e separa as seções do código de tal forma que é óbvio o que cada parte faz e por que está lá. Isso também significa que se você estiver trabalhando em equipe, poderá deixar notas para os outros membros e que eles saberão exatamente o que você fez ou para que serve uma seção específica do código. Mesmo se você for um desenvolvedor solo, deixar comentários em seu código é importante, especialmente se você tiver que deixar o projeto e voltar a ele mais tarde.

Comentar seu código é simples, pois você só precisa usar "//" antes de qualquer linha de texto ou você pode comentar parágrafos inteiros usando "/* text here */". Há até mesmo dois atalhos de teclado para comentar e descomentar as linhas de código selecionadas: / + "K" para comentar, e / + + "K" para des-comentar. Os scripts também podem ter certos comentários em estilo JSDoc adicionados para definir as propriedades das funções que eles contêm, de modo que quando você usa as funções em seu código, eles aparecem em auto-completados junto com seus argumentos e outros detalhes. Você pode saber mais sobre isso na seção JSDoc Script Comentários para mais informações.

Por padrão, todos os códigos escritos terão ícones de código dobráveis ao lado de cada conjunto de chaves correspondentes {}. Estes assumem a forma de um pouco mais de + ou menos de - ícones, e clicando sobre estes irá "dobrar" o código dentro das chaves correspondentes para que ele tome menos espaço visual.

Você também pode definir manualmente regiões dobráveis de código usando as tags especiais #region para marcar uma seção de código. Como isto funciona é que você adicione uma tag #region (junto com qualquer texto de comentário) ao topo de uma seção de código e depois adicione uma tag #endregion correspondente ao fundo da seção e então você pode "dobrar" o código, ou seja: esconda-o para que você possa cortar o número de linhas de código visíveis no editor a qualquer momento:

O código real se parece algo assim:

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

Como você deve ter notado, partes do texto script são coloridas. O editor do script "sabe" sobre objects existente, variáveis e funções embutidas, etc... e os códigos de cores correspondentes. Os códigos de cores ajudam muito a evitar erros, como você pode ver imediatamente se você escreveu mal algum nome ou usou uma palavra-chave como variável, ou tem um erro de sintaxe. No entanto, se você não gostar do código de cores, nas Preferências você pode switch ligar e desligar (você também pode usar F10 para ligar e desligar no editor), bem como personalizar as cores que são usadas.

Você também pode clicar com o botão direito em em qualquer lugar do editor para abrir o menu a seguir:

Além das funções regulares que você espera cortar, copiar e colar linhas de código, você também tem a opção de comentar ou descomentar qualquer texto selecionado, adicionar ou remover travessões, assim como dividir a janela de código em 1, 2 ou 3 painéis separados (note que as opções relacionadas a #regions não estarão disponíveis se você não tiver definido nenhum no código que está sendo editado).

A última opção para dividir em painéis dividirá a janela real em diferentes partes onde você pode então arrastar um script ou evento e fazer com que ele apareça junto com os outros para comparação e edição.

Isto é excepcionalmente útil para comparar dois scripts ou para visualizar scripts conectados ao mesmo tempo, sem precisar de outra janela ou ter que switch constantemente entre as abas.

As outras opções neste menu são alternar um breakpoint para uso com o Debugger, e converter o código GML em GML Visual ou vice-versa. Note que se converter o código para GML Visual, então ele não criará ações individuais para cada linha de código, mas usará uma única ação de código para manter tudo o que estava anteriormente no evento ou script.
