Social

Este evento só pode ser acionado pelas chamadas de retornode várias funções sociais específicas e funções UWP Xbox Live e retornará um Mapa DS armazenado na variável async_loadO evento, contendo diferentes pares de chaves/valores dependendo do retorno de chamada da função que acionou o evento.

Este evento também pode ser acionado pela Reprodução de Vídeo.

O mapa DS conterá uma série de chaves, a mais importante das quais é a chave "id". Isto retornará uma constante que poderá então ser verificada em código para determinar qual dos muitos callbacks o evento recebeu. Uma vez que você tenha analisado esta chave e comparado o valor retornado com as constantes disponíveis, você poderá então continuar a extrair o resto das informações do mapa.

As páginas de funções sobre Realizações e Leaderboards na seção Referência detalham as funções e callbacks, assim como as constantes usadas para acionar este evento, mas também há certas circunstâncias em que ele será acionado sem uma chamada de função do jogo que está em execução:

  • quando seu jogo é lançado do painel do sistema operacional
  • quando você completa um desafio
  • quando outro jogador completa um desafio
  • quando o jogo é iniciado a partir do painel do sistema operacional, selecionando um desafio

Qualquer uma das ligações acima desencadeará o Evento Social e um mapa async_load será gerado com os seguintes detalhes (note que há um valor chave diferente "id" para definir cada uma das diferentes razões pelas quais o evento foi chamado, mas todos os outros conteúdos do mapa são os mesmos):

  • "id" - O valor desta chave dependerá do tipo de callback que desencadeará o evento (utilizado em sua maioria por extensions).
  • "playerid" - A identificação do jogador para o desafio.
  • "issuerid" - A identificação da pessoa que lançou o desafio.
  • "state" - O estado do desafio, que terá um valor de 0 - 3 (como um string) para inválido, pendente, concluído ou recusado.
  • "message" - A mensagem de texto para o desafio.
  • "issueddate" - A data de emissão para o desafio
  • "completeddate" - A data de conclusão para o desafio.
  • "type" - O tipo de desafio dado.
  • "identifier" - A identificação string para o desafio.
  • "score" - A pontuação empatada com o desafio.
