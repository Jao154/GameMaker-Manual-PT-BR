Exemplos de fontes

Esta página contém exemplos e informações sobre extension código fonte para Android e iOS, para ajudar você a criar seu próprio site nativo extensions para essas plataformas.

Android (Exemplo básico)

Nosso exemplo extension para Android conterá funções que fazem o seguinte:

  • Devolver dois números somados
  • Devolver um string que consiste de uma entrada string e um valor
  • Devolver um string que consiste de duas cadeias de entrada
  • Acionar o Async - Evento social e preencher valores em seu async_load mapa

Este extension fará uso de um arquivo .java, cujo conteúdo é dado abaixo (clique para abrir):

GenéricosTest.java

Para adicionar tal arquivo ao seu extension, você precisa ir sob suas propriedades Android e primeiro digitar um nome de classe (para este exemplo, "GenericTest"), depois clicar em "Add Source" e selecionar o arquivo .java. Isto adicionará o arquivo selecionado ao diretório de seu projeto -- você pode clicar com o botão direito em seu extension, selecionar "Open in Explorer" e ver os arquivos fonte lá.

Agora você pode adicionar um novo arquivo de espaço reservado ao seu extension e criar funções dentro dele com as seguintes propriedades:

  • função / nome externo: AddTwoNumbers
    ajuda: AddTwoNumbers(value1, value2)
    argumentos: double, double
    tipo de retorno: double
  • função / nome externo: HowManyObjects
    ajuda: HowManyObjects(value1, value2, string)
    argumentos: double, double, string
    tipo de retorno: string
  • função / nome externo: BuildAString
    ajuda: BuildAString(string1, string2)
    argumentos: string, string
    tipo de retorno: string
  • função / nome externo: ReturnAsync
    ajuda: ReturnAsync(value1, value2)
    argumentos: double, double
    tipo de retorno: double

Aqui está um exemplo da função AddTwoNumbers():

Agora você pode chamar estas funções a partir de seu código de jogo e usá-las como qualquer outra função embutida:

var result1 = string(AddTwoNumbers(irandom(100), 50));
var result2 = BuildAString("Hello", "World");
var result3 = HowManyObjects(irandom(1000), irandom(1000), "Targets");
ReturnAsync(irandom(1000), irandom(1000));

A função ReturnAsync() desencadeará o evento Async - Social event, portanto você precisará adicionar esse evento ao seu object e ler os dados de seu mapa async_load (que é preenchido pelo código fonte da extensão dada acima).

var type = async_load[? "type"];

if (type == "finished")
{
    var result4 = "value1: " + string(async_load[? "argument0"]) + ", ";
    result4 += "value2: " + string(async_load[? "argument1"]);
}

iOS (Exemplo Base)

O exemplo fornecido para iOS é o mesmo que o exemplo Android dado acima; apenas os arquivos fonte diferem, o que cobriremos nesta seção.

iOS extensions faz uso de um arquivo .h e um .mm, e o conteúdo de ambos são fornecidos abaixo (clique para abrir):

GenéricosTest iOS Arquivos fonte

Para adicionar estes arquivos ao seu extension, você precisa primeiro ir sob suas propriedades iOS e digitar um nome de classe (para este exemplo, "GenericTest"). Você pode então usar a opção "Enter framework path" (se os arquivos estiverem presentes em sua máquina macOS usada para construção) ou a opção "Add Source" (se estiverem em sua máquina de desenvolvimento); independentemente da opção que você usar, você precisará indicá-los ao diretório que contém seus arquivos .h e .mm.

Você pode adicionar estes arquivos fonte ao mesmo extension que foi criado na seção anterior para Android, o que significa que quaisquer funções já definidas naquele extension agora funcionarão também em iOS (dado que os Nomes Externos de todas as funções em Android e iOS são os mesmos). Se você estiver criando um novo extension, você pode simplesmente criar as funções da mesma maneira que a descrita anteriormente.

Android (Callbacks)

public void onStop()

public void onNewIntent(android.content.Intent newIntent)
public void onWindowFocusChanged(boolean hasFocus)
public void onConfigurationChanged(Configuration newConfig)
