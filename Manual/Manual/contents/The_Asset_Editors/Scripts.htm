O Editor Script

Esta seção trata do editor de roteiro (também chamado editor de texto), que é onde você digita todo o código que seu jogo vai precisar para rodar. Este editor é usado para a criação personalizada script assets, assim como para codificar eventos de objetos, para adicionar código de criação de sala, e para muitas outras coisas.

Esta seção simplesmente detalha como funciona o editor script e não como criar script assets nem como usar o Código GML ou GML Visual para fazer scripts. Para isso, consulte as seguintes seções:

  • Código GML Scripts
  • Roteiros Visuais GML

Ao criar um script asset, você poderá ser solicitado a escolher entre GML Visual e GML Code. Consulte a linguagem GameMaker para mais informações.

O editor script é onde você escreve seu código GameMaker Language, que é a linguagem de programação integrada que o GameMaker usa (veja a seção Linguagem GameMaker para mais detalhes). Quando você se familiarizar mais com o GameMaker e quiser usá-lo ao máximo, é aconselhável começar a aprender a usar esta linguagem, pois ela expande muito suas possibilidades ao criar jogos.

Um script asset pode ser renomeado clicando com o botão direito do mouse sobre ele no Navegador de Ativos e selecionando Renomear, mas observe que o nome script deve estar de acordo com as regras de script para assets, de modo que eles só podem ser alfanuméricos, devem começar com uma letra, e o único símbolo que eles podem conter é o símbolo "_" abaixo da barra. O editor script em outros lugares não pode ser renomeado e terá um nome específico para o que está sendo editado, como um evento object, por exemplo.

Quando você abrir a janela do editor script, ela terá as seguintes opções e layout:

Roteiro / Aba de eventos

O editor script abre em uma janela com abas no topo para que você tenha vários scripts ou eventos em uma janela (embora este comportamento possa ser alterado das Preferências para dar uma nova janela para cada script ou evento). Você pode clicar em em uma aba e depois arrastá-la para alterar a ordem da aba, ou - se preferir - você pode puxar ela fora da janela atual e colocá-la no workspace para criar uma nova janela para isso script (ou adicioná-la a uma janela diferente), e você também pode maximizar o editor script para criar um novo workspace também. Se você arrastar uma aba script para fora da janela atual e a adicionar a uma janela diferente, você também pode maximizar o editor para criar um novo também. Janela IDE, então será gerado um novo IDE para manter este recurso script, e ele pode ser usado como se fosse a janela principal.

Observe que se você estiver editando código de um evento objeto no editor script e tiver maximizado o editor script ou tê-lo em uma janela separada ou workspace, então o menu do editor com o botão direito do mouse terá algumas opções extras:

  • Restaurar para o espaço de trabalho: Isto tirará o editor script de um estado maximizado/espaço de trabalho e o reconduzirá ao object no workspace.
  • Ir para Objeto: Isto o levará ao workspace em que o object com o código está e se concentra no object.
  • Adicionar /Open Event: Isto permite adicionar um novo evento ao object ao qual pertence o atual script, e abrirá uma nova aba de código no Editor Script para o evento adicionado. Se o evento selecionado já tiver código no mesmo, então isto ser aberto em uma nova aba.

Para obter informações sobre o restante das opções do menu clique com o botão direito do mouse, veja a seção sobre Edição, abaixo.

Gutter

A Gutter é usada para mostrar os números de linha de seu código e também para transmitir algumas informações específicas sobre o estado do código. Quando você alternar isto, a função de busca somente destacará aqueles strings que correspondem a toda a entrada string. Por exemplo, com a busca por "aleatório", aparecerão todas as palavras que contenham este string - como irandom(), ou randomise() - enquanto que alternar com ele mostraria apenas a função random().
Quando esta opção é alternada, você está dizendo ao GameMaker para verificar não apenas o conteúdo da busca string, mas também o caso. Por exemplo, se você tiver um sprite chamado "spr_Dog" e fizer uma busca por " cão" com esta chave múltipla desligada, então o sprite string será destacado, porém se a chave múltipla estiver ligada , não será mais, pois "cão" não é mais considerado o mesmo que "cão".

Editor Principal do Código

O editor de código principal é onde você escreverá todo o seu código para criar o script asset ou preencher um evento object. A edição real de seu código é abordada na seção de Edição mais abaixo nesta página.

Informações

A barra de informações no final da IDE mostra o número da linha atual e a posição ao longo da linha. É também onde você pode ver o ajudante de código, que é uma linha de texto que mostra a função que você está atualmente editando ao longo com os argumentos que ela requer. Ao preencher a função no editor, os argumentos destacarão qual deles você está editando atualmente.

Note que quaisquer argumentos opcionais em suas funções script serão mostrados com colchetes [] ao seu redor. Se você usou o JSDoc Script comentários dentro de um script asset, então o As informações que você forneceu também serão mostradas aqui.
