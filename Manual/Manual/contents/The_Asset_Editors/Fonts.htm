O Editor Font

Quando você quer desenhar um texto em seu jogo, este texto é desenhado em uma fonte padrão Arial 12 pontos, mas para fazer textos mais interessantes ou de aparência única, você provavelmente vai querer usar diferentes fonts. Portanto, para usar diferentes fonts que você tem em seu computador, você deve criar um recurso de fonte no GameMaker. Isto é feito clicando com o botão direito do mouse no Navegador de Ativos e selecionando "Create" -> "Font" (Criar). Para cada font asset você especifica um tipo particular de font de seu computador que pode então ser usado em seu jogo usando a ação ou código para definir um font para desenho.

Quando você criar um recurso font pela primeira vez, a janela do editor font será aberta com as seguintes opções:

Detalhes

Aqui você pode dar a seu font um nome único (somente caracteres alfanuméricos e o símbolo "_" abaixo da barra), bem como escolher o font para usar. O font deve ser instalado em sua máquina e, devido a problemas de licenciamento, o GameMaker não armazenar os arquivos reais font com o arquivo do projeto por padrão, mas o font é renderizado a uma página de textura como uma série de glifos bitmap. A página texture será atribuída a um grupo de textura, que você pode escolher a partir da opção Grupo também aqui.

Se você quiser distribuir o arquivo real font ( *.ttf ou *.otf somente) junto com o projeto ao fazer um Marketplace asset ou por qualquer outro motivo, você pode verificar a opção Copiar para Projeto. Este abrirá o explorador de arquivos onde você pode selecionar o arquivo font e ele será então adicionado aos arquivos agregados com o projeto.

IMPORTANTE! Você só deve usar fonts que você está legalmente permitido e licenciado para redistribuir ou usar em seu projeto.

Predefinições

Nesta seção você pode definir o tamanho (em pontos - o padrão é 12pt) e se o font deve ser desenhado em negrito ou itálico ou se deve ser aplicado anti-aliasing. A janela de visualização à direita lhe mostrará o texto aproximadamente como ele aparecerá em seu jogo com o tamanho e as transformações que você especificou.


Controles de alcance

Ao criar um recurso font é raro que você queira incluir cada glifo para o font. É por isso que o GameMaker não usa por padrão apenas os personagens de 32 a 127 do font escolhido, pois isso nos dá os números e as letras pequenas e maiúsculas juntamente com certos sinais de pontuação. Você pode, entretanto, alterar a faixa de caracteres utilizada para ajudar a otimizar seus jogos, ou seja: se você precisar apenas dos números de um font específico, então selecione apenas os números. Para fazer isto, você deve primeiro clicar no botão Excluir para limpar a faixa atual (você pode precisar selecionar a faixa para que isto funcione, clicando sobre ela na janela de pré-visualização) e depois clique no botão Adicionar para adicionar uma nova faixa. Isto abrirá a seguinte janela encadeada:

Esta janela tem alguns botões para ajudá-lo a estabelecer uma faixa padrão para sua fonte:

- A faixa Normal vai de 32 a 127

- As ASCII variam de 0 a 255

- A faixa de dígitos que contém apenas os 10 dígitos

- O intervalo de letras que contém todas as letras maiúsculas e minúsculas apenas

Outros intervalos podem ser usados digitando o primeiro e o último índice de caracteres do intervalo que você deseja definir nas caixas de entrada do intervalo de caracteres (se um caractere não estiver no intervalo, ele é substituído por um espaço). Além destes botões e caixas de entrada, você também tem duas outras opções que são especialmente úteis:

  • Do Código: se você clicar neste botão e depois clicar OK, o GameMaker analisará automaticamente seu código de jogo para strings e depois criará diferentes intervalos de caracteres para cobrir todo o texto em seu jogo. Observe que ele procura por todos strings (seja dentro de "" ou ''') e assim também pode incluir nomes de arquivos nas faixas de caracteres.
  • De arquivo: isto funciona de forma similar à opção "de código" explicada acima. Se você clicar nele, você será solicitado a fornecer um arquivo, e então uma vez que isso for feito, você deverá clicar em OK. O GameMaker então analisará o e criar intervalos de caracteres para o texto encontrado dentro dele.

Observe que você pode editar manualmente o intervalo de texto adicionando ou excluindo glifos de texto diretamente na janela de visualização.

Controles de textura

Como mencionado acima (na seção sobre Detalhes da Fonte) por padrão, todos os font glifos serão renderizados a uma página texture como parte de um grupo texture para convertê-los em recursos gráficos, e quando o jogo for compilado o font será adicionado a um texture temporário para criar uma única imagem com todos os glifos necessários font e então isto será adicionado ao principal grupo texture para exportação. Os botões aqui são para ajustar a forma como o font texture é gerado ou para regenerar um novo font texture.

O botão Opções abrirá as opções avançadas font texture:

Estas opções podem, às vezes radicalmente, alterar a forma como um font será exibido no jogo, e nem todos os fonts exigirão as mesmas configurações. As opções a seguir são projetadas para dar a você a maior personalização possível para sua fonte:

  • Sugestão: Font Sugestão (também conhecida como instrução) é o uso de instruções matemáticas para ajustar a exibição de um esboço font para que ele se alinhe com uma grade rasterizada. Em resoluções baixas de tela, a insinuação é crítica para produzir texto claro e legível, e aqui você pode selecionar entre diferentes algoritmos de insinuação para obter o melhor resultado possível. O padrão é "normal".
  • Opções de Glyph: Estas opções podem ser usadas para alterar a forma como os glifos são carregados, habilitando/desabilitando diferentes operações. Estas são:
    1. Desativar a indicação: Isto desativará a dica nativa fonts e o auto-hinteresse.
    2. Desativar a auto-interpretação: Isto desabilitará a auto-interpretação e, em vez disso, usará a fonts hinterização nativa (se ela tiver uma). Note que isto substitui a opção "Preferir a Auto-interpretação".
    3. Prefira a auto-interpretação: Isto dirá ao GameMaker para preferir usar a auto-interpretação em vez de qualquer dica nativa que o font possa ter. Note que isto é substituído pelas opções "Desabilitar Auto-hinter" e "Desabilitar Dica".
    4. Deformação: A ativação disto irá deformar ligeiramente os glifos para melhorar o alinhamento da grade. Note que isto requer o modo "Auto-hinter" e um modo "Normal" de insinuação.
    5. Sem Escala: Quando esta opção for marcada, ela manterá os glifos em unidades font e ignorará o tamanho solicitado. Isto implica em desabilitar a insinuação.
    O estado padrão para todas as opções acima é desativado.
  • Interpretador: Aqui você pode selecionar a versão TrueType que o GameMaker deve usar para a renderização font. O padrão é TrueType v40.
  • Arredondamento: Esta opção é para definir o método de arredondamento ao converter de um ponto fixo decimal para um número inteiro. O padrão é "piso".
  • Aplicar Kerning: Isto controla se os dados kerning do arquivo font são usados para renderizá-los dentro do jogo. Isto deve ser desativado se os dados kerning da fonte font estiverem incorretos e resultar na não exibição correta do font.

O botão Regenerate pode ser usado para regenerar o font texture. Isto é necessário se você quiser atualizar qualquer mudança que tenha feito em um font existente, ou se você tiver criado um novo font e quiser ver como ele fica na Janela Preview. Observe que você também pode regenerar fonts a partir do botão direito do menu para o font (ou fonts) no Navegador Asset.

É importante notar que quando você cria um novo font pela primeira vez, se você não regenerar o texture no editor, então o texture será criado na primeira vez que você executar ou testar seu jogo, mas esse é o único momento em que ele será gerado automaticamente. Isto significa que uma vez que um font tenha um texture gerado, qualquer alteração feita no recurso font exigirá que o font seja regenerado para que as alterações sejam visíveis no jogo, caso contrário, o "antigo" font texture ainda será utilizado.

OGameMaker apresentará o gerado font texture como um arquivo PNG junto com o resto dos arquivos do projeto, e se você carregar o projeto e a base font não for encontrada, então o jogo ainda exibirá o texto corretamente usando o arquivo texture armazenado. Um font em falta significará, entretanto, que você não poderá mais alterar nenhuma de suas propriedades, além de seu nome, família font e grupo texture (do Editor do Grupo de Textura).

Modo de pré-visualização

Nesta seção você pode escolher que a Janela de Visão Antecipada mostre algum texto de amostra do font selecionado, ou a imagem renderizada texture que foi gerada para o font.

Pré-visualização

O Preview Window é onde você pode ver uma prévia do font, seja como uma amostra de texto, ou como um texture. Quando o O modo de pré-visualização está definido para Amostra, você pode digitar qualquer texto que queira testar diretamente na janela de visualização para ver como ele fica, enquanto seleciona Game Preview mostrará a imagem renderizada texture que foi gerada para o font. Se nenhuma imagem estiver presente, então você deve clicar no Regenerar botão para gerar o texture.

Note que a imagem texture pode ser diferente da amostra, uma vez que o texture irá não ser regenerada, a não ser que seja explicitamente dito para fazê-lo. Isto significa que depois de ter gerado inicialmente um font texture, as alterações feitas no recurso font não serão transferidas para o jogo, a menos que o texture seja regenerado.
