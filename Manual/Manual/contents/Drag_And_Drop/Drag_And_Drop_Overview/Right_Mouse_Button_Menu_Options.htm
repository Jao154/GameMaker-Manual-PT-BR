Opções do Menu do Botão Direito do Rato

Ao trabalhar em seu GML Código visual, você pode clicar com o botão direito do mouse na ação, no workspace, ou no texto da lista restrita do lado esquerdo para abrir o menu do botão direito do mouse:

Este menu tem as seguintes opções:

  • Cortar / Copiar / Colar: Isto lhe permitirá cortar, copiar ou colar qualquer ação selecionada no workspace. Note que você pode usar + fora das ações e arrastar, ou usar / + em ações individuais, para selecionar mais de um e aplicar o comando a todos eles.
  • Comentar / Remover Comentário: Esta opção lhe permite adicionar e remover comentários de seus GML blocos de código visual. Quando você selecionar Comentar, o editor de texto será aberto à direita do bloco visual GML selecionado, e lá você poderá adicione qualquer texto necessário, seja um lembrete para mudar algo, ou um comentário útil para um tutorial, ou mesmo instruções para seus colegas de equipe:
    Depois de adicionar o comentário, você pode clicar nele e arrastá-lo para posicioná-lo verticalmente e horizontalmente - embora apenas à direita do nó pai - e colapsando o GML Nó visual também colapsará o comentário e mostrará um resumo de seu conteúdo (caso em que não poderá ser reposicionado). Se o comentário não for mais necessário, então você pode ou apagar todos os o texto no comentário, ou use a opção Remover Comentário do menu. Observe que quando você copia um nó, o comentário será copiado junto com ele, e se você converter o nó de GML Visual para GML, então o comentário será preservado. e adicionado ao GML que é criado, como ilustrado na imagem abaixo:
    Como mencionado acima, um comentário só pode ser aplicado a um único nó, mas você pode expandi-lo para cobrir vários nós, simplesmente adicionando intervalos na linha para estendê-la - embora isto possa exigir que você reposicione o comentário para evitar sobreposições com os nós subseqüentes:
    Uma vez que um comentário tenha sido adicionado, você pode selecionar a opção "Comentário". novamente a partir do menu RMB ou você pode clicar duas vezes sobre ele para habilitar a edição.
  • Desativar / Habilitar: Estes comandos podem ser utilizados para desativar ou ativar a ação (ou ações) selecionada. Quando uma ação é desativada, ela ainda é visível no workspace, mas não será executada quando você compilar o jogo. Neste como você pode adicionar/remover ações para testes e debugging. A imagem abaixo mostra alguns GML Código visual com ações desabilitadas:
    Você pode ver que as ações "If Score" e "Go To Next Room" foram desativadas... foram acinzentados, visualmente - e por isso serão pulados quando o jogo for testado ou compilado.
  • Troca de ponto de parada: Este comando irá alternar um breakpoint ligado ou desligado para a ação selecionada. Um breakpoint é simplesmente um ponto no GML Código visual onde o depurador deve parar quando o jogo está sendo executado no modo Debug. Uma ação com um breakpoint atribuído a ela será destacada, como mostrado na imagem abaixo (a ação "Se Saúde" foi definida como um breakpoint):
  • Apagar todos os pontos de parada: Com este comando você pode remover todos breakpoints do código visual GML.
  • Colapsar tudo / Expandir tudo: GML As ações visuais podem ser colapsadas ou expandidas, onde uma ação colapsada simplesmente mostrará o nome da ação mas não qualquer um dos valores internos, maximizando assim o espaço disponível. para o workspace. Por exemplo:
    Como você pode ver, as ações ocupam muito menos espaço, e você ainda pode ver o que elas fazem a partir do curto descrição dada à direita. Você pode selecionar a opção "Expandir tudo" para desfazer isto para cada nó no workspace.
  • Redefinir o layout: Isto irá redefinir o GML Visual workspace, ajustando o nível de zoom para 100%, focando na ação atualmente selecionada, e expandindo qualquer (e todas) as ações colapsadas também.
  • Pré-visualização ao vivo: Ao selecionar esta opção, você ativará ou desativará a janela de visualização de código GML Visual para GML (GameMaker Language). Isto é explicado com mais detalhes na página Mudando GML Visual para Código.
  • Converter para GML: Isto converterá todas as suas ações no evento atual para o código GML (GameMaker Language). Tenha muito cuidado com isto, pois você não poderá converter GML de volta para o código GML Ações visuais a partir das quais foi criado. Você pode converter o evento de volta para um evento visual de código GML, mas qualquer código será simplesmente envolto em uma ação de Código de Execução. Isto é explicado com mais detalhes na página Mudando GML Visual para Código.
  • Janela: Esta seção abre o seguinte sub-menu:
    Os itens de menu disponíveis aqui são:
    • Ir para Objeto: Isto vai levar o workspace para o object que está chamando o evento
    • Evento Aberto Herdado: Esta opção só está disponível se o object que está sendo editado for um filho de um dos pais object, e abrirá o evento que está sendo editado do pai object e levará o workspace para ele.
    • Painel Único / 2 Colunas / 3 Colunas: Estas opções podem ser usadas para dividir o atual workspace em 1, 2 ou 3 "painéis" diferentes. Cada painel é um workspace separado que pode ter scripts ou eventos arrastados sobre eles para que você possa editar e comparar 2 ou mais GML blocos de código visual de cada vez. As duas opções a seguir só estão disponíveis quando o menu RMB é aberto em um nó visual GML:

  • Pan: Ao selecionar esta opção em qualquer nó, o workspace irá pan para tocar aquele nó para o centro.
  • Mostrar Ajuda... : Selecionando isto abrirá o Manual em um novo workspace na página relevante para o nó que foi selecionado. Você também pode obter os mesmos resultados usando no nó.
