Visão geralvisual da GML

GML Visual é uma ferramenta de script visual que pode ser usada para criar seus jogos sem realmente digitar nenhum código. Isso não quer dizer que você não esteja programando código quando usa o GML Visual, como você é, apenas que seu código é criado e apresentado de forma visual usando "blocos" de ações em vez de texto. Ações sãoexpressões ouafirmações simples que podem ser "encadeadas" juntas para que uma instância de um object faça algo quando colocada em um room. Por exemplo, você poderia ter uma ação em um Create Event para mover uma instância em uma direção aleatória, e outra ação em um Collision Event para fazê-la ricochetear nas coisas com as quais colide (para mais informações sobre os diferentes eventos disponíveis e como eles funcionam juntos veja a seção sobre Object Events).

Fichas de eventos

Ao adicionar um Evento ao seu object, uma nova aba é aberta na janela de código para exibir o conteúdo desse evento. Você pode adicionar vários eventos e ter várias abas abertas ao mesmo tempo, alternando entre elas e editando-as como desejar. Você também pode fechar as abas a qualquer momento para reduzir a confusão (selecionando o evento no object novamente as reabrirá), e você pode clicar em e arrastar a aba para fora da janela em sua própria janela de código, neste caso fechando o object ou a janela de código não fechará a nova janela. As abas podem ser arrastadas para a esquerda ou direita para serem reordenadas também.

Visão geral das ações

Ao adicionar blocos de ação ao seu evento, eles aparecerão em forma de "estenografia" como uma simples visão geral em inglês do que está acontecendo. A ordem geral é a ordem em que cada um dos blocos de ação será executado para aquele evento, e você pode clicar em qualquer item da lista para que a visualização dos blocos de ação aumente para aquela posição para edição.

Bloco de Ação Espaço de Trabalho

Esta área workspace é onde você arrasta ações da Caixa de Ferramentas para criar seu código visual GML. Clique em em uma ação e arraste-a para esta área para adicioná-la à lista de ações e editá-la. A maioria das ações tem váriosargumentos (parâmetros) que você pode definir, e a maioria também utilizará palavras-chave, variáveis de exemplo, ou mesmo funções GML - por exemplo, você pode chamar a ação Set Direction e usar a função GML. random(val) para definir uma direção aleatória.

Ao adicionar blocos de ação ao seu código, eles serão visualmente "encadeados" para indicar a ordem em que as diferentes ações ocorrerão, embora a forma como esta cadeia é criada dependerá do bloco inicial que foi utilizado. Por exemplo, um bloco If Variable terá duas cadeias - uma para os blocos que serão chamados se a condição a ser verificada for true e depois disso, o próximo bloco. Para mais informações sobre como construir blocos de ação em código, veja as seções ligadas abaixo.

Enquanto estiver no bloco de ação workspace, você pode pressionar e segurar o botão central do mouse para girar e a roda do mouse para rolar a lista de ação para cima ou para baixo. Se você segurar enquanto move a roda do mouse, então a vista irá rolar para a esquerda/direita, e se você segurar / e mover a roda do mouse então a vista irá aumentar e diminuir, permitindo que você veja mais do workspace com as ações. Você pode clicar em qualquer ação da lista de ações à esquerda para se concentrar nessa ação, redefinindo o zoom e a posição. Observe também que você pode redimensionar os blocos de ação movendo o mouse em direção às bordas do bloco e depois - quando o cursor mudar - arrastando na direção desejada.

Você também pode clicar com o botão direito do mouse em qualquer lugar no GML Visual workspace para trazer o Menu RMB, que é explicado em detalhes aqui.

Caixa de ferramentas

A caixa de ferramentas é onde são armazenadas todas as ações visuais GML, com diferentes seções contendo coleções (bibliotecas) de ações que são similares ou relacionadas. Cada coleção de ações está relacionada a um tema específico ou propósito comum e basta clicar em e arrastar a ação necessária da caixa de ferramentas para o bloco de ações workspace para adicioná-la ao evento atual. Para tornar as coisas mais simples - como há muitas ações - você pode usar a barra "Pesquisar" no topo para procurar uma ação específica ou para filtrar as ações visíveis (digite "desenhar" por exemplo para obter todas as ações de desenho), e você também pode clicar e arrastar ícones de qualquer biblioteca para sua pasta Favoritos para manter as ações comumente usadas juntas. Isto é particularmente útil se houver certas coisas que você estará fazendo de novo e de novo. A pasta de Favoritos pode ser limpa clicando no pequeno ícone do "container".

Para uma lista completa das ações disponíveis, bem como quais argumentos eles levam e como eles podem ser encadeados, consulte a seção Referência Visual GML.

Você pode usar o botão direito do mouse em qualquer ação na caixa de ferramentas para abrir o seguinte menu:

Aqui você tem as seguintes opções:

  • Use <action>: Isto adicionará a ação que estava sob o mouse quando o botão direito do mouse foi clicado no evento atual workspace.
  • Faça / Remova o Favorito: Isto adicionará ou removerá a ação na biblioteca de Favoritos na parte superior da caixa de ferramentas. Esta biblioteca é onde você pode criar um grupo personalizado de ações que utiliza regularmente, reduzindo a quantidade de tempo necessária para procurá-las em suas bibliotecas reais.
  • Clear Favourites: Selecionando isto, todas as ações serão removidas da biblioteca Favourites.
  • Clear Recentemente utilizado: Selecionando isto, todas as ações serão removidas da biblioteca recém-usada.
  • Collapse All: Esta opção colapsará todas as bibliotecas visuais GML para que somente os títulos das bibliotecas sejam visíveis, e não os nós de ação. Cada biblioteca pode ser expandida novamente clicando nelas.
