Criar Sistema de Partículas

Esta ação inicializará o sistema de partículas pronto para você criar e utilizar tipos de partículas e emissores com ele. Você dá o nome da camada como um string - por exemplo, "Particle Layer" (que deve existir no editor room ) - e depois flag o como sendo Persistente ou não. Se o sistema for marcado como Persistente, ele permanecerá ativo em todo rooms após ter sido criado, e lhe será atribuída uma camada gerenciada se a camada inicial não existir em rooms subseqüente. Se a persistência não for assinalada, então o sistema será automaticamente destruído no final do room em que foi criado, assim como quaisquer emissores associados ao sistema. Se você desejar remover manualmente o sistema da memória, você precisa usar a ação Destroy Particle System (Sistema de Partículas Destroy).

Você também precisa fornecer uma variável de destino que mantenha o valor único de identificação do sistema de partículas. Este valor de identificação é o que será usado em outras funções para definir as propriedades do sistema ou para dizer ao GameMaker qual sistema usar ao criar partículas. Você pode escolher flag esta variável como sendo uma variável local temporária, mas isso significará que ela será descartada no final da ação script ou evento, significando que o sistema ainda existirá mas deixará você sem nenhum meio de acessá-la, portanto cuidado devem ser tomadas quando se usa isto (em geral, os sistemas nunca precisarão ser armazenados em uma variável local temporária, mas para certas coisas, como o desenho de partículas para uma superfície, pode ser necessário).

NOTA: Um sistema de partículas persistentes, uma vez criado, permanece residente na memória do sistema durante a duração do jogo, e assim, se você precisar recriá-lo a qualquer momento (por exemplo, se você chamar a ação Reiniciar Jogo ), então você deve destruir o sistema atual primeiro, caso contrário você terá um vazamento de memória. Um vazamento de memória acontece quando um recurso é adicionado à memória e depois "de-referenciado", significando que o GameMaker não tem como interagir com esse recurso não mais. Isto leva a que a memória fique entupida por recursos inacessíveis que, por sua vez, podem causar problemas com seu jogo. Para destruir um sistema de partículas persistentes quando não for mais necessário, use a ação Destroy Particle System (Destruir sistema de partículas).
