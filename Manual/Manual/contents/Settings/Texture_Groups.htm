Grupos de Textura

O Gerente de Grupos de Textura está disponível no menu Ferramentas no IDE. Aqui você pode adicionar, excluir e renomear Grupos de Textura.

O GameMaker permite que você atribua cada um dos gráficos assets (sprites, tilesets, e fonts) a diferentes grupos texture para tentar otimizar o número de swaps texture que o hardware faz a qualquer momento em seu jogo. Para que isso funcione, você precisa definir os grupos texture necessários a partir desta janela, e então passar por cada um dos tipos apropriados de asset e selecionar o grupo ao qual você deseja que ele pertença a partir do menu suspenso.

Alternativamente, você pode adicionar assets a um grupo texture através do botão direito do mouse no menu do Navegador de Ativos. Você pode selecionar um ou mais conjuntos de sprite/tilete/fonte assets e então usar o menu RMB para adicionar o assets a qualquer grupo, como mostrado abaixo para tilesets:

é importante entender que um único grupo de textura pode ter várias páginas de textura, e elas não são equivalentes. Se seu jogo tem muitos gráficos únicos para cada nível, então você pode definir um único grupo texture para cada um dos níveis, mas os gráficos podem requerer duas ou mais páginas texture por grupo. Isto depende se o tamanho da página texture selecionada (nas Opções de Jogo para seu alvo) é capaz de caber em todos os gráficos dentro de um grupo texture.

Por padrão todas as imagens em seu jogo serão adicionadas ao grupo "Default" texture, que sempre existe. Entretanto, você pode clicar no botão Adicionar novo para criar um novo grupo texture, e se você selecionar o nome do grupo, você pode mudar algo apropriado. Para começar com o grupo mostrará "este grupo está vazio" para cada seção do grupo (sprites, tilesets e fonts), e então você precisa passar pelo seu gráfico assets e adicioná-los a um grupo. Você pode adicionar um recurso usando o botão Adicionar Recurso na parte inferior, ou selecionando o grupo texture do editor de recursos para a imagem, tile set, ou font. Texture outros grupos além do "Padrão" também podem ser excluídos a qualquer momento, o que irá mover seu conteúdo para o grupo "Padrão" (note que se você excluir um grupo por engano você pode usar / + "Z" para desfazer a ação).

Uma vez que você tenha definido grupos texture - e atribuído ativos gráficos a eles - você os verá listados à esquerda da janela principal do editor de grupos Texture, e clicando em qualquer um dos sprites desta lista os mostrará na janela de visualização de imagens à direita. Isto é particularmente útil quando se trabalha em grandes projetos e você precisa ser capaz de ver quais imagens foram atribuídas a qual grupo. Observe que se as imagens atribuídas ao grupo tiverem múltiplas subimagens (ou seja: são animadas), então somente a primeira subimagem é mostrada na janela de pré-visualização. Você também pode clicar com o botão direito do mouse em um recurso para abrir um pequeno menu dando-lhe a opção de movê-lo para outro grupo texture ou abrir o editor para aquele asset.

Vale notar que mesmo que o assets seja mostrado na lista como sendo separado, todos sprites, tilesets e fonts serão adicionados à mesma página texture (ou páginas), e não obterão uma página individual para cada tipo de recurso. A única exceção a esta regra é quando um sprite asset foi marcado como sendo para uma "Página Separada Texture ", neste caso será dada sua própria página individual texture (veja a seção sobre "Configurações de Textura" no Editor Sprite para mais detalhes).

Configurações do grupo

As configurações de grupo no Editor de Grupo Texture são usadas para determinar exatamente como as páginas texture para um determinado grupo devem ser criadas, com as seguintes opções disponíveis:

  • Permitir a escala - Desmarcar isto forçará o GameMaker a não escalar o grupo texture sob nenhuma circunstância. Isto pode ser útil, especialmente quando se trata de aplicativos universais, pois permite colocar recursos específicos como fonts em um grupo próprio para ser usado quando o aplicativo é executado em um dispositivo específico. Por exemplo, fonts pode ficar ótimo em um display de retina, mas quando são escalados para baixo para desenhar em um display normal podem parecer terríveis para que você possa atribuí-los a um grupo especial sem escalas - se estiver desenvolvendo para o iPhone, você pode desejar que isto seja verificado, já que os aplicativos foram rejeitados devido ao fato de as imagens terem sido embaçadas pelo texture escalonamento. Isto é sinalizado como por padrão.
  • Automatically Crop - Quando você adiciona imagens no GameMaker, se elas têm algum "espaço" ao redor delas (ou seja, 100% de transparência) isto é automaticamente cortado quando é adicionado na página texture e "Automatically Crop" é sinalizado. Entretanto, se, por exemplo, você estiver usando a imagem como um texture não-repetente para primitivas ou qualquer coisa semelhante, isto significa que as coordenadas UV para o mapeamento texture não serão corretamente representadas. Nestes casos, você pode desmarcar esta opção e o sprite será adicionado à página texture "como está", mantendo quaisquer transparências que ela possa ter ao redor das bordas. Isto é sinalizado como por padrão.
  • Gerar Mipmaps - Marcar esta opção (está desligada por padrão) dirá ao GameMaker que a página original texture deve ser escalada e filtrada em múltiplas resoluções dentro do arquivo texture. Isto normalmente está desativado, mas se você estiver usando câmeras para aumentar ou diminuir o zoom em grandes áreas do jogo room, ou se você estiver trabalhando com um jogo 3D, então você pode querer ativar isto e então usar as funções de Mipmapping para ajustar como o jogo lida com os diferentes níveis de mipmap. Quando você flag, esta caixa de seleção será ajustada para 8 (se foi ajustada para menos de 8 originalmente). Isto porque, ao usar mipmaps, você precisa de bordas maiores entre as imagens nas páginas texture caso contrário, elas começam a sangrar juntas em níveis mais altos de mipmap. O valor padrão de 8 permitirá que a imagem seja reduzida a cerca de um oitavo de seu tamanho original antes que as sub-imagens comecem a sangrar juntas, mas você pode alterar isso para qualquer valor que seja necessário para atender às suas necessidades. Note que se você estiver usando a opção "Separate Texture Page" para sprites, então você não precisa passar por cada texture aqui e flag para gerar mipmaps, mas ao invés disso você pode ir para as Opções Gerais de Jogo e flag a opção "Gerar mipmaps para páginas separadas texture ".
  • Tamanho da borda - O tamanho da borda mudará a quantidade de sobreposição (ou "pixel doubling") em torno das bordas de todas assets nas páginas geradas para o grupo texture, de modo que quando as imagens forem escalonadas, não haja lacunas (ou costuras) visíveis entre elas. O padrão é de 2 pixels, mas em casos extremos em que você está ampliando muito a vista para dentro ou para fora, então você pode precisar aumentar este valor. Note que o Tile Set Editor tem a opção de definir a borda de saída para tiles (e também fornece uma explicação da necessidade disso), e isso é separado da definição da borda aqui - basicamente, tiles são gerados com uma borda de saída para cada tile em um texture temporário, e isso é então adicionado à página texture com a borda dada aqui ao redor da borda externa. Observe também que a habilitação do Mipmaps definirá automaticamente o tamanho padrão da borda para a página texture para ajudar a evitar "sangramento" entre textures em níveis mais altos de mip, mas isto pode ser alterado.
  • Grupo Parent Texture - Aqui você pode escolher qualquer um dos grupos existentes para ser o grupo "pai" para o grupo atualmente selecionado. Os grupos "criança" irão inherit quaisquer mudanças feitas no grupo pai texture e são principalmente para gerenciamento de recursos quando se trabalha com jogos entre plataformas. Um grupo texture com um pai ainda é um grupo texture por direito próprio e irá gerar páginas texture iguais a qualquer outro grupo, mas atribuindo-lhes um pai permite que você exclua grandes pedaços para diferentes plataformas. Por exemplo, você pode adicionar uma carga completa de Android player sprites a um grupo "player", e Android enemy sprites a um grupo "enemy", depois fazer com que ambos usem um grupo "android" pai. Se você excluir então o grupo de pais "andróide" do Windows ou outra plataforma construída, os grupos de crianças também serão excluídos.
  • Texture Group Format - Aqui você pode escolher o formato para sua página texture, entre BZ2 + QOI, QOI e PNG. É recomendável manter a opção padrão, BZ2 + QOI, pois oferece a melhor compressão, mantendo a qualidade.

    Você pode escolher QOI se a opção BZ2 afetar a velocidade em uma plataforma alvo (por exemplo Android), ou PNG se você desejar manipular os PNGs exportados manualmente ou usar a opção PNGCrush.

Além dessas configurações, você também pode optar por exportar (ou não) qualquer grupo texture para qualquer plataforma, simplesmente marcando ou desmarcando as caixas apropriadas na seção Use Group On... da janela. Observe que o grupo <Default> texture é sempre exportado para cada plataforma, e você não pode checar/desmarcar nenhuma delas (somente os grupos personalizados texture permitem isso).

Os Grupos de Textura também estão vinculados às configurações. Isto significa que você pode definir texture grupos para cada configuração individual de seu jogo, e então definir as atribuições de recursos sprite, tile set ou font para estes diferentes grupos e será "lembrado" quando você mudar para uma configuração diferente. O mesmo acontecerá com quaisquer opções de exportação que você tenha escolhido na janela Texture Group, assim você pode definir os grupos texture para exportar para plataformas específicas com base em cada configuração.

Um uso para isto é, por exemplo, que você poderia fazer uma configuração de iPad e para isto criar páginas 2048x2048px texture (configurando isto através das Opções de Jogo). Você então atribuiria todos os gráficos assets ao grupo texture apropriado, e ao terminar, mudaria para uma configuração diferente (para HTML5, por exemplo) e a partir da janela do grupo Texture, criaria novos grupos texture que são 512x512px, atribuindo o assets a estes novos grupos. Há também um botão na parte inferior da página que permite copiar texture configurações de grupo de uma configuração para outra.

Finalmente, texture grupos são relevantes apenas para imagens bitmap, e grupos não-bitmap sprites não serão atribuídos a grupos texture. Alguns deles já terão sua própria página exclusiva texture (animação do esqueleto sprites), enquanto outros simplesmente não precisam de textures, pois são compostos de vetores.
