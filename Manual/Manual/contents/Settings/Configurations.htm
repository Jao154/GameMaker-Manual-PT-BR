Configurações

Devido às exigências do desenvolvimento de plataformas cruzadas e à necessidade de que um produto esteja disponível e personalizado para diferentes clientes ou dispositivos ou plataformas, o GameMaker inclui configurações para tornar as coisas mais simples. Usando este sistema, você pode rapidamente switch entre diferentes telas splash, ícones e arquivos incluídos e testar ou exportar apenas aqueles que são relevantes para a plataforma ou cliente alvo a qualquer momento, tudo a partir do mesmo código base.

Para poder lidar com as diferentes demandas dos clientes e plataformas você pode usar o Editor de Configuração que é acessível a partir do menu Extras no Navegador de Ativos:

Todos os jogos feitos no GameMaker começam com uma configuração "padrão", que é listada no editor quando você o abre pela primeira vez:

Para configurar suas configurações, é recomendável que você primeiro prepare esta configuração "padrão", incorporando nela toda a configuração assets para o projeto. Para fazer isso, basta passar pela seguinte lista de opções e configurá-las para o que é exigido por padrão:

  • Opções de extensão: Configurar os vários campos "Copiar para..." para as plataformas necessárias para a configuração. Qualquer arquivo "Proxy" criado aqui também terá sua plataforma "Target" vinculada à configuração.
  • Editor Sprite: Defina o grupo texture ao qual você quer que o sprites seja adicionado.
  • Editor de Ladrilhos: Defina o grupo texture ao qual você quer que o tilesets seja adicionado.
  • Editor de fontes: Defina o grupo texture ao qual você quer que o fonts seja adicionado.
  • Gerente do Grupo de Textura: Aqui você pode definir qualquer uma das propriedades em uma base por configuração, incluindo o grupo pai ao qual pertence um grupo texture.
  • Gerente do Grupo de Áudio: Os sons que são designados para os diferentes grupos de áudio são confiúveis aqui.
  • Arquivos incluídos: Defina o campo "Copiar para..." para as plataformas necessárias para a configuração.

Uma vez que você tenha configurado a configuração padrão, você pode então adicionar uma nova ao Editor de Configuração clicando com o botão direito do mouse na pasta de configurações e selecionando Adicionar Configuração, ou clicando no botão "Criar" na parte inferior da janela. A nova configuração será mostrada como uma "criança" da configuração padrão - todas as configurações definidas pelo usuário são crianças da configuração padrão - e você pode dar a ela um nome apropriado.

Para editar a configuração, você deve clicar duas vezes sobre ela ou usar o botão direito do mouse opção do menu "Set Config", que selecionará a configuração no editor e, uma vez que isso tenha sido feito, você pode ir em frente e personalizá-la. Isto é feito simplesmente passando pelas opções listadas acima e mudando-as para as configurações desejadas para a configuração recém-criada. O GameMaker armazenará estes valores automaticamente para a configuração, e você pode mudar entre as configurações a qualquer momento simplesmente usando a opção do menu RMB ou clicando duas vezes em .

Outras configurações podem ser adicionadas como filhos da configuração padrão, ou filhos das configurações definidas pelo usuário que você criou, dependendo de qual configuração foi destacada no editor quando você a criou. Na imagem de exemplo abaixo, você pode ver que temos três configurações personalizadas definidas, duas das quais são filhos da configuração padrão, e outra que é filha de uma configuração definida pelo usuário:

Vale notar que você pode usar qualquer configuração com qualquer plataforma de destino ao compilar ou testar seus projetos. Isto é feito selecionando a configuração desejada na seção Config do Gerenciador de destino:

Você também pode abrir o Editor de Configuração em qualquer tme do Gerenciador de destino clicando no botão de menu no canto superior direito (como destacado na imagem acima).

Uma vez que você tenha definido uma configuração a ser usada desta forma, a qualquer momento você compilará para um alvo, seja para testes ou para criar um pacote final asset, a configuração escolhida será usada. Para mais informações sobre a Janela de Alvos, consulte a seção sobre Compilação.
