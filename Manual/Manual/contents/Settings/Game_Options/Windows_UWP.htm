Opções de jogo Windows UWP

Esta seção descreve as diferentes opções disponíveis que controlam como seus projetos de jogo UWP (Plataforma Universal Windows ) serão compilados. As diferentes seções são:

Geral

A seção geral das opções do UWP é onde você pode definir as Informações do Produto:

  • Nome de exibição: Especifica o nome amigável para o aplicativo que é exibido aos usuários (este string é localizável).
  • Nome do pacote: Este é o nome do pacote que você criou e deve corresponder ao usado para a chave do Desenvolvedor que você criou para o jogo (veja a seção sobre Embalagem, abaixo).
  • Nome do display da embalagem: Este é o "nome curto" da embalagem e deve idealmente ser o mesmo que o nome de exibição
  • Nome da editora Display Name: Este é o nome de sua empresa ou da pessoa que você deseja associar como editora de jogos.
  • Versão: O número da versão do seu jogo.

Depois de configurar isso, você pode selecionar as diferentes Orientações nas quais seu jogo pode ser executado para obter a melhor experiência de usuário. As opções disponíveis são:

  • Paisagem
  • Retrato
  • Landscape-flipped
  • Portrait-flipped

Por padrão, todos eles serão selecionados e você deve selecionar aqueles que não forem necessários.

Após definir as opções de orientação, você pode então sinalizar as diferentes permissões que seu aplicativo UWP precisará a partir das seguintes opções:

  • Habilitar a permissão de Internet: Flags o jogo como tendo acesso à Internet. Isto está desativado por padrão.
  • Habilite a permissão de microfone: Flags o jogo como tendo acesso ao microfone do dispositivo. Isto é desligado por padrão.
  • Habilite o IAP Sandbox: Flags o jogo como usando sandboxed In App Purchases. Isto está desativado por padrão.

Famílias de dispositivos

Aqui você pode selecionar qual dispositivo Famílias você quer que o Windows UWP exporte para o alvo ao criar seu executável. Por padrão, a opção "Support Universal device family" é marcada, o que significa que o aplicativo será direcionado a todos os dispositivos e famílias. A fim de restringir o aplicativo somente a famílias de dispositivos específicos, a opção "Support Universal" deve ser desmarcada e as famílias de dispositivos apropriadas devem ser checadas. Também é possível suportar a família Universal de dispositivos, mas restringir outras famílias de dispositivos a apenas certas versões de plataforma - neste caso, você manteria a família Universal de dispositivos verificada, mas também verificaria a outra família de dispositivos para a qual você deseja suportar uma versão de plataforma diferente, e introduziria os detalhes da versão para aquela família nas caixas de texto apropriadas. Veja nesta página a seguinte página de suporte da Microsoft para saber como esta lógica funciona:

  • Família do Dispositivo Alvo: Exemplos

Em geral, você não vai querer tocar nas versões de plataforma alvo e mínima e deve deixá-las nos valores padrão, mas se você estiver tendo problemas de construção - especificamente, se você não puder instalar o Windows SDK definido por padrão - então você precisará alterá-las para o que você instalou. Se você não tiver certeza dos valores a serem definidos, então crie um novo projeto UWP em Visual Studio e dê uma olhada no que ele define os valores.

Para mais informações sobre Famílias de Dispositivos, consulte o seguinte documento da Microsoft:

  • Visão geral das famílias de dispositivos

Gráficos

As opções gráficas são aquelas que você deve configurar para determinar como seu jogo utilizará a placa gráfica de seu dispositivo UWP alvo. As seguintes opções estão incluídas para que você possa modificar:

  • Interpolar as cores entre os pixels: Ativa a interpolação linear, que basicamente "suaviza" os pixels. Para gráficos de pixels nítidos, ele deve estar desligado, mas se você tiver misturas alfa agradáveis e gráficos alisados é melhor deixá-lo ligado.
  • Cursor de exibição: Quando esta opção estiver ativada, o cursor padrão do SO será mostrado em seu jogo. Se esta estiver desabilitada, então enquanto o mouse estiver sobre a janela do jogo, não haverá cursor visível a menos que você tenha criado um como parte do projeto do jogo (isto está ligado por padrão).
  • Iniciar tela cheia: Se isto for verificado, o jogo começará no modo de tela cheia, caso contrário, começará em tela cheia. Isto é desligado por padrão.
  • Permitir a troca de tela cheia: Com isso, o usuário pode switch de tela cheia para janela e voltar usando os atalhos padrão Ubuntu OS shortcuts. Isto está desativado por padrão.
  • Escalonamento: Aqui você pode optar por manter a relação de aspecto (assim um 4:3 room será "letter boxed" em um 16:9) ou escalar completamente (esticando a imagem para caber na tela inteira).

    ADVERTÊNCIA! A desativação da superfície de aplicação desativará todas as opções de escalonamento definidas no Windows Game Options até que tenha sido ligada novamente. Veja a superfície de aplicação para mais detalhes.

Finalmente, há a opção de definir o tamanho da página de textura. O tamanho padrão (e o mais compatível) é 2048x2048, mas você pode escolher entre 256x256 até 8192x8192. Há também um botão marcado Pré-visualização que vai gerar as páginas texture para esta plataforma e depois abrir uma janela para que você possa ver como elas são. Isto pode ser muito útil se você desejar ver como as páginas texture estão estruturadas e para evitar ter páginas texture maiores (ou menores) do que o necessário. Para mais informações sobre as páginas texture, por favor, veja aqui.

NOTA: Esteja ciente de que quanto maior o tamanho da página texture, menos compatível seu jogo estará nos PCs com especificações mais baixas.

Imagens

Na primeira parte da seção de Imagens você tem a opção de definir os vários Tiles que seu jogo usará nos diferentes dispositivos em que o UWP pode rodar. Você pode definir o seguinte (e todos eles devem ser arquivos no formato *.png ):

  • A imagem do pequeno azulejo é a imagem do logotipo que aparece ao lado do nome do aplicativo na barra de busca e em outras áreas da IU. Ela deve ter 44 x 44 pixels.
  • A imagem média da telha deve ser uma imagem de 71x71 pixels.
  • A próxima imagem tile é o logotipo da Loja tile e deve ser de 50 x 50 pixels.
  • Atelha de imagem ampla é aquela que aparece quando o tile está em seu formato amplo e deve ter 310 x 150 pixels. Se esta imagem não for fornecida, o tile pode exibir apenas no formato quadrado e não pode aceitar notificações baseadas em tipos de modelos largos. Como o usuário pode decidir qual formato o tile utiliza, é recomendável incluir uma imagem ampla tile. Observe também que se uma imagem ampla tile for fornecida, o tile pode aparecer inicialmente em seu formato amplo.
  • A telha de imagem grande é o grande formato tile ao vivo e deve ter 310 x 310 pixels. Note que isto só será usado no sistema operacional desktop.

A próxima seção é para definir o logotipo do jogo. Este é essencialmente o ícone que seu jogo usará dentro do ecossistema UWP, e vale a pena notar que estas imagens podem ser transparentes .png arquivos, neste caso serão desenhados sobre a cor de fundo especificada nesta seção. Esta cor de fundo também é aplicada à cor do botão em qualquer caixa de diálogo de propriedade do aplicativo e à página de Descrição do aplicativo na Loja em dispositivos.

Após configurar o logotipo, você pode configurar a tela Splash para o jogo. Esta é a tela que será exibida quando você iniciar seu jogo, e não pode ser maior que 620 x 300 pixels e como o logotipo pode ser um arquivo .png transparente, neste caso a cor de fundo que você escolher será usada para ele.

Vale notar que o GameMaker possui uma ferramenta Gerador de Imagens de Projeto que pode ser usada para criar automaticamente todas as imagens necessárias para todas as diferentes plataformas alvo para as quais seu jogo está sendo compilado. Se você usar esta ferramenta, você deve revisar as imagens criadas para garantir que elas sejam o que você precisa.

Embalagem

O GameMaker fornece uma chave de desenvolvimento padrão para o desenvolvimento Windows UWP, mas é recomendável que você gere sua própria chave e aponte para ela aqui. Para fazer isso, crie um projeto vazio em Visual Studio e depois armazene em algum lugar seguro uma cópia do arquivo *.pfx gerado automaticamente que terá incluído no projeto vazio (esta é a chave do desenvolvedor). Você deve então criar um link para o arquivo *.pfx aqui.

Após o link ao arquivo, você deve clicar no botão Instalar associado e então seguir as instruções no prompt de comando que será aberto. Se você não fizer isso, o uso dos botões Run/Debug (ou pressionando F5) falhará e seu jogo não poderá ser testado na plataforma UWP (entretanto, se você usar Create Executable o processo de instalação da certificação será automaticamente iniciado se o *.pfx ainda não tiver sido registrado).

NOTA: Quando chegar a hora de carregar seu aplicativo na loja, você precisará obter o *.pfx correto para seu título. Isto é feito através da associação de seu aplicativo com a Windows Store em Visual Studio que faz com que Visual Studio gere um novo *.pfx especificamente para o aplicativo. Vale notar também que você pode associar qualquer aplicativo à Windows Store em Visual Studio, o que significa que você pode associar um projeto em branco ao aplicativo Windows Store e simplesmente copiar o *.pfx gerado para um local seguro que pode então ser usado dentro das Opções de Jogo UWP. Você pode encontrar instruções sobre como isto é feito na Base de Conhecimento YoYo Games.

Nesta seção você também precisa especificar o nome da editora que foi usado para gerar o certificado de assinatura. A entrada deve ser sempre precedida por "CN=", por exemplo: CN=YoYoGames.

Xbox Live

A seção Xbox Live das Opções de Jogo UWP é onde você pode ativar seu projeto UWP para uso no console do Xbox One. A ativação do projeto para o Xbox aqui permite que você faça o seguinte quando o projeto for executado em um Xbox One:

  • Acesse o Xbox Live de dentro do aplicativo
  • Usar a identidade do usuário Xbox Live de alguma forma no projeto (por exemplo, mostrando a gamertag ou a gamer pic)

Antes de poder fazer estas coisas, porém, você precisará ter certeza de ter clicado na caixa de seleção Habilitar Xbox Live, e se você faz parte do Programa de Criadores da Microsoft, você também deve verificar a opção Programa de Criadores App. Se você fizer parte do Programa de Criadores, você precisará então adicionar o Title ID e o Service Configuration ID (mais informações sobre estes podem ser encontradas na Microsoft).

Observe que atualmente para testar a funcionalidade do Xbox Live em seu projeto, você precisará usar Create Executable em vez de Run, pois a saída do UWP Xbox precisa usar o certificado de assinatura da loja Windows que você dá como parte das informações de embalagem (veja acima) em vez do certificado temporário fornecido com o GameMaker, que é o que será usado quando você acertar Run.

Uma vez habilitado o XBox Live, você poderá então ir em frente e usar as funções do XBox Live em seu projeto, mas note que para a funcionalidade mais básica você precisará fazer parte do Programa Criador e para a funcionalidade avançada você precisará de uma conta ID@XBox.
