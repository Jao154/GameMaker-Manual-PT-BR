O Depurador

O Debugger é uma ferramenta poderosa para verificar seu jogo, e é especialmente útil para rastrear erros e bugs em seu código, assim como para verificar se scripts é executado quando deveria ser, e se as variáveis e arrays contêm o valores que você espera, etc... Você pode iniciar o Debugger rodando seu jogo a partir da IDE GameMaker usando o botão Debug no topo do IDE, ou indo para o menu Construir e selecionando "Debug".

NOTA: Você pode ser solicitado por Windows Firewall (ou qualquer outro programa de firewall que você execute) a criar uma exceção de segurança para este módulo. Você deve fazer isto caso contrário, não funcionará corretamente. Isto se deve ao modo como o o Debugger funciona e é essencial para as plataformas móveis debugging.

A seguir explicamos para que serve cada seção do debugger:

Vista gráfica

A vista gráfica mostra uma exibição gráfica de como seu projeto está usando a memória e seu enquadramento ao longo do tempo:

O gráfico mostra o uso atual e médio da memória, bem como o FPS atual e médio ao longo do tempo, e você pode aumentar ou diminuir o zoom usando o / junto com a roda do mouse , e se você posicionar o cursor do mouse sobre qualquer um dos elementos do gráfico, ele mostrará o valor para aquele elemento no ponto selecionado.

A vista gráfica também mostrará certos eventos de depuração como pontos ao longo da extensão do gráfico. Estes eventos são gerados automaticamente pelo GameMaker em certos pontos da execução de um jogo para mostrar eventos importantes específicos, mas você também pode adicionar em seus próprios elementos gráficos usando as funções show_debug_message() e debug_event(). Os eventos do sistema serão marcados com uma cor diferente daqueles eventos que são acionados através de código:

Quando o jogo for realizado em debug e mousing sobre esses eventos, será mostrada uma lista de todos os eventos que foram acionados naquele ponto do jogo, e você tem a opção de puxar a lista para fora em uma janela separada para revisão clicando no texto "Clique para abrir em uma nova janela":

Visão dos recursos

A Visão de Recursos mostra uma lista de todos os recursos sendo utilizados no projeto que podem conter código ou GML Ações visuais. Esta seção funciona muito como o Navegador de Recursos e você pode expandir os itens nele para mostrar mais informações, por exemplo, os diferentes eventos para um object específico. Você pode então clicar duas vezes em em qualquer um dos recursos para abrir o código para ele na janela Visão de Código, onde você pode verificar os valores e adicionar ou remover pontos de interrupção.

Pilha de chamadas

Esta janela exibirá a pilha atual de chamadas de eventos, bem como o número de linha para o código que está sendo atravessado. A pilha de chamadas é essencialmente uma estrutura de dados "última entrada - primeira saída" que mostra a ordem de execução de todo o jogo. código tanto do corredor GameMaker - uma vez que debugging é feito usando a Máquina Virtual (VM) - como do que você escreveu para o jogo. O debugger precisará foram Pausadas antes que qualquer informação seja mostrada aqui.

Barra de ferramentas

A barra de ferramentas tem os controles debugger junto com algumas informações básicas como o uso de memória, o FPS atual (real) para o jogo e um indicador para mostrar se o debugger está atualmente conectado ou não (o indicador estará verde quando conectado a uma instância em andamento de seu projeto e vermelho de outra forma). Os controles disponíveis são:

Isto começa o jogo novamente depois de ter sido pausado (atalho de teclado "f5").
Isto fará uma pausa (pausa) no jogo.
Reinicia o jogo.
Pára o jogo.
Entre em um bloco de código ou script (atalho de teclado "f11").
Passe sobre um bloco de código ou script (atalho de teclado "f10").
Saia de um bloco de código ou script (atalho de teclado + "f11").
Ativar ou desativar atualizações em tempo real para as informações de depuração (isto está desativado por padrão).

Os três primeiros botões são usados para controlar seu jogo, permitindo que você o inicie, pause, reinicie ou pare a qualquer momento. Quando o jogo é pausado (ou clicando no botão Pausa ou a partir de um ponto de parada) você pode então usar os três botões de etapa para passar pelo seu código.

Quando falamos em "pisar" no código, queremos simplesmente dizer que você pode dizer ao seu jogo para executar a linha de código atual, depois "pisar" na próxima, permitindo que você execute todo o seu jogo uma linha de código de cada vez se você desejar. Normalmente você simplesmente usaria o botão Step In para fazer isso, mas você também tem um para pisar no código e outro para sair do código. O passo em frente é útil quando você tem um grande scripts para percorrer, pois ao clicar neste, o script inteiro ou bloco de código como um único passo e parar na próxima linha depois, enquanto que ao sair o levará para fora de um script (eles correrão, mas todos de uma só vez) e parar no próximo passo depois, quando o escopo mudar.

O ícone final para atualizações em tempo real permite que a janela de depuração atualize seus dados em tempo real, para que você possa ver como as variáveis mudam ou instâncias são criadas e destruídas à medida que acontecem. Note que isto pode afetar o desempenho dos jogos e que nem todas as informações estão disponíveis em tempo real. Por exemplo, o relógio Render State (veja a seção sobre Relógios) não pode ser visto em tempo real.

Vista de código

A visualização do código é onde podemos ver e rever diferentes scripts ou código de evento de nosso projeto à medida que ele é executado. Por padrão isto começa vazio, mas você pode clicar duas vezes em em qualquer asset a partir da Visão de Recursos para abri-la aqui. Você pode abrir vários eventos diferentes ou scripts também, e eles serão adicionados à Visualização de Código workspace como abas ao longo da parte superior.

Enquanto qualquer janela de código estiver aberta na Vista de Código, você pode clicar à esquerda na sarjeta (onde estão os números das linhas) para adicionar ou remover pontos de parada (veja mais abaixo nesta página para informações em breakpoints).

Você também pode usar o botão direito do mouse para abrir o Menu de Contexto, que também lhe permite adicionar ou remover breakpoints e copiar qualquer código selecionado para a prancheta.

Outra parte importante do debugging seu jogo é o uso de Breakpoints. Um breakpoint é um lugar no código de seu projeto no qual você deseja que ele pare para que possa ver o que está acontecendo. Com o Debugger funcionando, você pode adicionar ou remover pontos de parada a qualquer momento e em qualquer código ou bloco script clicando à esquerda na sarjeta (onde estão os números das linhas), ou você pode usar o botão direito do mouse para abrir o Menu de Contexto, que também permite adicionar ou remover pontos de parada:

Pontos de parada também podem ser definidos a partir do GameMaker IDE. Ao escrever seu projeto, eles podem ser adicionados ou removidos a qualquer momento usando f9 (ou usando o botão direito do mouse e selecionando "toggle breakpoint"), seja em uma ação na janela object (para GML Usuários visuais), ou em uma linha em uma janela de editor de código. Agora, quando você executar seu jogo no modo de depuração, ele fará uma pausa no ponto (ou pontos) que você tiver definido.

Os pontos de parada definidos no IDE são salvos junto com seu projeto e persistirão em várias execuções até serem removidos novamente, porém se você adicionar um breakpoint ao seu código no Debugger, isto será perdido após o fechamento do módulo, ou uma nova execução de teste será iniciada. Note que todo breakpoints aparecerá na Janela de Saída onde eles podem ser habilitados ou desabilitados ou até mesmo removidos.
