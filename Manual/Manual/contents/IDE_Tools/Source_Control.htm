Controle da fonte

Source Control Management (SCM) é o nome dado ao método de trabalho com sub-versões ou backups de seus projetos através de um repositório e fonte local. Basicamente, uma solução SCM é um pacote de software independente que controla todos os aspectos da manutenção, mudança e comparação das versões de seu projeto enquanto você trabalha nele. Isto é especialmente útil para aqueles que trabalham em equipe e precisam ser capazes de controlar quem faz o quê e não se preocupar em perder dados ou fazer mudanças que podem precisar ser desfeitas posteriormente, mas os indivíduos também podem se beneficiar deste sistema poderoso mas flexível.

GameMaker faz uso de Git para seu SCM, que deve ser instalado e configurado separadamente antes que o SCM possa ser usado dentro de GameMaker. Veja a Documentação de Git para informações mais específicas sobre Git.

Abaixo você pode encontrar links para um pequeno tutorial sobre como configurar este plugin e usar as ferramentas SCM com um projeto.

GameMaker mostrará um menu suspenso na Barra de Menu para Source Control, que será preenchido com as seguintes opções uma vez que Source Control tenha sido ativado para seu projeto (veja a seção Configurando o Plugin Git acima para detalhes):

Aqui delineamos cada uma das opções disponíveis (a maioria delas é explicada com mais profundidade nas seções de tutorial acima):

  • Criar Repositório de Projetos: Esta opção permite que você crie um repositório local no mesmo diretório que seus arquivos de projeto.
  • Repositório de Clones: Esta opção lhe permitirá clonar um repositório de uma fonte para um novo destino.
  • Comprometer mudanças: Com esta opção você pode encenar arquivos alterados em seu projeto e depois submetê-los ao repositório.
  • Mudanças de impulso: Após fazer um commit ou uma fusão, esta opção é usada para empurrar as mudanças para o repositório principal.
  • Mudanças de Puxar: Com esta opção, você pode atualizar o repositório local puxando os arquivos alterados do repositório principal.
  • Ver Histórico: Isto abrirá a janela do histórico onde você poderá visualizar todo o histórico de versões do projeto e optar por reverter o arquivo específico paths ou revisões inteiras.
  • Mostrar Conflitos: Isto abrirá a janela Conflitos e listará quaisquer arquivos conflitantes que possam existir no projeto, permitindo que você lide com eles através do GameMaker IDE ou usando uma Ferramenta de Integração específica.
