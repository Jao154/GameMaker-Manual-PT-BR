Criando um ativo Marketplace

Ao selecionar a opção Criar Pacote a partir do Menu do Mercado IDE você será levado para o seguinte espaço de trabalho: Aqui você deve primeiro selecionar a identidade da Editora que você deseja publicar o pacote asset como (que você deveria ter criado a partir de seu Painel da Editora), depois dar um Nome de Exibição e um Número de Versão. Um ID de pacote será então criado para você com base no nome de exibição e na conta do editor utilizada, mas você pode editar isto para ser algo mais se desejar, desde que mantenha o formato de URL inversa. Em seguida, você precisa escolher o tipo de pacote asset a ser criado:

  • Ativo: Esta é uma coleção de assets (sprites, scripts, objects, etc...) que você deseja vender no Mercado
  • Demonstração: Esta é uma demonstração para demonstrar algum conceito ou idéia de jogo
  • Tutorial: O asset a ser carregado é um tutorial, projetado para ser executado a partir do GameMaker IDE (veja esta página para mais detalhes)

Uma vez feito isso, você pode escolher qual assets do projeto você quer adicionar ao pacote Asset usando a seção à direita. Você pode adicionar todo seu assets, ou apenas alguns deles, bem como remover qualquer um que você possa ter adicionado por engano. Presume-se que você tenha criado o asset para ser carregado anteriormente no GameMaker e que o projeto atual contenha tudo o que você precisar carregar.

LEMBRE-SE! Para poder criar ou utilizar pacotes asset, você deve ter criado previamente um certificado de perfil de usuário. Você pode fazer isso a partir das Preferências Marketplace, que são explicadas aqui.

Quando você estiver satisfeito com a seleção de assets e quiser carregar seu pacote no Marketplace, basta clicar no botão Ok na parte inferior, e o pacote asset será criado e carregado no Marketplace. Quando o carregamento estiver completo, um novo workspace será aberto com uma visão do navegador da página asset para que você possa editar. Para maiores informações sobre como configurá-lo, consulte a seguinte página:

  • Listagem de um ativo

Se você precisa atualizar um Marketplace Asset, você faz mais ou menos o mesmo que fez quando criou o pacote original, só que agora você precisa especificar o pacote a ser atualizado a partir de uma lista suspensa de pacotes disponíveis:

Ao atualizar, você não poderá alterar a ID do pacote, e deve estar ciente de que o Número da Versão mostrado será o número da versão atual e você deve incrementá-lo conforme apropriado, caso contrário a atualização falhará (já que você não pode ter dois pacotes de ativos com o mesmo número de versão). Há também um campo onde você pode adicionar algum texto para o registro de mudanças do asset, e este será carregado junto com o asset e exibido na página Marketplace.

IMPORTANTE! Assets feito com GameMaker versão 2.3+ só estará disponível para aqueles usuários que estiverem usando a versão 2.3 e superior. Se você estiver atualizando um asset a partir de uma versão anterior, esse asset ainda estará disponível para os usuários que não tiverem atualizado e coexistirão com a versão 2.3+ atualizada.

Carregando embora a página da Web

Você pode fazer o upload assets através da própria página web Marketplace em vez de IDE, algo que pode ser necessário se o upload através do IDE falhou por qualquer razão. Para começar, você criaria o pacote e tentaria fazer o upload através do IDE como mostrado acima, e se ele falhou, então vá para a Janela de Saída e verifique no log uma linha que mostra onde o pacote que você está criando foi salvo antes de tentar o upload. Por exemplo:

No Windows:

Created package file:
C:\Users\WINDOWSUSERNAME\AppData\Roaming\GameMaker\GM_USERNAME\Marketplace\Packages\com.publishername.assetname.yymp

Em macOS:

Created package file:
/Users/MACUSERNAME/.config/GameMaker/GM_USERNAME/Marketplace/Packages/com.publishername.assetname.yymp

Você precisa navegar até esse local de arquivo e copiar o arquivo *.YYMP para um local seguro, pois este será o arquivo que você estará carregando para o Marketplace.

Neste ponto você precisa abrir o Marketplace em seu navegador e navegar até a lista asset para o asset e depois ir até a seção Versões, e então clicar no botão Upload Version, como mostrado na imagem abaixo:

Isto o levará para a página de upload da nova versão, onde você pode adicionar informações sobre a versão que está sendo carregada, se necessário, e também fornecer o arquivo clicando no botão Choose FIle e selecionando o arquivo *.YYMP previamente criado:

Ao adicionar o arquivo, as versões serão carregadas e mostrarão uma barra de "progresso", e quando o upload estiver concluído e estiver satisfeito com os detalhes, você poderá então clicar no botão Salvar para salvar o asset na listagem. Note que o asset estará na fase de Processamento por alguns minutos antes de poder ser publicado.
