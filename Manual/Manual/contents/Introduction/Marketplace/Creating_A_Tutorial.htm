Criando um Tutorial Marketplace

Criar um tutorial é semelhante a criar um pacote regular Marketplace asset , só que agora você tem que adicionar em arquivos específicos incluídos, bem como incluir uma página inicial de tutorial (e uma página de vídeo opcional). O Tutorial workspace terá o seguinte aspecto:Como explicado acima para o geral Marketplace Assets, você precisa selecionar uma editora e então dar um nome e uma identificação única ao seu tutorial (formato de URL reversa), e então no menu suspenso "type" selecione Tutorial. Isto lhe dará opções adicionais para adicionar um arquivo HTML base para iniciar o tutorial, bem como um arquivo de vídeo (opcional) e URL. O arquivo de vídeo deve usar o modelo fornecido pelo Marketplace - assim como o resto do tutorial - que você pode obter aqui, e quando o usuário carregar o tutorial, o vídeo será carregado em um quadro separado dentro do IDE. Note que as páginas que você dá para a página inicial e a página de vídeo são relativas paths dentro dos Arquivos Incluídos, e você pode abrir um explorador para escolher algo que não seja o padrão com o qual o GameMaker preenche os campos.

Na verdade, a criação de seu tutorial exigirá que você adicione Arquivos Incluídos em seu projeto tutorial, e estes devem ser adicionados de uma forma muito específica para que o IDE detecte corretamente quais arquivos devem ser usados para o tutorial. Temos, portanto, duas pastas principais:

  • TutorialConteúdo: Todos os seus arquivos do tutorial HTML, o CSS para estilizá-los e as imagens necessárias para eles (não as imagens necessárias para os recursos, mas as que serão exibidas no tutorial), bem como o modelo de vídeo, se necessário, vão para aqui. Você pode ter também subpastas se desejar dividir o tutorial em capítulos, por exemplo, mas você deve ter pelo menos um arquivo base HTML para ser usado como página inicial para o tutorial.
  • TutorialRecursos: Esta pasta tem mais duas subpastas para Imagens e Ativos Sonoros . Se seu tutorial exige que o usuário adicione um sprite (por exemplo) e você deseja incluí-lo no tutorial, então você colocaria o recurso de imagem na pasta Imagens. Estas pastas são importadas para o projeto tutorial e quando o usuário abre o editor sprite e vai adicionar uma imagem, o padrão path para a janela do explorador será para este local, tornando mais fácil para o usuário encontrar o assets necessário.

Para que seu tutorial funcione dentro do IDE você deve ter esta estrutura de pastas assim como uma página inicial HTML com um arquivo CSS para estilização. Há uma série de links especiais que podem ser usados dentro de seus projetos de tutorial, que saltarão para partes de um vídeo, ou abrir o manual em uma seção ou função específica, e todos eles são fornecidos nos modelos oficiais do Tutorial do Marketplace.

IMPORTANTE! Sugerimos fortemente que você baixe os Modelos de Tutorial do Marketplace antes de tentar criar seus próprios tutoriais. Eles contêm exemplos de todas as partes importantes do HTML, bem como CSS e imagens para recriar botões e menus dentro do tutorial, deixando você livre para se preocupar com o conteúdo em vez de como exibi-lo.

Uma vez configurados os arquivos incluídos e fornecidas as informações necessárias, você pode clicar em Ok e o tutorial será carregado para Marketplace, abrindo um workspace na lista asset ao terminar para que você possa editá-lo e adicionar ícones, textos, capturas de tela, etc... Uma vez que o tutorial asset tenha sido publicado, os usuários poderão encontrá-lo dentro da seção Tutoriais da Página Inicial, onde poderá ser baixado e instalado.
